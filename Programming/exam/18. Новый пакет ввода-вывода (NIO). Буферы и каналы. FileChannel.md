# 18. Новый пакет ввода-вывода (NIO). Буферы и каналы. `FileChannel`

## 💡 Общая идея NIO

`java.nio` — это **новый (New I/O)** подход к вводу-выводу, который отличается от `java.io`.

Главная идея:

```text
Stream → поток данных
NIO → работа через буферы и каналы
```

---

# 🧠 Основная модель NIO

Вместо последовательного чтения:

```text
Stream-based I/O
```

используется:

```text
Channel ↔ Buffer
```

---

## 📌 Два ключевых компонента

| Компонент | Роль |
|-----------|------|
| Buffer | хранит данные в памяти |
| Channel | источник/приёмник данных |

---

# 🟡 Buffer (буфер)

## Что это

`Buffer` — это **область памяти для временного хранения данных**.

---

## Основные реализации

```java
ByteBuffer
CharBuffer
IntBuffer
```

---

## Ключевые параметры Buffer

```text
capacity → размер буфера
position → текущая позиция
limit → граница работы
```

---

## Состояния буфера

### 📌 Запись (write mode)

```text
position → растёт
limit = capacity
```

---

### 📌 Чтение (read mode)

```text
flip()
```

---

## Пример

```java
ByteBuffer buffer = ByteBuffer.allocate(1024);
```

---

## Запись данных

```java
buffer.put((byte) 65);
buffer.put((byte) 66);
```

---

## Переключение режимов

```java
buffer.flip();
```

---

## Чтение

```java
byte b = buffer.get();
```

---

# 🔵 Channel (канал)

## Что это

`Channel` — это **двусторонний канал связи с источником данных**

---

## Отличие от потоков

| Streams | Channels |
|--------|----------|
| односторонние | двусторонние |
| последовательные | более гибкие |
| блокирующие | могут быть неблокирующими |

---

## Основные типы

```java
FileChannel
SocketChannel
DatagramChannel
```

---

# 📂 FileChannel

## Что это

`FileChannel` — канал для работы с файлами на уровне NIO.

---

## Особенности

✔ работает с байтами  
✔ использует `ByteBuffer`  
✔ поддерживает позиционирование  
✔ эффективен для больших файлов  

---

## Открытие FileChannel

```java
FileChannel channel = FileChannel.open(
    Path.of("a.bin"),
    StandardOpenOption.READ
);
```

---

## Чтение данных

```java
ByteBuffer buffer = ByteBuffer.allocate(1024);

channel.read(buffer);
```

---

## Чтение с обработкой

```java
buffer.flip();

while (buffer.hasRemaining()) {
    System.out.println(buffer.get());
}
```

---

## Запись данных

```java
FileChannel channel = FileChannel.open(
    Path.of("a.bin"),
    StandardOpenOption.WRITE
);

ByteBuffer buffer = ByteBuffer.allocate(1024);
buffer.put((byte) 10);

buffer.flip();
channel.write(buffer);
```

---

# ⚙ Дополнительные возможности FileChannel

## 1. Позиционирование

```java
channel.position(0);
```

---

## 2. Размер файла

```java
long size = channel.size();
```

---

## 3. Перенос данных (очень важно)

```java
channel.transferTo(...)
channel.transferFrom(...)
```

👉 быстрые копирования файлов

---

# ⚖ IO vs NIO

| Характеристика | IO | NIO |
|----------------|----|-----|
| модель | Stream | Channel + Buffer |
| работа | последовательная | блочная |
| производительность | ниже | выше |
| гибкость | низкая | высокая |
| неблокирующий режим | нет | да |

---

# 💥 Частые ошибки

### ❌ не использовать flip()

→ данные не читаются

---

### ❌ путать read/write режимы

→ buffer работает неправильно

---

### ❌ использовать NIO без понимания буферов

→ ошибки логики

---

# 🔥 Где используется NIO

✔ серверы  
✔ большие файлы  
✔ высоконагруженные системы  
✔ сетевые приложения  

---

# 🧠 Шпаргалка

* NIO = Buffer + Channel
* Buffer хранит данные в памяти
* Channel — путь к данным
* `FileChannel` работает с файлами
* `ByteBuffer` — основной тип буфера
* `flip()` переключает режим write → read
* NIO быстрее и гибче IO
