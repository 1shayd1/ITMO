# 7. Интерфейсы `Set` и `SortedSet`, их реализации. `HashSet` и `TreeSet`

## Общее понятие `Set`

`Set` — это коллекция, которая хранит **только уникальные элементы**.

👉 Главный принцип:

> если элемент уже существует — второй раз он не добавляется

---

## Внутренняя идея Set

Set не хранит “позиции” как List.

Он работает через:

- сравнение (`equals`)
- хеширование (`hashCode`)
- или сравнение (`compareTo`)

---

## Иерархия

```text
Collection
   ↓
   Set
   ├── HashSet
   ├── LinkedHashSet
   └── SortedSet → TreeSet
```

---

# 🔵 SortedSet

## Что это

`SortedSet` — это расширение `Set`, которое:

✔ хранит уникальные элементы  
✔ автоматически сортирует их  

---

## Основные возможности

- отсортированный порядок
- работа с диапазонами
- доступ к min / max элементам

---

## Реализация

👉 `TreeSet`

---

## Важно (экзамен)

`SortedSet` — это интерфейс, который добавляет:

```text
упорядоченность + навигацию по диапазонам
```

---

# 🟡 HashSet

## Что это

`HashSet` — реализация `Set` на основе **хеш-таблицы**.

---

## Как работает внутри (ВАЖНО)

### Добавление элемента:

1. вызывается `hashCode()`
2. вычисляется индекс bucket
3. проверяется коллизия через `equals()`
4. если нет совпадений → добавляется

---

## Структура внутри

```text
HashSet = HashMap (внутренне)
value = фиктивный объект (PRESENT)
```

👉 фактически это обертка над HashMap

---

## Сложность

| операция | сложность |
|----------|----------|
| add | O(1) |
| remove | O(1) |
| contains | O(1) |

(в среднем случае)

---

## Особенности

- порядок НЕ гарантируется
- допускается 1 null
- очень быстрый

---

## Требование

Объекты должны корректно реализовывать:

```java
equals()
hashCode()
```

---

# 🟢 TreeSet

## Что это

`TreeSet` — реализация `SortedSet` на основе:

👉 красно-черного дерева (Red-Black Tree)

---

## Как работает

- элементы вставляются в дерево
- поддерживается балансировка
- порядок всегда отсортирован

---

## Сложность

| операция | сложность |
|----------|----------|
| add | O(log n) |
| remove | O(log n) |
| contains | O(log n) |

---

## Основа сравнения

TreeSet использует:

### 1. Comparable
или
### 2. Comparator

---

## Пример

```java
Set<Integer> set = new TreeSet<>();

set.add(5);
set.add(1);
set.add(3);
```

👉 результат:
```
1 3 5
```

---

## Особенности TreeSet

- всегда отсортирован
- не допускает null (в большинстве случаев)
- требует сравнимость элементов

---

## Почему нельзя null

Потому что:

```java
compareTo(null) ❌
```

---

# ⚖ HashSet vs TreeSet

| характеристика | HashSet | TreeSet |
|----------------|--------|--------|
| структура | hash table | red-black tree |
| порядок | нет | да (sorted) |
| скорость | O(1) | O(log n) |
| сравнение | equals + hashCode | compareTo / Comparator |
| null | можно | обычно нельзя |

---

# ⚙ Внутренние различия (очень важно)

## HashSet:

```text
hashCode → bucket → equals check
```

## TreeSet:

```text
compareTo → left/right subtree
```

---

# 💥 Частые ошибки (экзамен)

### ❌ 1. Несогласованность equals/hashCode

```java
a.equals(b) == true
но hashCode разный
```

→ HashSet ломается

---

### ❌ 2. Неправильный Comparator

```java
compare(a,b) == 0
но элементы разные
```

→ TreeSet удаляет элемент как дубликат

---

### ❌ 3. Использование mutable объектов

```java
set.add(user);
user.age = 100; // ❌ ломает структуру
```

---

# 🔥 Типовой вопрос преподавателя

❓ Почему HashSet быстрее TreeSet?

✔ потому что:
- HashSet использует O(1) доступ через хеш
- TreeSet использует O(log n) дерево

---

# 📌 Когда что использовать

## HashSet:
- максимальная скорость
- порядок не важен

## TreeSet:
- нужен отсортированный набор
- нужны min/max/диапазоны

---

# 🧠 Шпаргалка

* `Set` = уникальные элементы
* `HashSet` = hash table, O(1), без порядка
* `TreeSet` = red-black tree, O(log n), сортировка
* `SortedSet` = расширение Set с сортировкой
* HashSet → equals + hashCode
* TreeSet → compareTo / Comparator
* HashSet быстрее, TreeSet упорядочен
