# 22. Выполнение запросов. `execute`, `executeQuery`, `executeUpdate`

## 💡 Общая идея

После создания SQL-запроса (`Statement` / `PreparedStatement`) его нужно выполнить.

Для этого в JDBC есть 3 основных метода:

- `executeQuery`
- `executeUpdate`
- `execute`

---

# 🟢 executeQuery

## Что это

`executeQuery` используется для SQL-запросов, которые **возвращают результат (`ResultSet`)**.

---

## Когда использовать

✔ `SELECT`

---

## Пример

```java
ResultSet rs = ps.executeQuery();
```

---

## Особенности

- всегда возвращает `ResultSet`
- используется только для запросов выборки данных

---

# 🟡 executeUpdate

## Что это

`executeUpdate` используется для запросов, которые **изменяют данные в базе**.

---

## Когда использовать

✔ INSERT  
✔ UPDATE  
✔ DELETE  
✔ DDL (CREATE, DROP, ALTER — зависит от СУБД)

---

## Пример

```java
int rows = ps.executeUpdate();
```

---

## Что возвращает

```text
количество затронутых строк
```

---

## Пример

```java
PreparedStatement ps =
    conn.prepareStatement("UPDATE users SET name = ? WHERE id = ?");

ps.setString(1, "Alex");
ps.setInt(2, 10);

int updated = ps.executeUpdate();
```

---

# 🔵 execute

## Что это

`execute` — универсальный метод, который используется, когда заранее неизвестен тип результата.

---

## Пример

```java
boolean result = ps.execute();
```

---

## Что возвращает

| Значение | Значение |
|----------|----------|
| true | результат — ResultSet |
| false | результат — update count |

---

## Когда использовать

✔ редко  
✔ динамические SQL-запросы  
✔ сложные сценарии  

---

# ⚖ Сравнение методов

| Метод | Используется для | Возвращает |
|------|------------------|------------|
| executeQuery | SELECT | ResultSet |
| executeUpdate | INSERT / UPDATE / DELETE | int |
| execute | универсальный | boolean |

---

# 🔗 Логика выбора метода

## Если есть SELECT
```text
→ executeQuery
```

## Если изменяются данные
```text
→ executeUpdate
```

## Если неизвестно
```text
→ execute
```

---

# 💥 Частые ошибки

### ❌ использовать executeUpdate для SELECT

→ ошибка или пустой результат

---

### ❌ использовать executeQuery для INSERT

→ SQLException

---

### ❌ не проверять результат execute()

→ потеря типа результата

---

# 🧠 Шпаргалка

* `executeQuery` → SELECT → ResultSet
* `executeUpdate` → изменение данных → int
* `execute` → универсальный метод
* чаще всего используются первые два
