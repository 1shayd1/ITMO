# 51. Аннотации и их использование

**Аннотации** — структурированные **метаданные**, прикрепляемые к элементам кода (классам, методам, полям, параметрам, типам). Сами по себе логику не меняют — их читают компилятор, инструменты и фреймворки.

## Что такое аннотация «под капотом»

* Объявление `@interface X` компилируется в **интерфейс**, неявно расширяющий `java.lang.annotation.Annotation`;
* **элементы** аннотации — это методы без параметров с ограниченными типами возврата: примитивы, `String`, `Class`, enum, другая аннотация и **одномерные массивы** перечисленного;
* элемент может иметь `default`-значение; элемент с именем `value()` позволяет краткую запись `@Role("admin")`.

```java
@Retention(RetentionPolicy.RUNTIME)
@Target({ElementType.TYPE, ElementType.METHOD})
@interface Role {
    String value();
    boolean audit() default false;
}

@Role(value = "admin", audit = true)
class AdminService { }
```

## Встроенные аннотации

* `@Override` — проверка компилятором, что метод действительно переопределяет;
* `@Deprecated` (атрибуты `since`, `forRemoval` с Java 9);
* `@SuppressWarnings("unchecked")`;
* `@FunctionalInterface`, `@SafeVarargs`.

## Мета-аннотации (аннотации для аннотаций)

| Мета-аннотация | Назначение |
|---|---|
| `@Retention` | срок жизни: `SOURCE` (только исходник, напр. `@Override`), `CLASS` (в .class, дефолт), **`RUNTIME`** (видна рефлексии) |
| `@Target` | куда применима: `TYPE`, `METHOD`, `FIELD`, `PARAMETER`, `CONSTRUCTOR`, `TYPE_USE`... |
| `@Documented` | попадает в Javadoc |
| `@Inherited` | наследуется подклассами (работает только для аннотаций **на классах**) |
| `@Repeatable` | аннотацию можно ставить несколько раз (компилятор упаковывает повторы в скрытую контейнерную аннотацию) |

## Чтение аннотаций рефлексией

Работает только с `RetentionPolicy.RUNTIME`:

```java
if (AdminService.class.isAnnotationPresent(Role.class)) {
    Role r = AdminService.class.getAnnotation(Role.class);
    String role = r.value();          // "admin"
}
// для @Repeatable: getAnnotationsByType(Role.class)
```

Аннотации доступны у `Class`, `Method`, `Field`, `Constructor`, `Parameter` (все реализуют `AnnotatedElement`).

## Обработка на этапе компиляции

**Annotation processing** (`javax.annotation.processing`): процессоры подключаются к `javac` и генерируют код/проверяют его по аннотациям. Так работают Lombok (`@Getter`, `@Builder`), MapStruct, Dagger.

## Типичные применения во фреймворках

`@Test` (JUnit — ищет методы рефлексией), `@Autowired`/`@Component` (Spring DI), `@Entity`/`@Id` (JPA), `@JsonProperty` (Jackson).

### Шпаргалка

* Аннотация = интерфейс, расширяющий `Annotation`; элементы — методы с default-значениями; `value()` даёт краткую запись.
* `@Retention(RUNTIME)` — обязательное условие чтения через рефлексию; `SOURCE`/`CLASS` рантайму не видны.
* `@Target` ограничивает место применения; `@Inherited` работает только на классах.
* `@Repeatable` реализован через скрытую контейнерную аннотацию; читать `getAnnotationsByType`.
* Два мира обработки: compile-time (annotation processors — Lombok) и runtime (рефлексия — Spring, JUnit, JPA).
