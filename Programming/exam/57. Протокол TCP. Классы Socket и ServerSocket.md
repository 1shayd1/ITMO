# 57. Протокол TCP. Классы Socket и ServerSocket

`Socket` и `ServerSocket` (пакет `java.net`) — классический **блокирующий** API Java для TCP. `ServerSocket` принимает подключения, `Socket` представляет установленное TCP-соединение с обеих сторон.

## `ServerSocket`

* `new ServerSocket(port)` — привязывается к порту и начинает слушать; конструктор `ServerSocket(port, backlog)` задаёт длину **очереди ожидающих подключений** (по умолчанию 50) — при переполнении новые подключения отклоняются;
* `accept()` — **блокируется** до подключения клиента и возвращает новый `Socket` для общения именно с этим клиентом; сам `ServerSocket` продолжает слушать порт;
* `setReuseAddress(true)` — позволяет перезапустить сервер сразу, не дожидаясь, пока ОС освободит порт из состояния `TIME_WAIT`.

## `Socket`

Представляет TCP-соединение. Клиент создаёт его напрямую: `new Socket(host, port)` — конструктор выполняет three-way handshake и блокируется до установления соединения (или исключения).

Обмен данными — через потоки:

* `getInputStream()` — читать байты от собеседника; `read()` возвращает **−1** при закрытии соединения с той стороны (EOF);
* `getOutputStream()` — отправлять байты.

Поверх «голых» потоков обычно надевают обёртки: `BufferedReader`, `PrintWriter`, `DataInputStream/DataOutputStream` или `ObjectInputStream/ObjectOutputStream`. Поскольку TCP — поток **без границ сообщений**, прикладной код вводит framing: строки с `\n`, префикс длины и т.п.

## Схема работы и пример

1. Сервер создаёт `ServerSocket` и слушает порт.
2. Клиент создаёт `Socket(host, port)` — устанавливается соединение.
3. Сервер получает свой `Socket` из `accept()`.
4. Стороны обмениваются данными через потоки и закрывают соединение.

```java
// Сервер (echo)
try (ServerSocket server = new ServerSocket(8080)) {
    while (true) {
        Socket client = server.accept();              // блокируется
        new Thread(() -> {
            try (client;
                 var in  = new BufferedReader(
                         new InputStreamReader(client.getInputStream()));
                 var out = new PrintWriter(client.getOutputStream(), true)) {
                String line;
                while ((line = in.readLine()) != null) // null = клиент закрылся
                    out.println("echo: " + line);
            } catch (IOException ignored) {}
        }).start();
    }
}

// Клиент
try (Socket socket = new Socket("localhost", 8080);
     var out = new PrintWriter(socket.getOutputStream(), true);
     var in  = new BufferedReader(
             new InputStreamReader(socket.getInputStream()))) {
    out.println("hello");
    System.out.println(in.readLine());
}
```

Поскольку `accept()` и `read()` блокируются, сервер обслуживает многих клиентов через **поток на клиента** (или, лучше, `ExecutorService` с пулом). Для тысяч соединений эта модель не масштабируется — тогда переходят на NIO.

## Полезные опции сокета

| Метод | Назначение |
|---|---|
| `setSoTimeout(ms)` | таймаут блокирующего `read()`/`accept()` → `SocketTimeoutException` вместо вечного ожидания |
| `setTcpNoDelay(true)` | отключить алгоритм **Нейгла** (буферизацию мелких пакетов) — меньше задержка |
| `setKeepAlive(true)` | TCP keep-alive зондирование «мёртвых» соединений |

Сокеты — ресурсы ОС, закрывать обязательно: **try-with-resources**.

### Шпаргалка

* `ServerSocket(port, backlog)` слушает порт; `accept()` блокируется и возвращает `Socket` для клиента.
* Клиент: `new Socket(host, port)` выполняет handshake.
* Обмен — `getInputStream()`/`getOutputStream()`; `read() == −1` (или `readLine() == null`) — конец соединения.
* TCP — поток без границ: framing (строки, длина-префикс) — забота приложения.
* Блокирующая модель → поток/пул на клиента; `setSoTimeout`, `setTcpNoDelay`, try-with-resources.
