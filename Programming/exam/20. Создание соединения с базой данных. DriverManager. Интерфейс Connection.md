# 20. Создание соединения с базой данных. `DriverManager`. Интерфейс `Connection`

## 💡 Общая идея

Перед выполнением SQL-запросов в JDBC нужно установить соединение с базой данных.

Для этого используется:

- `DriverManager` — создаёт соединение
- `Connection` — представляет активное соединение с БД

---

# 🔵 DriverManager

## Что это

`DriverManager` — класс, который управляет JDBC-драйверами и создаёт соединения с базой данных.

---

## Как работает

```text
URL + логин + пароль → DriverManager → Connection
```

---

## Пример подключения

```java
Connection conn = DriverManager.getConnection(
    "jdbc:mysql://localhost:3306/test",
    "user",
    "password"
);
```

---

## Роль DriverManager

✔ выбирает подходящий драйвер  
✔ устанавливает соединение  
✔ возвращает объект Connection  

---

# 🟢 Connection

## Что это

`Connection` — это **активное соединение с базой данных**.

---

## Основные возможности

### 📌 1. Создание SQL-запросов

```java
Statement st = conn.createStatement();
```

---

### 📌 2. Управление транзакциями

```java
conn.setAutoCommit(false);
```

---

### 📌 3. Фиксация изменений

```java
conn.commit();
```

---

### 📌 4. Откат изменений

```java
conn.rollback();
```

---

### 📌 5. Закрытие соединения

```java
conn.close();
```

---

# ⚙ Транзакции в Connection

## Режим по умолчанию

```text
autoCommit = true
```

👉 каждое SQL-операция фиксируется автоматически

---

## Отключение автокоммита

```java
conn.setAutoCommit(false);
```

---

## Пример транзакции

```java
try (Connection conn = DriverManager.getConnection(url, user, pass)) {

    conn.setAutoCommit(false);

    // SQL операции

    conn.commit();

} catch (Exception e) {
    conn.rollback();
}
```

---

# ⚠ Важные правила

## 1. Всегда закрывать Connection

✔ лучше использовать try-with-resources

```java
try (Connection conn = DriverManager.getConnection(...)) {
    // работа
}
```

---

## 2. Управлять транзакциями вручную

Используется когда:

- несколько SQL-операций должны быть атомарными

---

## 3. Обрабатывать ошибки

При ошибке обязательно:

```java
rollback()
```

---

# 🔗 Полная схема подключения

```text
DriverManager
      ↓
   Connection
      ↓
 SQL операции
```

---

# 💥 Частые ошибки

### ❌ не закрывают Connection

→ утечка ресурсов

---

### ❌ забывают rollback

→ неконсистентные данные

---

### ❌ используют без try-with-resources

→ проблемы с производительностью

---

# 🧠 Шпаргалка

* `DriverManager` создаёт соединение
* `Connection` — активное соединение с БД
* через Connection выполняются SQL операции
* `setAutoCommit(false)` включает ручные транзакции
* `commit()` фиксирует изменения
* `rollback()` откатывает изменения
* соединение нужно всегда закрывать
