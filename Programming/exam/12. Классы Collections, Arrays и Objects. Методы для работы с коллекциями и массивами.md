# 12. Классы `Collections`, `Arrays` и `Objects`. Методы для работы с коллекциями и массивами

## 💡 Общая идея

Это **utility-классы** (вспомогательные классы), которые:

- не хранят данные
- содержат только `static` методы
- упрощают работу с коллекциями, массивами и объектами

---

# 🔵 `Collections`

## Что это

`Collections` — утилитный класс для работы с интерфейсом `Collection` и его реализациями.

---

## Основные методы

### 📌 Сортировка и порядок

```java
Collections.sort(list)
Collections.reverse(list)
Collections.shuffle(list)
```

---

### 📌 Поиск

```java
Collections.min(list)
Collections.max(list)
Collections.binarySearch(list, key)
```

---

### 📌 Обертки

```java
Collections.unmodifiableList(list)
Collections.unmodifiableSet(set)
Collections.unmodifiableMap(map)
```

👉 делает коллекцию **read-only view**

---

```java
Collections.synchronizedList(list)
Collections.synchronizedSet(set)
Collections.synchronizedMap(map)
```

👉 потокобезопасные обертки

---

## Пример

```java
List<Integer> list = new ArrayList<>(List.of(3, 1, 2));

Collections.sort(list);
System.out.println(list); // [1, 2, 3]
```

---

## Важно (экзамен)

👉 `binarySearch` работает ТОЛЬКО на отсортированном списке

---

## Внутренний момент

```text
sort → TimSort (O(n log n))
```

---

# 🟡 `Arrays`

## Что это

`Arrays` — утилиты для работы с **массивами (T[])**

---

## Основные методы

### 📌 Сортировка и поиск

```java
Arrays.sort(arr)
Arrays.binarySearch(arr, key)
```

---

### 📌 Работа с массивами

```java
Arrays.copyOf(arr, newLength)
Arrays.fill(arr, value)
Arrays.equals(a, b)
Arrays.toString(arr)
```

---

### 📌 Stream API

```java
Arrays.stream(arr)
```

---

## Пример

```java
int[] a = {3, 1, 2};

Arrays.sort(a);
System.out.println(Arrays.toString(a)); // [1, 2, 3]
```

---

## Внутренний момент

- для примитивов → dual-pivot quicksort
- для объектов → TimSort

---

## Важно

```text
Arrays.equals ≠ ==
```

`==` сравнивает ссылки, а `equals` сравнивает содержимое

---

# 🟢 `Objects`

## Что это

`Objects` — утилитный класс для **безопасной работы с объектами и null**

---

## Основные методы

### 📌 Null-check

```java
Objects.requireNonNull(obj)
```

👉 выбрасывает NullPointerException если null

---

### 📌 Null-safe сравнение

```java
Objects.equals(a, b)
```

👉 безопасно даже если a или b = null

---

### 📌 Hash

```java
Objects.hash(a, b, c)
```

👉 удобное создание hashCode

---

### 📌 toString с fallback

```java
Objects.toString(obj, "default")
```

---

## Пример

```java
String name = Objects.requireNonNull(inputName);
```

---

## Почему это важно

Раньше писали:

```java
if (obj == null) throw new NullPointerException();
```

Теперь:

```java
Objects.requireNonNull(obj);
```

---

# ⚖ Сравнение классов

| Класс | Работа с чем | Основные задачи |
|------|-------------|----------------|
| `Collections` | Collection API | сортировка, поиск, обертки |
| `Arrays` | массивы | сортировка, копирование |
| `Objects` | любые объекты | null-check, equals, hash |

# 💥 Типовые экзаменационные вопросы

## ❓ Чем отличается Collections от Arrays?

✔ Collections:
- работает с List/Set/Map
- использует интерфейсы коллекций

✔ Arrays:
- работает с T[]
- более низкий уровень

---

## ❓ Почему binarySearch требует сортировки?

✔ потому что алгоритм основан на делении диапазона пополам

---

## ❓ Зачем нужен Objects.equals?

✔ чтобы избежать NullPointerException:

```java
a.equals(b) ❌
Objects.equals(a, b) ✔
```

---

# 🧠 Шпаргалка

* `Collections` = операции над коллекциями
* `Arrays` = операции над массивами
* `Objects` = null-safe утилиты
* `binarySearch` требует сортировки
* `unmodifiable` = read-only view, не копия
* `Objects.equals` безопаснее чем `a.equals(b)`
