# 13. Байтовые и символьные потоки ввода-вывода. Базовые классы и их потомки

## 💡 Общая идея IO в Java

В Java вся система ввода-вывода построена на потоках (streams):

- поток = последовательность данных
- данные читаются/записываются последовательно

Есть 2 большие группы:

```text
1) Byte streams   → работа с байтами
2) Character streams → работа с символами
```

---

# 🔵 Байтовые потоки (Byte Streams)

## Базовые классы

```java
InputStream   // чтение байтов
OutputStream  // запись байтов
```

---

## Что читают/пишут

👉 сырые данные (bytes):

- изображения
- аудио
- видео
- любые бинарные файлы

---

## Основные потомки

### 📌 Файлы

```java
FileInputStream
FileOutputStream
```

---

### 📌 Память

```java
ByteArrayInputStream
ByteArrayOutputStream
```

---

### 📌 Буферизация (важно!)

```java
BufferedInputStream
BufferedOutputStream
```

👉 ускоряют работу за счет буфера

---

## Пример (байтовое чтение)

```java
try (InputStream in = new FileInputStream("data.bin")) {

    int b;

    while ((b = in.read()) != -1) {
        System.out.println(b);
    }
}
```

---

## Важный момент

```text
read() → возвращает int (0–255 или -1)
```

---

# 🟢 Символьные потоки (Character Streams)

## Базовые классы

```java
Reader   // чтение символов
Writer   // запись символов
```

---

## Что читают/пишут

👉 текстовые данные:

- строки
- файлы .txt
- JSON / XML (логически)

---

## Почему они нужны

Потому что учитывают:

```text
кодировку (UTF-8, UTF-16 и т.д.)
```

---

## Основные потомки

### 📌 Файлы

```java
FileReader
FileWriter
```

---

### 📌 Буферизация

```java
BufferedReader
BufferedWriter
```

---

### 📌 Мост между потоками

```java
InputStreamReader
OutputStreamWriter
```

👉 превращают byte → char и наоборот

---

## Пример (символьное чтение)

```java
try (Reader r = new FileReader("text.txt")) {

    int ch;

    while ((ch = r.read()) != -1) {
        System.out.print((char) ch);
    }
}
```

---

## Более правильный вариант (буфер)

```java
try (BufferedReader br =
         new BufferedReader(new FileReader("text.txt"))) {

    String line;

    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
}
```

---

# ⚖ Байтовые vs Символьные потоки

| Характеристика | Byte Streams | Character Streams |
|----------------|-------------|------------------|
| Базовые классы | InputStream / OutputStream | Reader / Writer |
| Единица данных | byte | char |
| Кодировка | нет | есть |
| Применение | бинарные файлы | текст |
| Примеры | images, video | txt, json |

---

# ⚙ Важные архитектурные идеи

## 1. Decorator pattern

Потоки можно “оборачивать”:

```java
BufferedReader(
    InputStreamReader(
        FileInputStream
    )
)
```

---

## 2. Набор слоев

```text
File → Stream → Reader → Buffered
```

---

# 💥 Частые ошибки

### ❌ Использовать byte stream для текста

→ приводит к проблемам с кодировкой

---

### ❌ Не использовать buffering

→ сильное падение производительности

---

### ❌ Читать char по одному

→ медленно, лучше readLine()

---

# 🔥 Типовой вопрос преподавателя

## ❓ Почему есть два типа потоков?

✔ потому что:

- байты = универсальный формат хранения
- символы = удобство работы с текстом + кодировки

---

## ❓ Почему Reader/Writer не работают с изображениями?

✔ потому что изображения — это не символы, а байты

---

# 🧠 Шпаргалка

* `InputStream/OutputStream` → байты
* `Reader/Writer` → символы
* текст → всегда через Reader
* файлы/медиа → через Stream
* `Buffered*` ускоряет работу
* `InputStreamReader` = мост byte → char
