# 16. Сериализация объектов. `Serializable`. Модификатор `transient`

## 💡 Общая идея

**Сериализация** — это процесс преобразования объекта в поток байтов.

👉 используется для:

- сохранения в файл
- передачи по сети
- кэширования

---

## 🔄 Процесс

```text
Object → bytes (serialization)
bytes → Object (deserialization)
```

---

# 🔵 Интерфейс `Serializable`

## Что это

`Serializable` — это **маркерный интерфейс**:

```java
public interface Serializable {}
```

👉 он не содержит методов

---

## Зачем он нужен

Он сообщает JVM:

```text
"этот объект можно сериализовать"
```

---

## Пример

```java
import java.io.Serializable;

class User implements Serializable {
    private String name;
    private int age;
}
```

---

## Как работает сериализация

Используются потоки:

```text
ObjectOutputStream → запись объекта
ObjectInputStream  → чтение объекта
```

---

## Пример записи

```java
try (ObjectOutputStream out =
         new ObjectOutputStream(new FileOutputStream("user.bin"))) {

    User user = new User();
    out.writeObject(user);
}
```

---

## Пример чтения

```java
try (ObjectInputStream in =
         new ObjectInputStream(new FileInputStream("user.bin"))) {

    User user = (User) in.readObject();
}
```

---

# 🟡 `transient`

## Что это

`transient` — модификатор, который **исключает поле из сериализации**

---

## Пример

```java
class User implements Serializable {

    private String name;

    private transient String password;
}
```

---

## Что произойдет

При сериализации:

```text
name → сохраняется
password → НЕ сохраняется
```

После десериализации:

```text
password = null
```

---

## Зачем нужен `transient`

✔ безопасность (пароли, токены)  
✔ временные данные  
✔ кэшируемые значения  
✔ вычисляемые поля  

---

# ⚙ Важные особенности сериализации

## 1. static поля

```java
static int x;
```

❌ не сериализуются

👉 потому что принадлежат классу, а не объекту

---

## 2. зависимости объекта

Если объект содержит другие объекты:

```java
class A {
    B b;
}
```

👉 B тоже должен быть `Serializable`

---

## 3. serialVersionUID

Рекомендуется:

```java
private static final long serialVersionUID = 1L;
```

👉 для контроля совместимости версий класса

---

## 4. риск безопасности

❗ десериализация может быть опасной:

- подмена данных
- атаки через crafted objects

---

# ⚖ Serializable vs transient

| Элемент | Роль |
|--------|------|
| Serializable | включает сериализацию |
| transient | исключает поле |

---

# 💥 Типовые вопросы преподавателя

## ❓ Почему Serializable пустой?

✔ потому что это маркер для JVM

---

## ❓ Что будет с transient полем?

✔ оно не сохранится и станет null (или default)

---

## ❓ Сериализуются ли static поля?

✔ нет, потому что они не принадлежат объекту

---

# 🧠 Шпаргалка

* сериализация = объект → байты
* десериализация = байты → объект
* `Serializable` = маркерный интерфейс
* `transient` = исключает поле
* static поля не сериализуются
* используется для файлов и сети
