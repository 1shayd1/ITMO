# 17. Работа с файлами в Java. `Path`, `File`, `Files`, `Paths`

## 💡 Общая идея

В Java есть два API для работы с файлами:

```text
1) java.io  → старый API (File)
2) java.nio → современный API (Path, Files)
```

👉 сейчас предпочтение отдаётся `java.nio.file`

---

# 🟠 Класс `File` (java.io)

## Что это

`File` — это объект, который представляет:

- файл
- или директорию
- или путь (не обязательно существующий)

---

## Пример

```java
File file = new File("data.txt");
```

---

## Основные возможности

```java
file.exists()        // существует ли файл
file.isFile()        // это файл?
file.isDirectory()   // это папка?
file.length()        // размер
file.getName()       // имя
file.getPath()       // путь
file.delete()        // удалить
file.createNewFile() // создать файл
```

---

## Минусы File

❌ устаревший  
❌ ограниченные возможности  
❌ нет удобных операций (copy/move/read/write)

---

# 🟢 Path (java.nio.file)

## Что это

`Path` — это **современное представление пути к файлу**

```java
Path path = Path.of("data.txt");
```

---

## Особенности

✔ не зависит от ОС  
✔ более гибкий  
✔ используется с `Files`  

---

## Пример

```java
Path path = Path.of("folder", "data.txt");
```

👉 автоматически строит правильный путь

---

# 🔵 Files

## Что это

`Files` — утилитный класс со **статическими методами для работы с файлами**

---

## Основные операции

### 📌 Проверка

```java
Files.exists(path)
```

---

### 📌 Создание

```java
Files.createFile(path)
Files.createDirectory(path)
```

---

### 📌 Чтение / запись

```java
Files.readString(path)
Files.writeString(path, "Hello")
```

---

### 📌 Работа с файлами

```java
Files.copy(source, target)
Files.move(source, target)
Files.delete(path)
```

---

## Пример

```java
Path path = Path.of("a.txt");

Files.writeString(path, "Hello Java");

String text = Files.readString(path);
```

---

# 🟡 Paths (устаревшее создание Path)

## Что это

`Paths` — старый способ создания `Path`

```java
Path p = Paths.get("data.txt");
```

---

## Сейчас

✔ заменён на:

```java
Path.of("data.txt")
```

---

# ⚖ Сравнение классов

| Класс | Назначение | Статус |
|------|------------|--------|
| File | файл/папка (старый API) | устаревший |
| Path | представление пути | современный |
| Files | операции с файлами | основной инструмент |
| Paths | создание Path | устаревший |

---

# 🔥 Важная архитектура NIO

## Правильная модель:

```text
Path → Files → операции
```

---

## Пример полной работы

```java
Path path = Path.of("test.txt");

Files.writeString(path, "Hello");

String content = Files.readString(path);

System.out.println(content);
```

---

# 💥 Частые ошибки

### ❌ Использовать File вместо Path

→ устаревший API

---

### ❌ Пытаться читать файл через File

→ File НЕ читает данные

---

### ❌ Игнорировать IOException

→ почти все Files методы могут бросать исключения

---

# ⚙ Важные особенности

## 1. Path не зависит от ОС

```java
Path.of("a", "b", "c")
```

→ Windows / Linux совместимость

---

## 2. Files работает через OS API

- файловая система
- системные вызовы

---

## 3. Path immutable

```text
Path нельзя изменить после создания
```

---

# 🔥 Типовые вопросы преподавателя

## ❓ Почему File устарел?

✔ потому что:
- нет нормальных операций
- плохая масштабируемость
- заменён NIO

---

## ❓ Почему Path лучше File?

✔ потому что:
- immutable
- OS-independent
- работает с Files API

---

## ❓ Что делает Files?

✔ предоставляет операции над файловой системой

---

# 🧠 Шпаргалка

* `File` = старый API (устарел)
* `Path` = современный путь
* `Files` = операции с файлами
* `Paths` = старый способ создания Path
* лучший стек:
  ```text
  Path + Files
  ```
* `File` почти не используется в новом коде
