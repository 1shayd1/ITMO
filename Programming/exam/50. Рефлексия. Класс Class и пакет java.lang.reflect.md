# 50. Рефлексия. Класс Class и пакет java.lang.reflect

**Рефлексия** — механизм исследования и изменения структуры программы **во время выполнения**: классы, поля, методы, конструкторы, аннотации.

## `Class<T>` — точка входа

Для каждого загруженного типа JVM держит **ровно один** объект `Class` (на каждый класс-лоадер). Способы получить:

```java
Class<String> c1 = String.class;            // литерал, compile-time
Class<?> c2 = obj.getClass();               // от объекта
Class<?> c3 = Class.forName("java.util.ArrayList"); // по имени (инициализирует класс)
Class<?> arr = int[].class;  Class<Integer> prim = int.class;
```

`Class` отвечает на вопросы: имя, пакет, модификаторы, суперкласс, интерфейсы, аннотации.

## Контракт `getX` vs `getDeclaredX`

| Метод | Что возвращает |
|---|---|
| `getMethods()` / `getFields()` | только **public**, включая унаследованные |
| `getDeclaredMethods()` / `getDeclaredFields()` | **все** объявленные в классе (private тоже), **без** унаследованных |

## `java.lang.reflect`

* `Field` — `get(obj)` / `set(obj, value)`, `getType()`;
* `Method` — `invoke(target, args...)` (для static target = `null`; аргументы боксируются);
* `Constructor` — `newInstance(args...)`;
* `Modifier` — разбор битовой маски: `Modifier.isStatic(m.getModifiers())`;
* `Parameter`, `Array`, динамические прокси `Proxy`.

```java
Class<?> c = Class.forName("com.example.User");
Object user = c.getDeclaredConstructor().newInstance();   // Class.newInstance() устарел

Field f = c.getDeclaredField("name");
f.setAccessible(true);                 // доступ к private
f.set(user, "Nik");

Method m = c.getMethod("toString");
String s = (String) m.invoke(user);
```

## `setAccessible` и модули

* `setAccessible(true)` снимает проверку доступа к private-членам;
* в **JPMS** (Java 9+) глубокая рефлексия в чужой модуль работает только если пакет **открыт** (`opens`); с Java 16 строгая инкапсуляция по умолчанию → иначе `InaccessibleObjectException`. Именно поэтому фреймворкам в `module-info.java` пишут `opens com.example.model to ...`.

## Производительность и риски

* Рефлексивный вызов медленнее прямого: проверки доступа, боксинг аргументов; JVM «инфлирует» горячие `Method` в сгенерированные байткод-аксессоры, но накладные расходы остаются;
* современная быстрая альтернатива — `MethodHandle`/`VarHandle` (`java.lang.invoke`);
* риски: разрушение инкапсуляции, хрупкость (нет проверки компилятором, ошибки в рантайме), проблемы с безопасностью.

## Где используется

Каркасы и инструменты, которым нужно работать с **неизвестными на этапе компиляции** классами: DI-контейнеры (Spring), ORM (Hibernate), сериализация (Jackson), тестирование (JUnit находит `@Test`-методы рефлексией), `ServiceLoader`, дебаггеры.

### Шпаргалка

* `Class` — один объект на тип (в рамках класс-лоадера); получить: `X.class`, `getClass()`, `Class.forName()`.
* `getMethods` = public + унаследованные; `getDeclaredMethods` = все свои, без унаследованных.
* Создание: `getDeclaredConstructor().newInstance()`; `Class.newInstance()` устарел.
* `setAccessible(true)` открывает private, но в JPMS нужен `opens`, иначе `InaccessibleObjectException` (Java 16+).
* Рефлексия медленнее прямых вызовов; используется фреймворками (Spring, Hibernate, JUnit, Jackson).
