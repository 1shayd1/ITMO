# 4. Коллекции. Виды коллекций. Интерфейсы List, Queue, Deque, Set, Map

## Что такое коллекции

**Коллекции (Collections Framework)** — это набор интерфейсов и классов в Java (`java.util`), предназначенных для хранения, обработки и управления группами объектов.

Основные цели:

* единый API для структур данных;
* готовые эффективные реализации;
* уменьшение количества собственного кода.

---

## Общая иерархия коллекций

В Java существует две независимые ветки:

```text
Iterable
  ↓
Collection
  ├── List
  ├── Set
  └── Queue
         ↓
       Deque

Map (отдельная иерархия, НЕ наследуется от Collection)
```

---

## Интерфейс `List`

### Основные свойства:

* упорядоченная коллекция;
* допускает дубликаты;
* доступ по индексу;
* сохраняет порядок добавления.

### Реализации:

```text
ArrayList
LinkedList
Vector (устаревший)
```

### Пример:

```java
List<String> list = new ArrayList<>();

list.add("A");
list.add("A");
list.add("B");

System.out.println(list.get(0));
```

---

### Особенности `List`

* элементы имеют индекс;
* можно вставлять в середину;
* допускаются повторяющиеся значения.

---

## Интерфейс `Set`

### Основные свойства:

* хранит только уникальные элементы;
* дубликаты автоматически удаляются;
* порядок зависит от реализации.

### Реализации:

```text
HashSet        — без порядка
LinkedHashSet  — сохраняет порядок вставки
TreeSet        — сортированное множество
```

### Пример:

```java
Set<String> set = new HashSet<>();

set.add("A");
set.add("A");
set.add("B");

System.out.println(set);
```

---

### Особенности `Set`

* запрещены дубликаты;
* проверка уникальности через `hashCode()` и `equals()`;
* в `TreeSet` используется сортировка через `Comparable` / `Comparator`.

---

## Интерфейс `Queue`

### Основная идея:

**Очередь (FIFO — First In First Out)**

### Основные операции:

```java
offer()  // добавить
poll()   // извлечь
peek()   // посмотреть первый элемент
```

### Реализации:

```text
PriorityQueue
ArrayDeque
LinkedList
```

### Пример:

```java
Queue<String> queue = new ArrayDeque<>();

queue.offer("A");
queue.offer("B");

System.out.println(queue.poll()); // A
```

---

### Особенности `Queue`

* обычно FIFO;
* может быть приоритетной (`PriorityQueue`);
* не поддерживает доступ по индексу.

---

## Интерфейс `Deque`

### Основная идея:

**Double Ended Queue (двусторонняя очередь)**

Можно работать с обоими концами.

### Основные операции:

```java
addFirst()
addLast()

removeFirst()
removeLast()
```

### Реализации:

```text
ArrayDeque
LinkedList
```

### Пример:

```java
Deque<Integer> deque = new ArrayDeque<>();

deque.addFirst(1);
deque.addLast(2);

System.out.println(deque.removeFirst()); // 1
```

---

### Особенности `Deque`

* поддерживает стек и очередь одновременно;
* эффективные операции с обоих концов;
* часто используется вместо `Stack`.

---

## Интерфейс `Map`

### Основная идея:

Хранение данных в виде пар:

```text
key -> value
```

### Важно:

* ключи уникальны;
* значения могут повторяться;
* не наследуется от `Collection`.

---

### Реализации:

```text
HashMap        — без порядка
LinkedHashMap  — порядок вставки
TreeMap        — сортировка по ключу
Hashtable      — устаревшая синхронизированная версия
```

---

### Пример:

```java
Map<String, Integer> map = new HashMap<>();

map.put("Java", 1);
map.put("Python", 2);

System.out.println(map.get("Java"));
```

---

### Особенности `Map`

* работает с парами (entry);
* доступ по ключу;
* используется хеширование (`HashMap`) или дерево (`TreeMap`);
* ключи должны быть уникальными.

---

## Почему `Map` — отдельная иерархия

`Map` не является `Collection`, потому что:

* `Collection` работает с одиночными элементами;
* `Map` работает с парами `(key, value)`.

---

## Сравнение интерфейсов

| Интерфейс | Дубликаты | Порядок | Индексы | Основная структура |
|------------|-----------|---------|----------|-------------------|
| `List` | Да | Да | Да | последовательность |
| `Set` | Нет | зависит | Нет | уникальные элементы |
| `Queue` | Да | Да (FIFO) | Нет | очередь |
| `Deque` | Да | Да | Нет | двусторонняя очередь |
| `Map` | ключи — нет | зависит | Нет | ключ → значение |

---

## Пример использования всех типов

```java
List<String> list = new ArrayList<>();
list.add("A");

Set<Integer> set = new HashSet<>();
set.add(1);

Queue<String> queue = new ArrayDeque<>();
queue.offer("task");

Deque<Integer> deque = new ArrayDeque<>();
deque.addFirst(10);

Map<String, Integer> map = new HashMap<>();
map.put("key", 100);
```

---

## Внутренние механизмы (важно для экзамена)

### Hash-based структуры (`HashMap`, `HashSet`)

Используют:

* `hashCode()`
* `equals()`
* бакеты (buckets)

Сложность:

```text
O(1) — среднее
O(n) — худший случай
```

---

### Tree-based структуры (`TreeSet`, `TreeMap`)

Используют:

* красно-черное дерево
* сортировку

Сложность:

```text
O(log n)
```

---

## Шпаргалка

* `List` — порядок + индексы + дубликаты.
* `Set` — только уникальные элементы.
* `Queue` — FIFO очередь.
* `Deque` — очередь с двух концов.
* `Map` — ключ → значение, отдельная иерархия.
* `Map` не наследуется от `Collection`.
* Hash-структуры работают через `hashCode()` + `equals()`.
* Tree-структуры — отсортированные (O(log n)).
