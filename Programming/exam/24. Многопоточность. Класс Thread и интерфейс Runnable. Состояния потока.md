# 24. Многопоточность. Класс `Thread` и интерфейс `Runnable`. Состояния потока

## 💡 Общая идея

Многопоточность в Java позволяет выполнять несколько задач **параллельно или псевдопараллельно** в рамках одного процесса.

Основные инструменты:

- `Runnable` — описание задачи
- `Thread` — выполнение задачи в отдельном потоке

---

# 🟢 Интерфейс Runnable

## Что это

`Runnable` — функциональный интерфейс, который описывает задачу для выполнения в потоке.

```java
@FunctionalInterface
public interface Runnable {
    void run();
}
```

---

## Пример

```java
Runnable task = () -> System.out.println("Work");
```

---

## Особенность

✔ не создаёт поток  
✔ только описывает работу  

---

# 🔵 Класс Thread

## Что это

`Thread` — это класс, который представляет **поток выполнения**.

---

## Пример создания потока

```java
Thread t = new Thread(() -> System.out.println("Hello"));
t.start();
```

---

## ⚠ Важно: start() vs run()

### ❌ Неправильно

```java
t.run(); // просто обычный метод
```

### ✅ Правильно

```java
t.start(); // создаёт новый поток
```

---

## Как работает start()

```text
start()
  ↓
новый поток JVM
  ↓
run() выполняется параллельно
```

---

# 🔗 Способы создания потоков

## 1. Через Runnable

```java
Runnable task = () -> System.out.println("Task");
new Thread(task).start();
```

---

## 2. Через наследование Thread

```java
class MyThread extends Thread {
    public void run() {
        System.out.println("Hello");
    }
}

new MyThread().start();
```

---

## ⚠ Рекомендуется

✔ Runnable (гибче)  
❌ наследование Thread (реже используется)

---

# 🟡 Состояния потока (Thread States)

## Перечень состояний

### 🟢 NEW
```text
поток создан, но не запущен
```

```java
Thread t = new Thread();
```

---

### 🟡 RUNNABLE
```text
поток готов к выполнению или выполняется
```

---

### 🔴 BLOCKED
```text
ожидание монитора (synchronized)
```

---

### 🟠 WAITING
```text
ожидание без тайм-аута
```

Примеры:
- `wait()`
- `join()`

---

### 🟠 TIMED_WAITING
```text
ожидание с тайм-аутом
```

Примеры:
- `sleep(ms)`
- `wait(ms)`

---

### ⚫ TERMINATED
```text
поток завершил выполнение
```

---

## 📊 Жизненный цикл потока

```text
NEW
 ↓ start()
RUNNABLE
 ↓ выполнение run()
TERMINATED
```

(возможны промежуточные BLOCKED / WAITING состояния)

---

# 💥 Частые ошибки

### ❌ вызов run() вместо start()

→ нет нового потока

---

### ❌ ожидание без понимания состояния

→ зависания программы

---

### ❌ неправильная синхронизация

→ race condition / deadlock

---

# 🧠 Шпаргалка

* `Runnable` — задача (что делать)
* `Thread` — поток (как выполнять)
* `start()` запускает новый поток
* `run()` — обычный метод
* состояния: NEW → RUNNABLE → BLOCKED/WAITING → TERMINATED
* `Runnable` предпочтительнее `Thread`
