# 45. Контейнеры. Классы Container, JPanel, Parent, Region, Scene

**Контейнеры** — это компоненты графического интерфейса, которые могут содержать другие компоненты (дочерние элементы), организуя их в иерархическую структуру.

---

## 📊 Сравнение контейнеров по технологиям

| Технология | Базовый контейнер | Типовой контейнер | Корневой контейнер |
|------------|-------------------|-------------------|---------------------|
| **AWT** | `java.awt.Container` | `Panel` | `Frame`, `Dialog` |
| **Swing** | `java.awt.Container` (через `JComponent`) | `JPanel` | `JFrame`, `JDialog` |
| **JavaFX** | `javafx.scene.Parent` | `Region` (через `Pane`) | `Scene` + `Stage` |

---

## 1. AWT — `Container`

`Container` — прямой наследник `Component`. Базовый класс для всех AWT-контейнеров.

```java
// Основные методы Container
Container container = new Panel();

// Добавление/удаление компонентов
container.add(new Button("OK"));
container.remove(Component comp);
container.removeAll();

// Layout Manager
container.setLayout(new FlowLayout());
container.setLayout(new BorderLayout());

// Получение дочерних компонентов
int count = container.getComponentCount();
Component[] children = container.getComponents();
```

### Иерархия Container в AWT

```
java.awt.Component
  └── java.awt.Container
       ├── Panel
       │    └── Applet
       ├── ScrollPane
       └── Window
            ├── Frame
            └── Dialog
```

### Пример AWT

```java
Frame frame = new Frame("AWT");
Panel panel = new Panel();
panel.setLayout(new FlowLayout());
panel.add(new Button("Btn1"));
panel.add(new Button("Btn2"));
frame.add(panel);
frame.setSize(200, 100);
frame.setVisible(true);
```

## 2. Swing — `JPanel`

`JPanel` — наследник `JComponent` (который наследует `Container`). Основной контейнер Swing.

```java
JPanel panel = new JPanel();

// Layout Manager
panel.setLayout(new BorderLayout());
panel.setLayout(new GridLayout(2, 2));

// Границы (Swing-специфично)
panel.setBorder(BorderFactory.createTitledBorder("Panel Title"));
panel.setBorder(BorderFactory.createEmptyBorder(10, 10, 10, 10));

// Прозрачность
panel.setOpaque(true);
panel.setBackground(Color.WHITE);
```

### Иерархия Swing-контейнеров

```
java.awt.Container
  └── javax.swing.JComponent
       ├── JPanel
       ├── JScrollPane
       ├── JTabbedPane
       ├── JSplitPane
       ├── JToolBar
       └── JInternalFrame
```

### Пример Swing

```java
JFrame frame = new JFrame("Swing");
JPanel panel = new JPanel();
panel.setLayout(new FlowLayout());
panel.add(new JButton("Btn1"));
panel.add(new JButton("Btn2"));
frame.add(panel);
frame.setSize(200, 100);
frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
frame.setVisible(true);
```

## 3. JavaFX — `Parent`, `Region`, `Scene`

### `Parent` — базовый контейнер

`Parent` — абстрактный базовый класс для всех узлов, которые могут иметь детей. Наследует `Node`.

```java
// Parent — абстрактный, нельзя создать напрямую
// Его конкретные подклассы:
// - Group
// - Region
// - Control
```

### `Region` — контейнер с компоновкой

`Region` — наследник `Parent`. Базовый класс для всех контейнеров, которые поддерживают CSS-стилизацию, имеют размеры и могут содержать детей.

```java
// Основные методы Region
Region region = new Pane();

// Размеры
region.setPrefSize(200, 100);
region.setMinSize(100, 50);
region.setMaxSize(300, 200);

// Отступы и бордеры (CSS)
region.setStyle("-fx-padding: 10; -fx-border-color: black;");

// Скругление углов
region.setStyle("-fx-background-radius: 10;");
```

### Конкретные контейнеры JavaFX (наследники Region)

| Класс | Назначение |
|-------|------------|
| `Pane` | Простой контейнер (абсолютное позиционирование через `setLayoutX()`/`setLayoutY()`) |
| `HBox` | Горизонтальный layout |
| `VBox` | Вертикальный layout |
| `BorderPane` | Верх/низ/лево/право/центр |
| `GridPane` | Сетка (rows/columns) |
| `FlowPane` | Потоковая компоновка |
| `StackPane` | Наложение элементов друг на друга |
| `ScrollPane` | Прокручиваемая область |
| `TabPane` | Вкладки |
| `AnchorPane` | Привязка к краям |

```java
// Примеры контейнеров
HBox hbox = new HBox(10);  // spacing = 10
hbox.getChildren().addAll(new Button("A"), new Button("B"));

VBox vbox = new VBox(5);
vbox.getChildren().addAll(new Label("Name:"), new TextField());

BorderPane border = new BorderPane();
border.setTop(new Label("Header"));
border.setCenter(new Button("Center"));
border.setBottom(new Label("Footer"));

GridPane grid = new GridPane();
grid.add(new Label("Name:"), 0, 0);
grid.add(new TextField(), 1, 0);
grid.add(new Label("Age:"), 0, 1);
grid.add(new TextField(), 1, 1);
```

### `Scene` — корень сцены

`Scene` не является `Node` или контейнером. Это **контейнер верхнего уровня**, который содержит корневой узел (который обычно является `Parent`).

```java
// Создание Scene
Parent root = new VBox();
Scene scene = new Scene(root);
Scene scene2 = new Scene(root, 800, 600);  // с явными размерами

// Подключение CSS
scene.getStylesheets().add("style.css");

// Обработка событий клавиатуры/мыши на уровне сцены
scene.setOnKeyPressed(event -> System.out.println("Key pressed: " + event.getCode()));
```

### Полный пример JavaFX

```java
import javafx.application.Application;
import javafx.scene.Scene;
import javafx.scene.control.*;
import javafx.scene.layout.*;
import javafx.stage.Stage;

public class JavaFxContainers extends Application {
    @Override
    public void start(Stage stage) {
        // Контейнеры
        VBox vbox = new VBox(10);
        vbox.setStyle("-fx-padding: 20; -fx-alignment: center;");
        
        Label label = new Label("User Form");
        TextField nameField = new TextField();
        nameField.setPromptText("Enter name");
        Button button = new Button("Submit");
        
        vbox.getChildren().addAll(label, nameField, button);
        
        // Scene содержит корневой контейнер
        Scene scene = new Scene(vbox, 300, 200);
        scene.getStylesheets().add("style.css");  // опционально
        
        stage.setTitle("JavaFX Containers");
        stage.setScene(scene);
        stage.show();
    }
    
    public static void main(String[] args) {
        launch(args);
    }
}
```

| Характеристика | AWT `Container` | Swing `JPanel` | JavaFX `Parent`/`Region` |

| Границы | Нет | `setBorder(Border)` | CSS (`-fx-border-color`) |
| CSS-стилизация | Нет | PLAF (ограниченно) | ✅ Полная поддержка CSS |

## 🔧 Подкапотные детали

### AWT/Swing: иерархия контейнеров

Каждый `Container` хранит список дочерних компонентов в `java.awt.Container$Component[]` (массив). При добавлении компонента массив расширяется. Layout Manager вычисляет позиции и размеры детей при каждом `validate()`.

```java
// Внутреннее устройство Container (упрощённо)
public class Container extends Component {
    private Component[] component = new Component[0];  // массив детей
    private LayoutManager layoutMgr;
    
    public Component add(Component comp) {
        // расширение массива
        component = Arrays.copyOf(component, component.length + 1);
        component[component.length - 1] = comp;
        comp.parent = this;
        return comp;
    }
}
```

### JavaFX: граф сцены

`Parent` хранит детей в `ObservableList<Node>`. `Scene` содержит корневой `Parent` и управляет рендерингом через `Stage`.

```java
// Внутреннее устройство Parent (упрощённо)
public abstract class Parent extends Node {
    private ObservableList<Node> children;
    
    public ObservableList<Node> getChildren() {
        return children;
    }
}

// Scene содержит корневой узел
public class Scene {
    private Parent root;
    
    public void setRoot(Parent root) {
        this.root = root;
        // обновление рендеринга
    }
}
```

### Принцип работы Scene

`Scene` связана с `Stage` (окном). При изменении свойств узлов (позиция, размер, текст) JavaFX запускает **Pulse** — синхронизацию с рендерингом (60 раз в секунду). Перерисовывается только изменённая область.

```java
// Привязка Stage → Scene → Root
Stage stage = new Stage();
Scene scene = new Scene(root);
stage.setScene(scene);
stage.show();
```

## ⚠️ Частые ошибки

### ❌ Неправильно

```java
// 1. AWT/Swing: добавление детей после show()
frame.setVisible(true);
frame.add(new Button("Late"));  // ❌ Не отобразится

// 2. Swing: неправильный Layout Manager
panel.setLayout(null);
button.setBounds(10, 10, 100, 30);  // ❌ Абсолютное позиционирование — антипаттерн

// 3. JavaFX: добавление одного узла в несколько контейнеров
Button btn = new Button("Shared");
hbox.getChildren().add(btn);
vbox.getChildren().add(btn);  // ❌ IllegalStateException (узел уже имеет родителя)

// 4. JavaFX: прямое изменение children не в JavaFX потоке
new Thread(() -> {
    vbox.getChildren().add(new Button());  // ❌ Не из JavaFX Application Thread
}).start();
```

### ✅ Правильно

```java
// 1. Сначала add, потом show
frame.add(button);
frame.setVisible(true);

// 2. Использовать Layout Manager
panel.setLayout(new FlowLayout());
panel.add(button);

// 3. Создавать отдельные узлы
hbox.getChildren().add(new Button("A"));
vbox.getChildren().add(new Button("B"));

// 4. Platform.runLater
new Thread(() -> {
    Platform.runLater(() -> vbox.getChildren().add(new Button()));
}).start();
```

## 🔗 Связи с другими темами

| Тема | Связь |
|------|-------|
| **Layout Managers** | AWT/Swing: `FlowLayout`, `BorderLayout`, `GridBagLayout` |
| **JavaFX Layouts** | `HBox`, `VBox`, `GridPane`, `BorderPane`, `StackPane` |
| **Scene Graph** | Древовидная структура узлов JavaFX |
| **Event Handling** | События всплывают от узла к сцене (event bubbling) |
| **CSS in JavaFX** | `Scene.getStylesheets().add()` |
| **Threading** | EDT (Swing) / JavaFX Application Thread |

---

### Шпаргалка

* **`Container` (AWT)** — базовый контейнер, `add()`, `setLayout()`, `Panel`, `Frame`
* **`JPanel` (Swing)** — основной контейнер, наследник `JComponent`, используется в 99% случаев
* **`Parent` (JavaFX)** — базовый класс для всех узлов с детьми (абстрактный)
* **`Region` (JavaFX)** — наследник `Parent`, основа для `Pane`, `HBox`, `VBox`, `GridPane` и др.
* **`Scene` (JavaFX)** — не контейнер, а корень сцены, содержит граф узлов
* **Главное отличие:** в AWT/Swing дети добавляются через `add()`, в JavaFX — через `getChildren().add()`
* **JavaFX контейнеры имеют встроенную компоновку** (не нужен отдельный Layout Manager)
* **Один узел в JavaFX не может быть в двух контейнерах** (в отличие от AWT/Swing)
* **Все изменения JavaFX-интерфейса** — только в `Platform.runLater()` или внутри `start()`
