# 19. Взаимодействие с базами данных. JDBC. Основные элементы

## 💡 Общая идея JDBC

JDBC (Java Database Connectivity) — это стандартный API Java для работы с реляционными базами данных.

Он позволяет:
- подключаться к базе данных
- выполнять SQL-запросы
- получать результаты
- управлять транзакциями

Архитектура взаимодействия:

```
Java приложение → JDBC API → JDBC Driver → СУБД
```

---

## 🔵 Основные элементы JDBC

### Driver
Драйвер — это мост между Java и конкретной СУБД (MySQL, PostgreSQL и т.д.).
Он переводит JDBC-вызовы в команды базы данных.

---

### DriverManager
Управляет драйверами и создаёт соединение с базой данных.

---

### Connection
Соединение с базой данных.

```java
Connection conn = DriverManager.getConnection(url, user, password);
```

---

### Statement
Используется для выполнения SQL-запросов.

```java
Statement st = conn.createStatement();
ResultSet rs = st.executeQuery("SELECT * FROM users");
```

❌ Минус: уязвим к SQL-injection

---

### PreparedStatement
Параметризованный и безопасный SQL-запрос.

```java
PreparedStatement ps =
    conn.prepareStatement("SELECT * FROM users WHERE id = ?");

ps.setInt(1, 10);

ResultSet rs = ps.executeQuery();
```

✔ Защита от SQL-injection  
✔ Лучше производительность при повторных запросах  

---

### CallableStatement
Используется для вызова хранимых процедур.

```java
CallableStatement cs =
    conn.prepareCall("{call get_users()}");
```

---

### ResultSet
Результат SQL-запроса.

```java
while (rs.next()) {
    String name = rs.getString("name");
    int id = rs.getInt("id");
}
```

---

## 🔗 Общая цепочка работы JDBC

```
DriverManager → Connection → Statement / PreparedStatement → ResultSet
```

---

## ⚖ Statement vs PreparedStatement

| Критерий | Statement | PreparedStatement |
|----------|----------|------------------|
| Безопасность | низкая | высокая |
| SQL injection | уязвим | защищён |
| Производительность | ниже | выше |
| Параметры | нет | есть |

---

## ⚙ Важные моменты

### 1. Закрытие ресурсов
Всегда закрывать соединения:

```java
try (Connection conn = DriverManager.getConnection(...)) {
    // работа
}
```

---

### 2. SQL Injection

❌ плохо:
```java
"SELECT * FROM users WHERE id = " + id;
```

✔ правильно:
```java
PreparedStatement
```

---

### 3. Транзакции

```java
conn.setAutoCommit(false);

conn.commit();
conn.rollback();
```

---

## 🧠 Шпаргалка

- JDBC — стандарт работы с БД
- DriverManager создаёт Connection
- Connection — соединение с БД
- Statement — простой SQL
- PreparedStatement — безопасный SQL
- CallableStatement — хранимые процедуры
- ResultSet — результат запроса
