# 6. Сортировка элементов коллекций. `Comparable` и `Comparator`

## Общая идея сортировки

Сортировка в Java — это процесс упорядочивания элементов по некоторому правилу сравнения.

Java не “знает”, как сравнивать объекты автоматически → поэтому используется:

- `Comparable` → встроенное (естественное) сравнение
- `Comparator` → внешнее сравнение (гибкое)

---

# 🔵 Comparable (естественный порядок)

## Что это

`Comparable<T>` задает **естественный порядок объекта внутри самого класса**.

```java
public interface Comparable<T> {
    int compareTo(T o);
}
```

---

## Пример

```java
class User implements Comparable<User> {

    private int age;

    public User(int age) {
        this.age = age;
    }

    public int getAge() {
        return age;
    }

    @Override
    public int compareTo(User other) {
        return Integer.compare(this.age, other.age);
    }
}
```

---

## Как работает compareTo

```text
< 0  → this < other
0    → равны по порядку
> 0  → this > other
```

---

## Использование

```java
List<User> users = new ArrayList<>();
Collections.sort(users);
```

или

```java
users.sort(null);
```

---

## Внутреннее устройство сортировки

В Java используется:

- TimSort (гибрид merge sort + insertion sort)
- стабильная сортировка

Сложность:

```text
O(n log n)
```

---

## Важное правило (экзамен!)

👉 `compareTo` должен быть согласован с `equals`

Иначе:
- ломается `TreeSet`
- ломается `TreeMap`

---

## Ошибка нарушения контракта

```java
a.compareTo(b) == 0
```

но

```java
!a.equals(b)
```

→ приводит к некорректному поведению коллекций

---

# 🟠 Comparator (внешний порядок)

## Что это

`Comparator<T>` — отдельный объект, задающий правило сравнения **снаружи класса**.

```java
public interface Comparator<T> {
    int compare(T o1, T o2);
}
```

---

## Пример

```java
Comparator<User> byAge =
        (u1, u2) -> Integer.compare(u1.getAge(), u2.getAge());
```

---

## Использование

```java
users.sort(byAge);
```

или

```java
Collections.sort(users, byAge);
```

---

## Почему Comparator важен

Позволяет:

✔ несколько вариантов сортировки  
✔ не менять сам класс  
✔ сортировать сторонние классы  

---

## Несколько критериев сортировки

```java
Comparator<User> cmp =
        Comparator.comparingInt(User::getAge)
                  .thenComparing(User::getName);
```

---

## Обратный порядок

```java
Comparator<User> desc =
        Comparator.comparingInt(User::getAge)
                  .reversed();
```

---

## Comparator vs Comparable (глубокое различие)

| Критерий | Comparable | Comparator |
|----------|-----------|------------|
| Где задается | внутри класса | снаружи |
| Кол-во правил | одно | много |
| Изменение класса | требуется | не требуется |
| Гибкость | низкая | высокая |

# ⚙ Внутренний механизм сортировки (важно для 5+)

## Что происходит при sort()

### 1. преобразование в массив (внутри List)

### 2. применение TimSort

TimSort:
- оптимизирован под частично отсортированные данные
- использует insertion sort на малых сегментах
- merge sort для объединения

---

## Почему это важно

👉 объясняет, почему `List.sort()` быстрее в реальных данных, чем “чистый merge sort”

# ⚠ Частые ошибки студентов

### ❌ 1. нарушение equals/compareTo

```java
compareTo == 0, но equals != true
```

→ ломает TreeSet

---

### ❌ 2. непоследовательный Comparator

```java
a > b, b > c, но c > a
```

→ нарушение транзитивности

---

### ❌ 3. использование subtraction

```java
return a.age - b.age; // ❌ опасно
```

→ overflow bug

✔ правильно:

```java
Integer.compare(a.age, b.age);
```

---

# 💥 Типовой вопрос преподавателя

❓ Почему нельзя делать сортировку через `a - b`?

✔ потому что возможен integer overflow → нарушение порядка

---

# 📌 Когда что использовать

## Comparable:
- “естественный порядок”
- один главный способ сортировки

## Comparator:
- разные способы сортировки
- внешняя логика
- сортировка библиотечных классов

---

# 🧠 Шпаргалка

* `Comparable` → естественный порядок внутри класса
* `Comparator` → внешний способ сортировки
* `compareTo()` → внутри класса
* `compare()` → снаружи
* сортировка → TimSort O(n log n)
* `Comparator` позволяет цепочки (`thenComparing`)
* нельзя использовать `a - b` (overflow)
* `compareTo` должен быть согласован с `equals`
