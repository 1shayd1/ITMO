# 10. Интерфейсы `Queue` и `Deque`. Классы `PriorityQueue` и `ArrayDeque`

## Общая идея

### `Queue`
`Queue` — это структура данных **очередь**, чаще всего реализующая принцип:

```text
FIFO (First In First Out)
```

👉 первый пришёл — первый вышел

---

### `Deque`
`Deque` (Double Ended Queue) — это расширение очереди:

👉 можно работать с обоих концов

```text
head ↔ tail
```

---

# 🔵 Интерфейс Queue

## Основные методы

### Добавление:

```java
add(e)   // выбрасывает исключение при ошибке
offer(e) // безопасный вариант (возвращает false)
```

---

### Удаление:

```java
remove() // исключение если пусто
poll()   // null если пусто
```

---

### Просмотр:

```java
element() // исключение если пусто
peek()    // null если пусто
```

---

## Разница add/remove vs offer/poll

| Метод | Поведение при ошибке |
|------|----------------------|
| add | исключение |
| remove | исключение |
| offer | false |
| poll | null |

👉 На практике чаще используют `offer/poll`

---

## Внутренний смысл Queue

Queue — это **абстракция очереди**, а реализация может быть разной:

- массив
- список
- куча (heap)

---

# 🟡 PriorityQueue

## Что это

`PriorityQueue` — очередь, где порядок определяется **приоритетом**, а не FIFO.

---

## Внутреннее устройство

👉 основана на **binary heap (куча)**

```text
parent ≤ children (min-heap по умолчанию)
```

---

## Как работает

При добавлении:

1. элемент вставляется в конец массива
2. выполняется heapify-up

При удалении:

1. берётся корень (минимальный элемент)
2. последний элемент поднимается
3. heapify-down

---

## Сложность

| операция | сложность |
|----------|----------|
| add | O(log n) |
| poll | O(log n) |
| peek | O(1) |

---

## Пример

```java
Queue<Integer> q = new PriorityQueue<>();

q.offer(3);
q.offer(1);
q.offer(2);

System.out.println(q.poll()); // 1
```

---

## Особенности PriorityQueue

- всегда извлекается минимальный элемент (по умолчанию)
- можно задать Comparator
- не гарантирует полный порядок внутри структуры
- null запрещён

---

## Max-heap (через Comparator)

```java
PriorityQueue<Integer> q =
    new PriorityQueue<>(Comparator.reverseOrder());
```

---

# 🟢 Deque

## Что это

`Deque` = двусторонняя очередь:

```text
add/remove с начала и конца
```

---

## Основные методы

### С обоих концов:

```java
addFirst()
addLast()

removeFirst()
removeLast()

peekFirst()
peekLast()
```

---

## Как стек

```java
push() → addFirst
pop()  → removeFirst
```

---

## Как очередь

```java
offerLast()
pollFirst()
```

---

## Почему Deque важен

👉 заменяет устаревший `Stack`

---

# 🟠 ArrayDeque

## Что это

`ArrayDeque` — реализация `Deque` на основе **циклического массива**

---

## Внутреннее устройство

```text
Object[] array
head index
tail index
```

Массив работает как:

```text
кольцо (circular buffer)
```

---

## Почему он быстрый

- нет объектов как в LinkedList
- нет переходов по ссылкам
- cache-friendly

---

## Сложность

| операция | сложность |
|----------|----------|
| addFirst | O(1) |
| addLast | O(1) |
| removeFirst | O(1) |
| removeLast | O(1) |

---

## Особенности

- не допускает null
- быстрее LinkedList для stack/queue задач
- предпочтительная реализация Deque

---

## Пример

```java
Deque<Integer> dq = new ArrayDeque<>();

dq.addFirst(1);
dq.addLast(2);

System.out.println(dq.removeFirst()); // 1
```

---

# ⚖ PriorityQueue vs ArrayDeque

| характеристика | PriorityQueue | ArrayDeque |
|---------------|--------------|-------------|
| принцип | приоритет | FIFO / LIFO |
| структура | heap | circular array |
| порядок | сортировка частичная | порядок вставки |
| null | нельзя | нельзя |
| сложность | O(log n) | O(1) |

---

# 💥 Типовые вопросы преподавателя

## ❓ Почему PriorityQueue не отсортирован полностью?

✔ потому что heap хранит только частичный порядок:
- гарантирует минимум/максимум
- но не полную сортировку

---

## ❓ Почему ArrayDeque быстрее LinkedList?

✔ потому что:
- массив (cache-friendly)
- нет объектов Node
- нет указателей next/prev

---

## ❓ Когда использовать PriorityQueue?

✔ когда нужен:
- scheduler
- задачи с приоритетами
- алгоритмы (Dijkstra, A*)

---

# 🧠 Шпаргалка

* `Queue` = FIFO очередь
* `Deque` = двусторонняя очередь
* `PriorityQueue` = heap, приоритет, O(log n)
* `ArrayDeque` = массив-кольцо, быстрый stack/queue
* `offer/poll/peek` — безопасные операции
* `PriorityQueue` не сортирует полностью, только поддерживает heap
