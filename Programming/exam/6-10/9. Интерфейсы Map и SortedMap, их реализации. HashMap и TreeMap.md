# 9. Интерфейсы `Map` и `SortedMap`, их реализации. `HashMap` и `TreeMap`

## Что такое `Map`

`Map<K, V>` — структура данных, которая хранит пары:

```text
key → value
```

### Особенности:

- ключи уникальны
- значения могут повторяться
- не наследуется от `Collection`

---

## Основные операции Map

```java
put(K key, V value)
get(Object key)
remove(Object key)
containsKey(Object key)
containsValue(Object value)
keySet()
values()
entrySet()
```

---

## Внутренняя иерархия

```text
Map
 ├── HashMap
 ├── LinkedHashMap
 ├── TreeMap (SortedMap)
```

---

# 🔵 SortedMap

## Что это

`SortedMap` — расширение `Map`, где:

👉 ключи автоматически отсортированы

---

## Основные возможности

- упорядоченные ключи
- доступ к диапазонам
- min / max ключи

---

## Реализация

👉 `TreeMap`

---

# 🟡 HashMap

## Что это

`HashMap` — реализация `Map` на основе хеш-таблицы.

---

## Внутреннее устройство (ВАЖНО)

### 1. Структура

```text
array of buckets (Node<K,V>[])
```

---

### 2. put()

1. вычисляется `hashCode(key)`
2. определяется индекс bucket
3. если коллизия:
   - equals() проверка
   - цепочка или дерево (Java 8+)

---

## Сложность

| операция | сложность |
|----------|----------|
| get | O(1) avg |
| put | O(1) avg |
| remove | O(1) avg |

---

## Важно (Java 8+)

Коллизии:

```text
linked list → red-black tree (если много элементов)
```

---

## Особенности HashMap

- порядок НЕ гарантируется
- допускает:
  - 1 null key
  - много null values
- быстрый доступ

---

## Почему HashMap быстрый

Потому что доступ идет через:

```text
hash → index
```

---

## ВАЖНО: проблема ключей

Если ключ изменился после вставки:

```java
map.put(key, value);
key изменили → map.get(key) ❌ null
```

---

# 🟢 TreeMap

## Что это

`TreeMap` — реализация `SortedMap`, основанная на:

👉 красно-чёрном дереве

---

## Внутренний принцип

```text
compare(key) → left/right subtree
```

---

## Сложность

| операция | сложность |
|----------|----------|
| get | O(log n) |
| put | O(log n) |
| remove | O(log n) |

---

## Сортировка ключей

Используется:

- `Comparable`
или
- `Comparator`

---

## Пример

```java
Map<Integer, String> map = new TreeMap<>();

map.put(3, "C");
map.put(1, "A");
map.put(2, "B");
```

👉 результат:

```
1 → A
2 → B
3 → C
```

---

## Особенности TreeMap

- всегда отсортирован
- поддерживает диапазоны
- не допускает null ключ (в общем случае)

---

# ⚖ HashMap vs TreeMap

| характеристика | HashMap | TreeMap |
|----------------|--------|--------|
| структура | hash table | red-black tree |
| порядок | нет | сортировка |
| скорость | O(1) avg | O(log n) |
| доступ | по ключу | по ключу |
| null key | да | обычно нет |

---

# ⚙ SortedMap (важный момент)

`SortedMap` добавляет:

```text
firstKey()
lastKey()
subMap()
headMap()
tailMap()
```

---

# 💥 Внутренние детали (уровень 5+)

## HashMap

- capacity (обычно 16)
- load factor = 0.75
- resize → ×2
- treeify при больших коллизиях

---

## TreeMap

- красно-чёрное дерево
- self-balancing
- гарантирует O(log n)

---

# 💥 Типовые вопросы преподавателя

## ❓ Почему HashMap быстрее TreeMap?

✔ потому что:
- HashMap → прямой доступ через hash
- TreeMap → обход дерева

---

## ❓ Что будет, если изменить ключ в HashMap?

✔ потеряется доступ к элементу (broken mapping)

---

## ❓ Почему Map не Collection?

✔ потому что хранит пары (key-value), а не одиночные элементы

---

# 🧠 Шпаргалка

* `Map` = key → value
* `HashMap` = hash table, O(1), без порядка
* `TreeMap` = red-black tree, O(log n), сортировка
* `SortedMap` = упорядоченные ключи
* HashMap = hashCode + equals
* TreeMap = compareTo / Comparator
* изменение ключа ломает HashMap
