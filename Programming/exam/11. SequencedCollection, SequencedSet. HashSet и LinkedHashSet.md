# 11. `SequencedCollection`, `SequencedSet`. `HashSet` и `LinkedHashSet`

## 💡 Общая идея (очень важно)

Начиная с Java 21 появились **sequenced collections** — это попытка стандартизировать:

> работу с “первым”, “последним” и порядком элементов в коллекциях

---

# 🔵 SequencedCollection

## Что это

`SequencedCollection` — это интерфейс, который добавляет **гарантированный порядок элементов** и операции с краями коллекции.

---

## Основные операции

```java
T getFirst()
T getLast()

void addFirst(T)
void addLast(T)

SequencedCollection<T> reversed()
```

---

## Что это значит концептуально

Теперь коллекции явно поддерживают:

```text
начало ↔ конец
```

---

## Зачем это ввели

Раньше:
- у List было
- у Deque было
- у Set — не было унифицировано

👉 SequencedCollection делает единый контракт

---

# 🟡 SequencedSet

## Что это

`SequencedSet` =

```text
Set + порядок + операции начала/конца
```

---

## Особенности

- элементы уникальны (как в Set)
- порядок гарантирован
- можно получить:
  - первый элемент
  - последний элемент
  - обратный порядок

---

## Пример идеи

```java
SequencedSet<Integer> set = ...
set.getFirst();
set.getLast();
```

---

## Важно (экзамен)

👉 SequencedSet НЕ меняет модель Set:
- уникальность сохраняется
- отличие только в гарантии порядка

---

# 🟠 HashSet

## Что это

`HashSet` — это множество на основе **хеш-таблицы**.

---

## Внутреннее устройство (ключевое)

Фактически:

```text
HashSet = HashMap
```

```text
value = dummy object (PRESENT)
```

---

## Как работает

1. hashCode()
2. bucket index
3. equals() при коллизиях

---

## Сложность

| операция | средняя |
|----------|--------|
| add | O(1) |
| remove | O(1) |
| contains | O(1) |

---

## Особенности

- порядок НЕ гарантируется
- допускает 1 null
- быстрый доступ

---

## Почему порядок не гарантируется

Потому что:

```text
hashCode → bucket
```

порядок зависит от хеш-функции, а не от вставки

---

# 🟢 LinkedHashSet

## Что это

`LinkedHashSet` =

```text
HashSet + связный список порядка вставки
```

---

## Внутреннее устройство

```text
HashMap + doubly linked list
```

---

## Как работает

- хранит элементы в hash table
- дополнительно хранит:
  ```text
  next/prev ссылки
  ```

---

## Что это даёт

✔ порядок вставки сохраняется  
✔ уникальность сохраняется  
✔ быстрый доступ сохраняется  

---

## Сложность

| операция | сложность |
|----------|--------|
| add | O(1) |
| remove | O(1) |
| contains | O(1) |

---

## Отличие от HashSet

| характеристика | HashSet | LinkedHashSet |
|----------------|--------|----------------|
| порядок | нет | insertion order |
| структура | hash table | hash + linked list |
| память | меньше | больше |
| скорость | чуть быстрее | чуть медленнее |

---

# ⚖ HashSet vs LinkedHashSet (суть)

## HashSet:
```text
максимальная скорость, но хаотичный порядок
```

## LinkedHashSet:
```text
чуть медленнее, но предсказуемый порядок
```

---

# 🔥 Важный экзаменационный момент

## LinkedHashSet используется когда:

- нужен Set
- но важен порядок добавления

---

# ⚙ Внутренний механизм Set (очень важно)

## HashSet / LinkedHashSet:

```text
hashCode → bucket → equals
```

## LinkedHashSet дополнительно:

```text
добавление в linked list цепочку
```

---

# 💥 Типовые вопросы преподавателя

## ❓ Почему HashSet быстрее LinkedHashSet?

✔ потому что:
- нет поддержки связного списка
- меньше операций при вставке

---

## ❓ Чем LinkedHashSet отличается от TreeSet?

✔ LinkedHashSet:
- порядок вставки

✔ TreeSet:
- сортировка (natural order)

---

## ❓ Зачем нужен SequencedSet?

✔ чтобы унифицировать:
- доступ к первому/последнему
- работу с порядком
- без привязки к конкретной реализации

---

# 🧠 Шпаргалка

* `SequencedCollection` = коллекции с порядком + first/last
* `SequencedSet` = Set с порядком
* `HashSet` = hash table, быстрый, без порядка
* `LinkedHashSet` = hash + linked list, порядок вставки
* HashSet быстрее, LinkedHashSet предсказуемее
* Set всегда гарантирует уникальность
