# 2. Работа с параметризованными методами. Ограничение типа сверху или снизу

## Параметризованные методы (Generic Methods)

**Параметризованный метод** — это метод, который объявляет собственные параметры типа независимо от параметров типа класса.

Он используется, когда:

* метод должен быть универсальным;
* тип данных не связан с классом;
* требуется максимальная повторная используемость.

---

## Объявление параметризованного метода

Синтаксис:

```java
<Т> returnType methodName(...)
```

Пример:

```java
class Utils {

    public static <T> void print(T value) {
        System.out.println(value);
    }
}
```

Здесь:

```java
<T>
```

— параметр типа метода, а не класса.

---

## Вызов параметризованного метода

```java
Utils.print("Hello");
Utils.print(100);
Utils.print(3.14);
```

Компилятор автоматически выводит тип (`type inference`).

Эквивалентно:

```java
Utils.<String>print("Hello");
```

---

## Метод с несколькими параметрами типа

```java
class PairUtils {

    public static <K, V> void show(K key, V value) {
        System.out.println(key + " = " + value);
    }
}
```

Использование:

```java
PairUtils.show("Age", 20);
PairUtils.show(1, "One");
```

---

## Ограничение типа сверху (Upper Bound)

Используется для ограничения допустимых типов.

### Синтаксис:

```java
<T extends Type>
```

---

### Пример

```java
public static <T extends Number> double sum(T a, T b) {
    return a.doubleValue() + b.doubleValue();
}
```

---

### Что означает

```text
T может быть Number или его наследником
```

Допустимо:

```java
sum(10, 20);
sum(5.5, 2.3);
```

Недопустимо:

```java
sum("A", "B"); // ошибка
```

---

## Несколько ограничений типов

Можно задавать несколько ограничений:

```java
<T extends Number & Comparable<T>>
```

---

### Правила:

1. Сначала указывается класс (если есть)
2. Затем интерфейсы

Пример:

```java
class Box<T extends Number & Comparable<T>> {
}
```

---

## Ограничение типа в wildcard (важно различать)

Ограничения бывают в двух местах:

### 1. В параметре типа метода

```java
<T extends Number>
```

### 2. В wildcard

```java
List<? extends Number>
```

Это разные механизмы.

---

## Upper Bound wildcard (`? extends`)

Используется, когда:

> мы читаем данные (producer)

### Пример:

```java
public static double total(List<? extends Number> list) {
    double sum = 0;

    for (Number n : list) {
        sum += n.doubleValue();
    }

    return sum;
}
```

---

### Особенности `? extends`

✔ Можно читать как `Number`  
❌ Нельзя добавлять элементы (кроме `null`)

```java
list.add(10); // ошибка
```

---

## Lower Bound wildcard (`? super`)

Используется, когда:

> мы записываем данные (consumer)

### Пример:

```java
public static void addNumbers(List<? super Integer> list) {
    list.add(10);
    list.add(20);
}
```

---

### Что это означает:

Подходит:

```java
List<Integer>
List<Number>
List<Object>
```

---

### Особенности `? super`

✔ Можно добавлять `Integer`  
⚠ При чтении тип — `Object`

```java
Object obj = list.get(0);
```

---

## Сравнение `extends` и `super`

| Конструкция | Назначение | Чтение | Запись |
|------------|------------|--------|--------|
| `? extends T` | producer | безопасно как `T` | запрещено |
| `? super T` | consumer | только `Object` | разрешено |

---

## Практический пример (extends)

```java
public static double sum(List<? extends Number> list) {
    double result = 0;

    for (Number n : list) {
        result += n.doubleValue();
    }

    return result;
}
```

Можно передать:

```java
List<Integer>
List<Double>
List<Float>
```

---

## Практический пример (super)

```java
public static void fill(List<? super Integer> list) {
    list.add(1);
    list.add(2);
}
```

---

## PECS правило

Очень важное для экзамена:

```text
PECS = Producer Extends, Consumer Super
```

---

### Объяснение:

* Producer (производит данные) → `extends`
* Consumer (потребляет данные) → `super`

---

## Частая ошибка

```java
<T super Number> // ❌ нельзя
```

`super` нельзя использовать с параметром типа `<T>` — только в wildcard `?`.

---

## Итоговая логика использования

| Ситуация | Что использовать |
|----------|----------------|
| Универсальный метод | `<T>` |
| Только числа и наследники | `<T extends Number>` |
| Чтение из коллекции | `? extends T` |
| Запись в коллекцию | `? super T` |

---

### Шпаргалка

* Параметризованный метод объявляется как `<T>`.
* `<T extends X>` — ограничение сверху для параметра типа.
* `? extends X` — верхняя граница wildcard (для чтения).
* `? super X` — нижняя граница wildcard (для записи).
* PECS: Producer Extends, Consumer Super.
* Несколько ограничений: `T extends Class & Interface1 & Interface2`.
