# 1. Обобщенные и параметризованные типы. Создание параметризованных классов

## Суть Generics

**Generics** — механизм параметризации типов, позволяющий писать типобезопасный и переиспользуемый код.

Основная цель:
* перенос ошибок типов на этап компиляции;
* устранение ручного кастинга;
* повышение читаемости.

---

## Проблема до Generics

```java
List list = new ArrayList();

list.add("Java");
list.add(10);

String s = (String) list.get(1); // ClassCastException
```

---

## Решение с Generics

```java
List<String> list = new ArrayList<>();

list.add("Java");
// list.add(10); // ошибка компиляции

String s = list.get(0);
```

---

## Type Parameter и Parameterized Type

| Термин | Пример |
|--------|--------|
| Type Parameter | `T` в `class Box<T>` |
| Type Argument | `String` в `Box<String>` |
| Parameterized Type | `Box<String>` |

---

## Создание обобщенного класса

```java
class Box<T> {

    private T value;

    public void set(T value) {
        this.value = value;
    }

    public T get() {
        return value;
    }
}
```

Использование:

```java
Box<String> box = new Box<>();
box.set("Hello");

String s = box.get();
```

---

## Несколько параметров типа

```java
class Pair<K, V> {

    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() { return key; }
    public V getValue() { return value; }
}
```

---

## Diamond operator

```java
Box<String> box = new Box<>();
```

---

## Ограничения (Type Erasure)

* Нельзя `new T()`
* Нельзя `new T[]`
* Нельзя `instanceof Box<String>`
* Нельзя использовать примитивы (`Box<int>` ❌)

---

## Raw Type (опасно)

```java
Box box = new Box(); // без типа
```

---

## Шпаргалка

* Generics = параметризация типов
* Проверка на этапе компиляции
* `Box<T>` — generic class
* `Box<String>` — parameterized type
* Реализация через type erasure
* Нельзя использовать примитивы


---

# 2. Работа с параметризованными методами. Ограничения типов

## Generic методы

```java
class Utils {

    public static <T> void print(T value) {
        System.out.println(value);
    }
}
```

---

## Метод с возвращаемым типом

```java
public static <T> T first(T[] array) {
    return array[0];
}
```

---

## Upper Bound (extends)

```java
public static <T extends Number> double sum(T a, T b) {
    return a.doubleValue() + b.doubleValue();
}
```

✔ допускает `Number` и наследников

---

## Wildcards

### Верхняя граница

```java
List<? extends Number>
```

✔ можно читать  
❌ нельзя добавлять

---

### Нижняя граница

```java
List<? super Integer>
```

✔ можно добавлять  
❌ ограниченное чтение

---

## PECS правило

* Producer → `extends`
* Consumer → `super`

---

## Шпаргалка

* `<T>` — generic method
* `extends` — верхняя граница
* `super` — нижняя граница
* `?` — wildcard
* PECS правило

# 3. Number и Wrapper классы. Autoboxing

## Класс Number

Абстрактный базовый класс:

```text
Number
├─ Integer
├─ Double
├─ Long
├─ Float
├─ Short
├─ Byte
```

Методы:

```java
intValue()
doubleValue()
longValue()
```

---

## Wrapper классы

| primitive | wrapper |
|----------|--------|
| int | Integer |
| double | Double |
| char | Character |
| boolean | Boolean |

---

## Зачем нужны

* коллекции работают только с объектами
* поддержка null
* утилитные методы

---

## Autoboxing

```java
Integer x = 10; // int -> Integer
```

---

## Unboxing

```java
int y = x; // Integer -> int
```

---

## Проблемы

### NullPointerException

```java
Integer x = null;
int y = x; // NPE
```

---

### Кэширование Integer

```java
Integer a = 100;
Integer b = 100;
a == b // true

Integer c = 200;
Integer d = 200;
c == d // false
```

---

## Шпаргалка

* Number — базовый класс чисел
* Wrapper — объектная форма примитивов
* Autoboxing / Unboxing автоматические
* Integer cache (-128..127)
* Нельзя использовать примитивы в generics

# 4. Коллекции Java

## Иерархия

```text
Collection
├─ List
├─ Set
└─ Queue
    └─ Deque

Map (отдельно)
```

---

## List

* порядок сохраняется
* дубликаты разрешены

```java
List<String> list = new ArrayList<>();
```

---

## Set

* уникальные элементы
* без дубликатов

```java
Set<String> set = new HashSet<>();
```

---

## Queue

FIFO очередь

```java
offer()
poll()
peek()
```

---

## Deque

Двусторонняя очередь

```java
addFirst()
addLast()
removeFirst()
```

---

## Map

ключ → значение

```java
Map<String, Integer> map = new HashMap<>();
```

---

## Сложность (Hash структуры)

| структура | доступ |
|----------|--------|
| HashMap | O(1) |
| HashSet | O(1) |
| TreeMap | O(log n) |

---

## Шпаргалка

* List — порядок + дубликаты
* Set — уникальность
* Queue — FIFO
* Deque — двусторонняя очередь
* Map — ключ-значение

# 5. Iterable, Iterator, ListIterator

## Iterable

Основа for-each

```java
for (String s : list)
```

---

## Iterator

```java
Iterator<String> it = list.iterator();

while (it.hasNext()) {
    String s = it.next();
}
```

---

## Удаление через Iterator

❌ неправильно:

```java
for (String s : list) {
    list.remove(s);
}
```

✔ правильно:

```java
Iterator<String> it = list.iterator();

while (it.hasNext()) {
    if (it.next().equals("A")) {
        it.remove();
    }
}
```

---

## ListIterator

Дополнительно:

* движение назад
* изменение элементов
* индексы

```java
ListIterator<String> it = list.listIterator();
```

---

## Возможности

```text
next()
previous()
add()
set()
```

---

## Fail-fast

Если коллекция изменяется вне итератора:

```text
ConcurrentModificationException
```

---

## Шпаргалка

* Iterable → for-each
* Iterator → обход
* ListIterator → расширенный обход
* remove через iterator безопасен
* fail-fast защищает коллекции
