# 3. Класс Number. Классы-обертки (Wrappers). Автоупаковка и автораспаковка

## Класс `Number`

`Number` — это абстрактный базовый класс, который объединяет все числовые типы-обертки в Java.

Он задает общий интерфейс для преобразования числа в разные примитивные типы.

```java
public abstract class Number {

    public abstract int intValue();

    public abstract long longValue();

    public abstract float floatValue();

    public abstract double doubleValue();
}
```

---

## Иерархия `Number`

Основные наследники:

```text
Number
├── Byte
├── Short
├── Integer
├── Long
├── Float
└── Double
```

Важно:

```text
BigInteger и BigDecimal НЕ наследуются от Number напрямую в классическом смысле и реализуют отдельную модель работы с числами.
```

---

## Роль `Number`

Класс `Number` нужен для:

* унификации числовых типов;
* полиморфной работы с числами;
* возможности хранить разные числа в одной переменной.

Пример:

```java
Number n = Integer.valueOf(10);

System.out.println(n.doubleValue());
System.out.println(n.intValue());
```

---

## Классы-обертки (Wrapper Classes)

Каждому примитивному типу соответствует объектный аналог.

| Примитив | Обертка |
|----------|---------|
| `byte` | `Byte` |
| `short` | `Short` |
| `int` | `Integer` |
| `long` | `Long` |
| `float` | `Float` |
| `double` | `Double` |
| `char` | `Character` |
| `boolean` | `Boolean` |

---

## Зачем нужны обертки

### 1. Работа с коллекциями

Коллекции хранят только объекты:

```java
List<int> list; // ❌ нельзя
List<Integer> list; // ✅ правильно
```

---

### 2. Возможность использовать `null`

```java
Integer x = null; // допустимо
int y = null;     // нельзя
```

---

### 3. Дополнительные методы

Примеры:

```java
Integer.parseInt("123");
Integer.valueOf("123");
Integer.compare(10, 20);
Double.isNaN(0.0 / 0.0);
```

---

## Автоупаковка (Autoboxing)

**Autoboxing** — автоматическое преобразование примитива в объект-обертку.

```java
Integer a = 10;
```

Компилятор преобразует это в:

```java
Integer a = Integer.valueOf(10);
```

---

## Автораспаковка (Unboxing)

**Unboxing** — автоматическое преобразование обертки в примитив.

```java
Integer a = 10;
int b = a;
```

Компилятор делает:

```java
int b = a.intValue();
```

---

## Где происходит автоупаковка

Автоупаковка работает в:

```java
// присваивание
Integer a = 10;

// методы
List<Integer> list = new ArrayList<>();
list.add(5);

// выражения
Integer x = 10 + 20;
```

---

## Проблема NullPointerException при unboxing

Если обертка `null`, при распаковке возникает ошибка:

```java
Integer x = null;

int y = x; // NullPointerException
```

Причина:

```java
x.intValue(); // вызов метода у null
```

---

## Кэширование Integer (важный экзаменационный момент)

Java кеширует значения `Integer` в диапазоне:

```text
-128 .. 127
```

Пример:

```java
Integer a = 100;
Integer b = 100;

System.out.println(a == b); // true
```

Но:

```java
Integer a = 200;
Integer b = 200;

System.out.println(a == b); // false
```

### Почему?

Одинаковые ссылки используются только для кешируемых значений.

---

## `==` vs `equals`

### `==`

Сравнивает ссылки:

```java
Integer a = 100;
Integer b = 100;

a == b // сравнение объектов
```

---

### `equals`

Сравнивает значения:

```java
a.equals(b) // сравнение чисел
```

---

### Правило:

```text
Для оберток всегда использовать equals()
```

## Полиморфизм через `Number`

Можно хранить разные числа:

```java
Number n1 = Integer.valueOf(10);
Number n2 = Double.valueOf(5.5);
Number n3 = Long.valueOf(100L);
```

И работать единообразно:

```java
System.out.println(n1.doubleValue());
System.out.println(n2.intValue());
```

---

## Важные особенности оберток

### 1. Immutable (неизменяемые)

```java
Integer a = 10;
a = a + 5;
```

Создается новый объект.

---

### 2. Производительность

Обертки медленнее примитивов из-за:

* объектов в heap;
* GC нагрузки.

---

### 3. Частое использование в коллекциях

```java
List<Integer> list = new ArrayList<>();
```

---

## Сравнение примитивов и оберток

| Характеристика | primitive | wrapper |
|----------------|----------|---------|
| null | ❌ | ✅ |
| коллекции | ❌ | ✅ |
| производительность | выше | ниже |
| методы | нет | есть |
| память | stack | heap |

---

## Шпаргалка

* `Number` — базовый класс всех числовых оберток.
* Обертки: `Integer`, `Double`, `Long`, `Float`, `Short`, `Byte`, `Boolean`, `Character`.
* Нужны для коллекций и объектной модели Java.
* Autoboxing: `int → Integer`.
* Unboxing: `Integer → int`.
* При `null` во время unboxing возникает `NullPointerException`.
* `==` сравнивает ссылки, `equals()` — значения.
* Integer кеширует значения `-128..127`.
* Все обертки immutable.
* Примитивы быстрее, но не имеют объектных возможностей.
