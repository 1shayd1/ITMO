# 5. Обход элементов коллекции. Интерфейсы `Iterable`, `Iterator` и `ListIterator`

## Общая идея обхода коллекций

Обход коллекций в Java — это последовательный доступ к элементам структуры данных.

Основные механизмы:

* `for-each` цикл (синтаксический сахар)
* `Iterator`
* `ListIterator`

Все они основаны на интерфейсе `Iterable`.

---

## Интерфейс `Iterable`

`Iterable<T>` — базовый интерфейс, который означает, что объект можно перебирать.

### Основной метод:

```java
Iterator<T> iterator();
```

---

## Роль `Iterable`

Если класс реализует `Iterable`, то он может использоваться в:

```java
for (T element : collection)
```

---

### Пример

```java
List<String> list = List.of("A", "B", "C");

for (String s : list) {
    System.out.println(s);
}
```

---

### Как это работает внутри

Компилятор преобразует `for-each` в:

```java
Iterator<String> it = list.iterator();

while (it.hasNext()) {
    System.out.println(it.next());
}
```

---

## Интерфейс `Iterator`

`Iterator<T>` — основной инструмент последовательного обхода коллекции.

---

### Основные методы:

```java
boolean hasNext();
T next();
void remove();
```

---

### Пример использования

```java
List<String> list = new ArrayList<>(List.of("A", "B", "C"));

Iterator<String> it = list.iterator();

while (it.hasNext()) {
    String value = it.next();
    System.out.println(value);
}
```

---

## Метод `remove()` в Iterator

Позволяет безопасно удалять элементы во время обхода.

```java
Iterator<String> it = list.iterator();

while (it.hasNext()) {
    String value = it.next();

    if (value.equals("B")) {
        it.remove();
    }
}
```

---

### Важно

❌ Нельзя так:

```java
for (String s : list) {
    list.remove(s); // ConcurrentModificationException
}
```

✔ Правильно через iterator:

```java
Iterator<String> it = list.iterator();
```

---

## ConcurrentModificationException (fail-fast)

Большинство коллекций Java используют механизм:

```text
modCount
```

Если коллекция изменяется не через `Iterator`, возникает:

```text
ConcurrentModificationException
```

---

## Интерфейс `ListIterator`

`ListIterator<T>` — расширение `Iterator`, работающее **только для List**.

---

### Получение:

```java
ListIterator<String> it = list.listIterator();
```

---

## Возможности `ListIterator`

### 1. Движение вперед и назад

```java
hasNext()
next()

hasPrevious()
previous()
```

---

### 2. Индексы элементов

```java
nextIndex()
previousIndex()
```

---

### 3. Изменение элементов

```java
set(T e)
add(T e)
remove()
```

---

## Пример `ListIterator`

```java
List<String> list =
        new ArrayList<>(List.of("A", "B", "C"));

ListIterator<String> it = list.listIterator();

while (it.hasNext()) {

    String value = it.next();

    if (value.equals("B")) {
        it.set("BB");
    }
}
```

---

## Добавление элементов через ListIterator

```java
ListIterator<String> it = list.listIterator();

while (it.hasNext()) {
    String value = it.next();

    if (value.equals("A")) {
        it.add("NEW");
    }
}
```

---

## Сравнение `Iterator` и `ListIterator`

| Возможность | `Iterator` | `ListIterator` |
|------------|------------|----------------|
| `for-each` поддержка | да | да (через List) |
| движение вперед | да | да |
| движение назад | нет | да |
| изменение элементов | нет | да (`set`) |
| добавление элементов | нет | да |
| получение индекса | нет | да |
| работа с коллекциями | любые `Collection` | только `List` |

---

## Связь интерфейсов

```text
Iterable
   ↓
Collection
   ↓
Iterator (создается через iterator())
   ↓
ListIterator (только для List)
```

---

## Практическое правило

* `for-each` → удобно, но ограничено
* `Iterator` → безопасное удаление
* `ListIterator` → полный контроль над списком

---

## Частые ошибки

### 1. Удаление в for-each

```java
for (String s : list) {
    list.remove(s); // ошибка
}
```

---

### 2. next() без проверки

```java
it.next(); // если элементов нет → исключение
```

---

## Шпаргалка

* `Iterable` — поддержка `for-each`.
* `Iterator` — последовательный обход коллекции.
* `Iterator.remove()` — безопасное удаление.
* `ListIterator` — расширенный итератор для `List`.
* `ListIterator` умеет идти назад и менять элементы.
* Нельзя модифицировать коллекцию вне iterator → `ConcurrentModificationException`.
* `for-each` внутри использует `Iterator`.
