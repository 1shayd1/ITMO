# 41. Паттерны проектирования. Структурные паттерны. Примеры

**Структурные паттерны** (structural patterns) описывают, как классы и объекты объединяются в более крупные структуры, решая проблемы "связывания" и наследования.

---

## 📋 Классификация структурных паттернов (GoF)

| Паттерн | Суть | Ключевая идея |
|---------|------|---------------|
| **Adapter** | Преобразует интерфейс класса в другой ожидаемый интерфейс | "Обертка" для несовместимых API |
| **Decorator** | Динамически добавляет новые обязанности | Композиция вместо наследования |
| **Facade** | Предоставляет унифицированный интерфейс к подсистеме | Упрощение сложности |
| **Composite** | Объединяет объекты в древовидную структуру | Единообразие для примитивов и контейнеров |
| **Proxy** | Заместитель для контроля доступа | Ленивая загрузка, кэширование, защита |
| **Bridge** | Разделяет абстракцию и реализацию | Независимое развитие двух иерархий |
| **Flyweight** | Разделяет общее состояние для экономии памяти | Кэширование мелких объектов |

## 1. Adapter (Адаптер)

Преобразует интерфейс одного класса в интерфейс, ожидаемый клиентом.

### Пример: интеграция разных платёжных систем

```java
// Target interface (ожидаемый клиентом интерфейс)
interface PaymentProcessor {
    void pay(double amount);
}

// Adaptee 1 — старая система
class LegacyPaySystem {
    void sendPayment(String currency, double amount) {
        System.out.println("Legacy: " + amount + " " + currency);
    }
}

// Adapter для LegacyPaySystem
class LegacyAdapter implements PaymentProcessor {
    private LegacyPaySystem legacy;
    
    LegacyAdapter(LegacyPaySystem legacy) { this.legacy = legacy; }
    
    @Override
    public void pay(double amount) {
        legacy.sendPayment("USD", amount);
    }
}

// Adaptee 2 — Stripe API
class StripeAPI {
    void charge(String customerId, int cents) {
        System.out.println("Stripe: " + cents + " cents for " + customerId);
    }
}

// Adapter для Stripe
class StripeAdapter implements PaymentProcessor {
    private StripeAPI stripe;
    private String customerId;
    
    StripeAdapter(StripeAPI stripe, String customerId) {
        this.stripe = stripe;
        this.customerId = customerId;
    }
    
    @Override
    public void pay(double amount) {
        stripe.charge(customerId, (int)(amount * 100));
    }
}

// Клиент
class Checkout {
    private PaymentProcessor processor;
    
    void setProcessor(PaymentProcessor p) { this.processor = p; }
    void checkout(double total) { processor.pay(total); }
}

// Использование
Checkout cart = new Checkout();
cart.setProcessor(new LegacyAdapter(new LegacyPaySystem()));
cart.checkout(99.99);  // Legacy: 99.99 USD

cart.setProcessor(new StripeAdapter(new StripeAPI(), "cust_123"));
cart.checkout(49.99);  // Stripe: 4999 cents for cust_123
```

### Complexity

| Операция | Complexity |
|----------|------------|
| Вызов адаптированного метода | O(1) (плюс делегация) |

## 2. Decorator (Декоратор)

Динамически добавляет объекту новую функциональность через обёртки.

### Пример: кофе с добавками

```java
// Component
interface Coffee {
    String getDescription();
    double getCost();
}

// Concrete component
class SimpleCoffee implements Coffee {
    public String getDescription() { return "Simple coffee"; }
    public double getCost() { return 2.0; }
}

// Decorator
abstract class CoffeeDecorator implements Coffee {
    protected Coffee coffee;
    CoffeeDecorator(Coffee coffee) { this.coffee = coffee; }
}

// Concrete decorators
class Milk extends CoffeeDecorator {
    Milk(Coffee coffee) { super(coffee); }
    public String getDescription() { return coffee.getDescription() + ", milk"; }
    public double getCost() { return coffee.getCost() + 0.5; }
}

class Sugar extends CoffeeDecorator {
    Sugar(Coffee coffee) { super(coffee); }
    public String getDescription() { return coffee.getDescription() + ", sugar"; }
    public double getCost() { return coffee.getCost() + 0.2; }
}

class Whip extends CoffeeDecorator {
    Whip(Coffee coffee) { super(coffee); }
    public String getDescription() { return coffee.getDescription() + ", whipped cream"; }
    public double getCost() { return coffee.getCost() + 0.7; }
}

// Использование
Coffee coffee = new SimpleCoffee();
coffee = new Milk(coffee);
coffee = new Sugar(coffee);
coffee = new Whip(coffee);

System.out.println(coffee.getDescription() + " $" + coffee.getCost());
// Simple coffee, milk, sugar, whipped cream $3.4
```

### Decorator в Java I/O

```java
// Классический пример из JDK
InputStream in = new BufferedInputStream(
    new GZIPInputStream(
        new FileInputStream("file.gz")
    )
);
```

### Complexity

| Операция | Complexity |
|----------|------------|
| Оборачивание | O(1) |
| Вызов через цепочку | O(глубина) |

## 3. Facade (Фасад)

Предоставляет упрощённый интерфейс к сложной подсистеме.

### Пример: домашний кинотеатр

```java
// Сложная подсистема
class Amplifier {
    void on() { System.out.println("Amp on"); }
    void setVolume(int v) { System.out.println("Volume: " + v); }
    void off() { System.out.println("Amp off"); }
}

class DVDPlayer {
    void on() { System.out.println("DVD on"); }
    void play(String m) { System.out.println("Playing " + m); }
    void stop() { System.out.println("DVD stop"); }
    void off() { System.out.println("DVD off"); }
}

class Projector {
    void on() { System.out.println("Projector on"); }
    void wideScreen() { System.out.println("Wide screen"); }
    void off() { System.out.println("Projector off"); }
}

class Lights {
    void dim(int level) { System.out.println("Lights dim " + level + "%"); }
    void on() { System.out.println("Lights on"); }
}

// Facade
class HomeTheaterFacade {
    private Amplifier amp;
    private DVDPlayer dvd;
    private Projector projector;
    private Lights lights;
    
    HomeTheaterFacade(Amplifier amp, DVDPlayer dvd, Projector projector, Lights lights) {
        this.amp = amp;
        this.dvd = dvd;
        this.projector = projector;
        this.lights = lights;
    }
    
    void watchMovie(String movie) {
        lights.dim(10);
        projector.on();
        projector.wideScreen();
        amp.on();
        amp.setVolume(5);
        dvd.on();
        dvd.play(movie);
    }
    
    void endMovie() {
        dvd.stop();
        dvd.off();
        amp.off();
        projector.off();
        lights.on();
    }
}

// Использование
HomeTheaterFacade theater = new HomeTheaterFacade(
    new Amplifier(), new DVDPlayer(), new Projector(), new Lights()
);
theater.watchMovie("Inception");
theater.endMovie();
```

### Complexity

| Операция | Complexity |
|----------|------------|
| Вызов фасада | O(1) + сумма сложностей подсистемы |

## 4. Composite (Компоновщик)

Позволяет работать с деревом объектов единообразно (отдельный объект или группа).

### Пример: файловая система

```java
// Component
interface FileSystemNode {
    String getName();
    int getSize();
    void print(String indent);
}

// Leaf
class File implements FileSystemNode {
    private String name;
    private int size;
    
    File(String name, int size) { this.name = name; this.size = size; }
    
    public String getName() { return name; }
    public int getSize() { return size; }
    public void print(String indent) {
        System.out.println(indent + "📄 " + name + " (" + size + " KB)");
    }
}

// Composite
class Directory implements FileSystemNode {
    private String name;
    private List<FileSystemNode> children = new ArrayList<>();
    
    Directory(String name) { this.name = name; }
    
    void add(FileSystemNode node) { children.add(node); }
    void remove(FileSystemNode node) { children.remove(node); }
    
    public String getName() { return name; }
    
    public int getSize() {
        return children.stream().mapToInt(FileSystemNode::getSize).sum();
    }
    
    public void print(String indent) {
        System.out.println(indent + "📁 " + name);
        children.forEach(c -> c.print(indent + "  "));
    }
}

// Использование
Directory root = new Directory("root");
Directory docs = new Directory("docs");
docs.add(new File("resume.pdf", 250));
docs.add(new File("cover.txt", 15));
root.add(docs);
root.add(new File("readme.md", 5));

root.print("");
System.out.println("Total: " + root.getSize() + " KB");
// 📁 root
//   📁 docs
//     📄 resume.pdf (250 KB)
//     📄 cover.txt (15 KB)
//   📄 readme.md (5 KB)
// Total: 270 KB
```

### Complexity

| Операция | Complexity |
|----------|------------|
| Обход дерева | O(n) |
| Получение размера (без кэша) | O(n) |

## 5. Proxy (Заместитель)

Контролирует доступ к другому объекту (ленивая загрузка, кэширование, защита).

### Пример: Virtual Proxy (ленивая загрузка)

```java
interface Image {
    void display();
}

// Тяжёлый объект
class RealImage implements Image {
    private String filename;
    
    RealImage(String filename) {
        this.filename = filename;
        loadFromDisk();  // дорогая операция
    }
    
    private void loadFromDisk() {
        System.out.println("Loading " + filename + " (expensive)");
    }
    
    public void display() { System.out.println("Displaying " + filename); }
}

// Proxy — ленивая инициализация
class ImageProxy implements Image {
    private String filename;
    private RealImage realImage;
    
    ImageProxy(String filename) { this.filename = filename; }
    
    public void display() {
        if (realImage == null) {
            realImage = new RealImage(filename);  // только при первом вызове
        }
        realImage.display();
    }
}

// Использование
Image img = new ImageProxy("photo.jpg");  // ничего не загружено
img.display();  // Loading photo.jpg (expensive) \n Displaying photo.jpg
img.display();  // Displaying photo.jpg (из кэша)
```

### Protection Proxy (контроль доступа)

```java
interface Database {
    void query(String sql);
}

class RealDatabase implements Database {
    public void query(String sql) { System.out.println("Exec: " + sql); }
}

class DatabaseProxy implements Database {
    private RealDatabase real = new RealDatabase();
    private String role;
    
    DatabaseProxy(String role) { this.role = role; }
    
    public void query(String sql) {
        if (sql.toLowerCase().contains("drop") && !"ADMIN".equals(role)) {
            System.out.println("Access denied: DROP requires ADMIN");
            return;
        }
        real.query(sql);
    }
}

// Использование
Database db = new DatabaseProxy("USER");
db.query("SELECT * FROM users");  // Exec: SELECT * FROM users
db.query("DROP TABLE users");     // Access denied!
```

### Dynamic Proxy (Java Reflection)

```java
import java.lang.reflect.*;

interface Service {
    String serve(String input);
}

class RealService implements Service {
    public String serve(String input) { return "Echo: " + input; }
}

class LoggingHandler implements InvocationHandler {
    private Object target;
    LoggingHandler(Object target) { this.target = target; }
    
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("[LOG] Before " + method.getName());
        Object result = method.invoke(target, args);
        System.out.println("[LOG] After -> " + result);
        return result;
    }
}

// Использование
Service real = new RealService();
Service proxy = (Service) Proxy.newProxyInstance(
    Service.class.getClassLoader(),
    new Class[]{Service.class},
    new LoggingHandler(real)
);
proxy.serve("Hello");
// [LOG] Before serve
// [LOG] After -> Echo: Hello
```

### Complexity

| Тип Proxy | Time | Memory |
|-----------|------|--------|
| Virtual (ленивый) | O(инициализация) один раз | O(1) до инициализации |
| Cache | O(1) для кэшированных | O(кэш) |
| Protection | O(1) | Минимальный |

## 6. Bridge (Мост)

Разделяет абстракцию и реализацию, позволяя изменять их независимо.

### Пример: устройства и пульты

```java
// Implementation (устройства)
interface Device {
    void on();
    void off();
    void setVolume(int percent);
    int getVolume();
}

class TV implements Device {
    private int volume = 10;
    public void on() { System.out.println("TV on"); }
    public void off() { System.out.println("TV off"); }
    public void setVolume(int v) { volume = v; System.out.println("TV volume: " + v); }
    public int getVolume() { return volume; }
}

class Radio implements Device {
    private int volume = 20;
    public void on() { System.out.println("Radio on"); }
    public void off() { System.out.println("Radio off"); }
    public void setVolume(int v) { volume = v; System.out.println("Radio volume: " + v); }
    public int getVolume() { return volume; }
}

// Abstraction (пульты)
abstract class Remote {
    protected Device device;
    Remote(Device device) { this.device = device; }
    abstract void up();
    abstract void down();
    void toggle() { /* optional */ }
}

class BasicRemote extends Remote {
    BasicRemote(Device device) { super(device); }
    void up() { device.setVolume(device.getVolume() + 10); }
    void down() { device.setVolume(device.getVolume() - 10); }
}

class AdvancedRemote extends Remote {
    AdvancedRemote(Device device) { super(device); }
    void up() { device.setVolume(device.getVolume() + 5); }
    void down() { device.setVolume(device.getVolume() - 5); }
    void mute() { device.setVolume(0); }
}

// Использование
Device tv = new TV();
Remote remote = new AdvancedRemote(tv);
remote.up();   // TV volume: 15
((AdvancedRemote)remote).mute(); // TV volume: 0
```

### Без Bridge: 2×2 = 4 класса (TVBasicRemote, TVAdvancedRemote, RadioBasicRemote, RadioAdvancedRemote). С Bridge: 2+2 = 4, но при росте (3 устройства × 4 пульта = 12 классов вместо 3+4=7)

### Complexity

| Операция | Complexity |
|----------|------------|
| Вызов через мост | O(1) (делегация) |

## 7. Flyweight (Приспособленец)

Разделяет общее состояние между множеством объектов для экономии памяти.

### Пример: текстовый редактор

```java
// Flyweight — разделяемое состояние (intrinsic)
class CharacterFlyweight {
    private final char symbol;
    private final String font;
    private final int size;
    
    CharacterFlyweight(char symbol, String font, int size) {
        this.symbol = symbol;
        this.font = font;
        this.size = size;
    }
    
    void render(int x, int y, String color) { // extrinsic state (x, y, color)
        System.out.printf("'%c' at (%d,%d), font %s, size %d, color %s%n",
                          symbol, x, y, font, size, color);
    }
}

// Flyweight Factory
class CharacterFactory {
    private static final Map<String, CharacterFlyweight> CACHE = new HashMap<>();
    
    static CharacterFlyweight get(char symbol, String font, int size) {
        String key = symbol + ":" + font + ":" + size;
        return CACHE.computeIfAbsent(key, k -> new CharacterFlyweight(symbol, font, size));
    }
    
    static int cacheSize() { return CACHE.size(); }
}

// Клиент
class TextEditor {
    private List<RenderRequest> chars = new ArrayList<>();
    
    void add(char symbol, String font, int size, int x, int y, String color) {
        CharacterFlyweight fly = CharacterFactory.get(symbol, font, size);
        chars.add(new RenderRequest(fly, x, y, color));
    }
    
    void render() { chars.forEach(r -> r.flyweight.render(r.x, r.y, r.color)); }
    
    private record RenderRequest(CharacterFlyweight flyweight, int x, int y, String color) {}
}

// Использование
TextEditor editor = new TextEditor();
editor.add('A', "Arial", 12, 10, 10, "red");
editor.add('A', "Arial", 12, 20, 10, "blue");
editor.add('B', "Arial", 12, 30, 10, "black");
editor.render();

System.out.println("Cache size: " + CharacterFactory.cacheSize()); // 2 (A и B)
```

### Экономия памяти

| Без Flyweight | С Flyweight |
|---------------|-------------|
| 1000 символов = 1000 объектов | 1000 символов = ~50 объектов (уникальных) |

### Complexity

| Операция | Complexity |
|----------|------------|
| Получение flyweight | O(1) (HashMap) |
| Память | O(уникальные_состояния) |

## 📊 Сравнительная таблица

| Паттерн | Проблема | Решение | Когда использовать |
|---------|----------|---------|---------------------|
| **Adapter** | Несовместимые интерфейсы | Обёртка с нужным интерфейсом | Интеграция legacy кода, сторонних библиотек |
| **Decorator** | Расширение функциональности | Композиция вместо наследования | Динамическое добавление поведения (I/O, UI) |
| **Facade** | Сложная подсистема | Упрощённый интерфейс | Библиотеки, API, сложные конфигурации |
| **Composite** | Древовидные структуры | Единый интерфейс для листьев и узлов | Файловые системы, UI-компоненты |
| **Proxy** | Контроль доступа, ленивость | Заместитель | Ленивая загрузка, кэш, логирование |
| **Bridge** | Картезианское произведение классов | Разделение абстракции и реализации | Когда две иерархии меняются независимо |
| **Flyweight** | Перерасход памяти | Разделение общего состояния | Много мелких однотипных объектов |

## 🔗 Связи с Java

| Паттерн | Пример в JDK |
|---------|--------------|
| **Adapter** | `Arrays.asList()`, `InputStreamReader` |
| **Decorator** | `BufferedInputStream`, `ZipInputStream` |
| **Facade** | `CompletableFuture` (упрощает async) |
| **Composite** | `JComponent` в Swing (контейнеры и компоненты) |
| **Proxy** | `Proxy` (динамический прокси), RMI |
| **Bridge** | JDBC (драйверы — реализация, `Connection` — абстракция) |
| **Flyweight** | `Integer.valueOf()` (кэш -128..127), `String.intern()` |

```java
// Flyweight в JDK
Integer a = Integer.valueOf(127); // из кэша
Integer b = Integer.valueOf(127); // тот же объект
System.out.println(a == b); // true

// Decorator в I/O
BufferedReader reader = new BufferedReader(
    new FileReader("file.txt")
);
```

### Шпаргалка

* **Adapter** — чинит интерфейсы (`Arrays.asList()`)
* **Decorator** — добавляет поведение обёртками (I/O потоки)
* **Facade** — прячет сложность (один метод вместо 10)
* **Composite** — единообразная работа с деревьями (файлы + папки)
* **Proxy** — контролирует доступ (ленивая загрузка, кэш)
* **Bridge** — разделяет "что" и "как" (JDBC)
* **Flyweight** — экономит память (кэш `Integer`)
* **Структурные паттерны говорят о том, как построить систему из классов и объектов**
