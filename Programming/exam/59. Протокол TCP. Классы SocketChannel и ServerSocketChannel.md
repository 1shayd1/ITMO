# 59. Протокол TCP. Классы SocketChannel и ServerSocketChannel

`SocketChannel` и `ServerSocketChannel` (пакет `java.nio.channels`) — **NIO-аналоги** классических `Socket` и `ServerSocket`. Главное отличие — поддержка **неблокирующего режима** и интеграция с `Selector`, что позволяет строить масштабируемые серверы.

## `ServerSocketChannel`

Принимает входящие TCP-подключения:

```java
ServerSocketChannel server = ServerSocketChannel.open();
server.bind(new InetSocketAddress(8080));
server.configureBlocking(false);
SocketChannel client = server.accept();   // в неблокирующем режиме: null, если никто не подключился
```

* в **блокирующем** режиме `accept()` ждёт клиента (как у `ServerSocket`);
* в **неблокирующем** — мгновенно возвращает `null` при отсутствии подключений; обычно вместо опроса канал регистрируют в селекторе с `OP_ACCEPT`.

## `SocketChannel`

Представляет TCP-соединение; работает с `ByteBuffer`:

```java
SocketChannel ch = SocketChannel.open();
ch.configureBlocking(false);
ch.connect(new InetSocketAddress("localhost", 8080));
// в неблокирующем режиме connect возвращает управление сразу:
while (!ch.finishConnect()) { /* заняться другим / ждать OP_CONNECT */ }
```

### Чтение

```java
ByteBuffer buf = ByteBuffer.allocate(4096);
int n = ch.read(buf);
```

`read()` возвращает: **> 0** — прочитано байт; **0** — данных сейчас нет (неблокирующий режим); **−1** — собеседник закрыл соединение (EOF). TCP — поток: одна «логическая» посылка может прийти за несколько `read()`.

### Запись и частичная отправка

```java
buf.flip();
while (buf.hasRemaining()) {
    int written = ch.write(buf);   // может записать МЕНЬШЕ, чем remaining
    if (written == 0) { /* буфер ОС полон: зарегистрировать OP_WRITE и выйти */ }
}
```

`write()` в неблокирующем режиме пишет столько, сколько влезает в буфер отправки ОС, — возможна **частичная запись**. Правильный паттерн: дописывать в цикле, а при `0` — включить интерес `OP_WRITE` и продолжить, когда селектор сообщит о готовности.

Поддерживаются **scatter/gather**: `read(ByteBuffer[])` и `write(ByteBuffer[])` — чтение/запись сразу в несколько буферов (удобно для заголовок+тело).

## Интеграция с `Selector`

```java
server.register(selector, SelectionKey.OP_ACCEPT);
client.register(selector, SelectionKey.OP_READ, attachmentState);
```

Типичный сервер: `OP_ACCEPT` на серверном канале → принятый `SocketChannel` переводится в неблокирующий режим и регистрируется с `OP_READ`; `OP_WRITE` добавляется только при незавершённой записи.

Опции настраиваются через `setOption()`: `ch.setOption(StandardSocketOptions.TCP_NODELAY, true)`, `SO_REUSEADDR`, `SO_RCVBUF` и др.

## Сравнение с классическими классами

| Свойство | `Socket` / `ServerSocket` | `SocketChannel` / `ServerSocketChannel` |
|---|---|---|
| Пакет | `java.net` (IO) | `java.nio.channels` (NIO) |
| Данные | `InputStream` / `OutputStream` | `ByteBuffer` |
| Режим | Только блокирующий | Блокирующий **или** неблокирующий |
| `Selector` | Нет | Да |
| Модель сервера | Поток на клиента | Event loop, один поток на тысячи соединений |

### Шпаргалка

* NIO-аналоги TCP-классов: `ServerSocketChannel.accept()` в неблокирующем режиме возвращает **null**.
* Неблокирующий `connect()` завершается через `finishConnect()` (сигнал — `OP_CONNECT`).
* `read()`: >0 / 0 (нет данных) / **−1** (EOF); `write()` может записать **частично** → цикл + `OP_WRITE`.
* Регистрация в `Selector` (`OP_ACCEPT`/`OP_READ`/`OP_WRITE`) — основа масштабируемого сервера.
* Данные — через `ByteBuffer` (помнить `flip()`); опции — `setOption(StandardSocketOptions...)`.
