# 33. Функциональные интерфейсы и lambda-выражения. Интерфейсы пакета java.util.function

Функциональный интерфейс — это интерфейс с одним абстрактным методом. Он служит целью для lambda-выражений и ссылок на методы.

---

## Функциональный интерфейс

```java
@FunctionalInterface
interface Task {
    void run();
}
```

Аннотация `@FunctionalInterface` необязательна, но рекомендуется — компилятор проверит, что в интерфейсе ровно один абстрактный метод, и выдаст ошибку если это не так.

Интерфейс может содержать:

- **один** абстрактный метод (обязательно)
- любое количество `default`-методов
- любое количество `static`-методов
- методы из `Object` (`equals`, `toString`) — они не считаются абстрактными

```java
@FunctionalInterface
interface Transformer<T> {
    T transform(T input);          // единственный абстрактный метод

    default T transformTwice(T input) { // default — не нарушает правило
        return transform(transform(input));
    }

    static <T> Transformer<T> identity() { // static — тоже не нарушает
        return x -> x;
    }
}
```

## Lambda-выражение

Lambda — это краткая запись реализации функционального интерфейса.

```java
Task t = () -> System.out.println("Hello");
```

### Синтаксис

```java
// Без параметров:
() -> System.out.println("Hello")

// Один параметр (скобки необязательны):
x -> x * 2
(x) -> x * 2

// Несколько параметров:
(a, b) -> a + b

// С указанием типов:
(String s, int n) -> s.repeat(n)

// Блок кода (нужен return):
(a, b) -> {
    int sum = a + b;
    return sum;
}
```

### Effectively final

Переменные из внешнего контекста, используемые в lambda, должны быть **effectively final** — то есть не изменяться после присвоения (явный `final` необязателен):

```java
int x = 10;
Runnable r = () -> System.out.println(x); // OK — x не меняется

x = 20; // Ошибка компиляции! После этого x уже не effectively final,
         // и lambda выше не скомпилируется
```

## Ссылки на методы

Ссылки на методы — ещё более краткая запись lambda, когда она просто делегирует вызов.

|Вид|Синтаксис|Эквивалентная lambda|
|---|---|---|
|Статический метод|`ClassName::staticMethod`|`x -> ClassName.staticMethod(x)`|
|Метод экземпляра (произвольного)|`ClassName::instanceMethod`|`(obj, x) -> obj.instanceMethod(x)`|
|Метод экземпляра (конкретного)|`instance::method`|`x -> instance.method(x)`|
|Конструктор|`ClassName::new`|`x -> new ClassName(x)`|

```java
Function<String, Integer> f1 = String::length;       // метод экземпляра
Consumer<String> c1 = System.out::println;            // метод конкретного объекта
Supplier<List<String>> s1 = ArrayList::new;           // конструктор
Function<String, String> f2 = String::toUpperCase;   // метод экземпляра
```

## Основные интерфейсы `java.util.function`

- `Predicate<T>` — принимает `T`, возвращает `boolean`
- `Function<T, R>` — преобразует `T` в `R`
- `Consumer<T>` — принимает `T`, ничего не возвращает
- `Supplier<T>` — ничего не принимает, возвращает `T`
- `UnaryOperator<T>` — `T -> T`
- `BinaryOperator<T>` — `(T, T) -> T`

---

## Predicate\<T\>

```java
@FunctionalInterface
public interface Predicate<T> {
    boolean test(T t);
}
```

```java
Predicate<String> isEmpty = String::isEmpty;
Predicate<String> isLong = s -> s.length() > 10;

isEmpty.test("");       // true
isEmpty.test("hi");     // false
```

### Композиция Predicate

```java
Predicate<String> notEmpty = isEmpty.negate();           // NOT
Predicate<String> both = isEmpty.and(isLong);            // AND
Predicate<String> either = isEmpty.or(isLong);           // OR

Predicate<String> isHello = Predicate.not(String::isEmpty); // статический not (Java 11+)
```

---

## Function<T, R>

```java
@FunctionalInterface
public interface Function<T, R> {
    R apply(T t);
}
```

```java
Function<String, Integer> length = String::length;
Function<Integer, String> toStr = Object::toString;

length.apply("hello"); // 5
```

### Композиция Function

```java
Function<String, Integer> f = String::length;
Function<Integer, Integer> doubleIt = n -> n * 2;

Function<String, Integer> composed = f.andThen(doubleIt); // сначала f, потом doubleIt
composed.apply("hello"); // 10

Function<String, Integer> composed2 = doubleIt.compose(f); // то же самое, другой порядок записи
```

### BiFunction<T, U, R>

Принимает два аргумента:

```java
BiFunction<String, Integer, String> repeat = String::repeat;
repeat.apply("ab", 3); // "ababab"
```

---

## Consumer\<T\>

```java
@FunctionalInterface
public interface Consumer<T> {
    void accept(T t);
}
```

```java
Consumer<String> print = System.out::println;
Consumer<String> printUpper = s -> System.out.println(s.toUpperCase());

print.accept("hello");

// Цепочка consumers:
Consumer<String> both = print.andThen(printUpper);
both.accept("hello"); // напечатает "hello", потом "HELLO"
```

### BiConsumer<T, U>

```java
BiConsumer<String, Integer> repeat = (s, n) -> System.out.println(s.repeat(n));
repeat.accept("ha", 3); // "hahaha"
```

---

## Supplier\<T\>

```java
@FunctionalInterface
public interface Supplier<T> {
    T get();
}
```

```java
Supplier<Double> random = Math::random;
Supplier<List<String>> listFactory = ArrayList::new;

random.get();        // случайное число
listFactory.get();   // новый пустой ArrayList
```

Часто используется для **ленивой инициализации**:

```java
// Значение вычисляется только когда нужно, а не при создании:
Optional<String> result = Optional.empty();
String value = result.orElseGet(() -> expensiveComputation());
```

---

## UnaryOperator\<T\> и BinaryOperator\<T\>

`UnaryOperator<T>` — частный случай `Function<T, T>`:

```java
UnaryOperator<String> toUpper = String::toUpperCase;
toUpper.apply("hello"); // "HELLO"
```

`BinaryOperator<T>` — частный случай `BiFunction<T, T, T>`:

```java
BinaryOperator<Integer> sum = Integer::sum;
BinaryOperator<String> concat = String::concat;

sum.apply(3, 4);           // 7
concat.apply("Hi", "!");   // "Hi!"
```

## Примитивные специализации

Для избежания автобоксинга существуют специализированные интерфейсы:

|Интерфейс|Аналог|
|---|---|
|`IntPredicate`|`Predicate<Integer>` без боксинга|
|`IntFunction<R>`|`Function<Integer, R>` без боксинга|
|`IntConsumer`|`Consumer<Integer>` без боксинга|
|`IntSupplier`|`Supplier<Integer>` без боксинга|
|`IntUnaryOperator`|`UnaryOperator<Integer>` без боксинга|
|`ToIntFunction<T>`|`Function<T, Integer>` без боксинга|

Аналогично для `Long` и `Double`.

```java
IntPredicate isEven = n -> n % 2 == 0; // int, не Integer — нет боксинга
isEven.test(4); // true
```

Predicate<String> p = s -> s.isEmpty();

## Важные особенности

- Lambda делает код короче и читабельнее.
- Переменные, используемые в lambda, должны быть effectively final.
- Lambda не создаёт отдельный именованный класс в коде.
- Для повторяющегося поведения можно использовать ссылки на методы.
- Все интерфейсы поддерживают **композицию** через `andThen`, `compose`, `and`, `or`, `negate`.
- Примитивные специализации (`IntPredicate`, `ToIntFunction` и др.) устраняют накладные расходы автобоксинга.

---

## Шпаргалка

- Функциональный интерфейс имеет один абстрактный метод.
- Lambda-выражения реализуют такие интерфейсы.
- `Predicate` возвращает `boolean`; поддерживает `and`, `or`, `negate`.
- `Function` преобразует значение; поддерживает `andThen`, `compose`.
- `Consumer` потребляет значение, `Supplier` его предоставляет.
- `UnaryOperator` и `BinaryOperator` — специализации `Function` для одного типа.
- Ссылки на методы (`::`) — краткая запись lambda без дополнительной логики.
- Effectively final: переменные из внешнего контекста нельзя изменять внутри lambda.
- Примитивные специализации (`IntPredicate` и др.) — для горячих путей без боксинга.
