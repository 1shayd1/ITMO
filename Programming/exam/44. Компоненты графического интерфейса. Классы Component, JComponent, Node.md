
# 44. Компоненты графического интерфейса. Классы Component, JComponent, Node

Графический интерфейс в Java строится из **компонентов** — визуальных элементов. В зависимости от технологии (AWT, Swing, JavaFX) используются разные иерархии базовых классов.

---

## 📚 Три GUI-стэка Java

| Технология | Базовый класс | Контейнер верхнего уровня | Статус |
|------------|---------------|---------------------------|--------|
| **AWT** | `java.awt.Component` | `Frame`, `Dialog` | Устаревшая |
| **Swing** | `javax.swing.JComponent` | `JFrame`, `JDialog` | Legacy, но работает |
| **JavaFX** | `javafx.scene.Node` | `Stage`, `Scene` | Современная |

---

## 1. AWT — `Component`

`Component` — базовый класс всех AWT-компонентов. Абстрактный класс, содержащий общее состояние (позиция, размер, цвет, шрифт, видимость, фокус).

```java
// Базовые методы Component
Component c = new Button("Click");

c.setBounds(10, 10, 100, 30);  // позиция + размер
c.setSize(100, 30);
c.setLocation(10, 10);
c.setVisible(true);
c.setEnabled(false);
c.setBackground(Color.BLUE);
c.setForeground(Color.WHITE);
c.setFont(new Font("Arial", Font.BOLD, 12));
c.requestFocus();
```

### Иерархия AWT (упрощённо)

```
java.awt.Component
  ├── Button
  ├── Label
  ├── TextComponent (TextField, TextArea)
  ├── Checkbox
  ├── Choice
  ├── List
  ├── Scrollbar
  ├── Canvas
  └── Container
       ├── Panel
       └── Window
            ├── Frame
            └── Dialog
```

### Пример AWT

```java
Frame frame = new Frame("AWT Example");
Label label = new Label("0");
Button button = new Button("Click");

button.addActionListener(e -> {
    int val = Integer.parseInt(label.getText());
    label.setText(String.valueOf(val + 1));
});

frame.setLayout(new FlowLayout());
frame.add(label);
frame.add(button);
frame.setSize(200, 100);
frame.setVisible(true);
```

## 2. Swing — `JComponent`

`JComponent` — базовый класс Swing, наследник `Container` из AWT. Добавляет расширенные возможности по сравнению с AWT:

* **Плавающие тултипы** (`setToolTipText()`)
* **Границы** (`setBorder()`)
* **Поддержка клавиатурного навигации**
* **Двойная буферизация** (устранение мерцания)
* **Action API** (действия без привязки к компоненту)
* **InputMap / ActionMap** (маппинг клавиш)
* **Поддержка всплывающих меню**

```java
// Дополнительные методы JComponent
JButton button = new JButton("OK");

button.setToolTipText("Click to submit");
button.setBorder(BorderFactory.createLineBorder(Color.BLACK));
button.setOpaque(true);
button.setFocusable(true);
button.setMnemonic(KeyEvent.VK_O);  // Alt+O
button.registerKeyboardAction(e -> System.out.println("Pressed"),
    KeyStroke.getKeyStroke(KeyEvent.VK_ENTER, 0),
    JComponent.WHEN_FOCUSED);
```

### Иерархия Swing (упрощённо)

```
            ├── JButton

```

### Пример Swing

```java
JFrame frame = new JFrame("Swing Example");
JLabel label = new JLabel("0", SwingConstants.CENTER);
JButton button = new JButton("Click");

label.setFont(new Font("Arial", Font.BOLD, 24));
button.addActionListener(e -> {
    int val = Integer.parseInt(label.getText());
    label.setText(String.valueOf(val + 1));
});

frame.setLayout(new BorderLayout());
frame.add(label, BorderLayout.CENTER);
frame.add(button, BorderLayout.SOUTH);
frame.setSize(200, 150);
frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
frame.setVisible(true);
```

## 3. JavaFX — `Node`

`Node` — базовый класс **всех элементов сцены** в JavaFX. В отличие от AWT/Swing, JavaFX использует **граф сцены** (Scene Graph) — древовидную структуру узлов.

### Основные подклассы Node

```
javafx.scene.Node
  ├── Parent (узлы, которые могут иметь детей)
  │    ├── Region
  │    │    ├── Pane
  │    │    ├── HBox, VBox, FlowPane, GridPane, BorderPane
  │    │    ├── StackPane
  │    │    └── ScrollPane
  │    ├── Group
  │    └── Control
  │         ├── Button
  │         ├── Label
  │         ├── TextField
  │         ├── TextArea
  │         ├── CheckBox
  │         ├── RadioButton
  │         ├── ComboBox
  │         ├── ListView
  │         ├── TableView
  │         ├── TreeView
  │         └── Slider
  ├── Shape
  │    ├── Rectangle, Circle, Ellipse, Line
  │    └── Polygon, Polyline
  ├── Text
  ├── ImageView
  ├── Canvas
  └── MediaView
```

### Основные свойства и методы Node

```java
Button button = new Button("Click");

// Свойства (JavaFX использует Property API)
button.setText("Click me");
button.setLayoutX(10);
button.setLayoutY(10);
button.setPrefWidth(100);
button.setPrefHeight(30);
button.setVisible(true);
button.setDisable(false);
button.setOpacity(0.8);
button.setRotate(45);
button.setScaleX(1.5);
button.setStyle("-fx-background-color: blue; -fx-text-fill: white;");

// Обработка событий
button.setOnAction(e -> System.out.println("Clicked"));

// Эффекты
button.setEffect(new DropShadow());

// Анимация
TranslateTransition tt = new TranslateTransition(Duration.seconds(2), button);
tt.setByX(100);
tt.setAutoReverse(true);
tt.setCycleCount(Animation.INDEFINITE);
tt.play();
```

### Пример JavaFX

```java
import javafx.application.Application;
import javafx.scene.Scene;
import javafx.scene.control.*;
import javafx.scene.layout.*;
import javafx.stage.Stage;

public class JavaFxExample extends Application {
    @Override
    public void start(Stage stage) {
        Label label = new Label("0");
        label.setStyle("-fx-font-size: 24px; -fx-alignment: center;");
        
        Button button = new Button("Click");
        button.setOnAction(e -> {
            int val = Integer.parseInt(label.getText());
            label.setText(String.valueOf(val + 1));
        });
        
        VBox root = new VBox(10, label, button);
        root.setStyle("-fx-padding: 20; -fx-alignment: center;");
        
        Scene scene = new Scene(root, 200, 150);
        stage.setTitle("JavaFX Example");
        stage.setScene(scene);
        stage.show();
    }
    
    public static void main(String[] args) {
        launch(args);
    }
}
```

## 📊 Сравнение трёх подходов

| Характеристика | AWT | Swing | JavaFX |
|----------------|-----|-------|--------|
| **Базовый класс** | `Component` | `JComponent` | `Node` |
| **Архитектура** | Тяжёлые компоненты (peer) | Лёгкие компоненты (pure Java) | Граф сцены |
| **Отрисовка** | Нативная (OS) | Java (Swing painting) | Java + GPU (Prism) |
| **Стилизация** | Минимальная | PLAF (Metal, Nimbus, GTK) | CSS |
| **Анимация** | Нет | `Timer` + ручная отрисовка | `Animation`, `Transition` |
| **Медиа** | Нет | Нет | Audio, Video |
| **FXML** | Нет | Нет | ✅ |
| **Приоритет** | ❌ Не использовать | ⚠️ Легаси | ✅ **Рекомендован** |

## 🔧 Подкапотные детали

### AWT: тяжелые компоненты

Каждый AWT-компонент имеет **native peer** — объект на уровне ОС (окно, кнопка).  
Плюсы: нативный вид.  
Минусы: проблемы с переносимостью, "мигание" при перерисовке.

### Swing: легкие компоненты

Swing компоненты рисуются Java-кодом, native peer есть только у корневого окна (`JFrame`).  
Плюсы: однородный вид на всех платформах.  
Минусы: медленнее нативной отрисовки.

### JavaFX: граф сцены

Сцена — это дерево `Node`. Изменение свойств узлов автоматически перерисовывает только изменённые области.  
Использует **приватный кумулятивный буфер** (Pulse) для синхронизации анимации с кадрами (60 FPS).  
Доступ к `Node` **должен быть из потока JavaFX** (Application Thread).

```java
// Правильный способ обновить UI из другого потока
Platform.runLater(() -> label.setText("Updated"));
```

## ⚠️ Частые ошибки

### ❌ Неправильно

```java
// 1. AWT/Swing — добавление компонента после setVisible(true)
Frame frame = new Frame();
frame.setVisible(true);
frame.add(new Button("Late"));  // Не отобразится

// 2. Swing — прямое изменение UI не в EDT
new Thread(() -> {
    myLabel.setText("Hello");  // ❌ Вне Event Dispatch Thread
}).start();

// 3. JavaFX — изменение Node не в JavaFX потоке
new Thread(() -> {
    label.setText("Hello");  // ❌ IllegalStateException
}).start();
```

### ✅ Правильно

```java
// 1. Сначала добавить, потом показать
frame.add(new Button("OK"));
frame.setVisible(true);

// 2. Swing — оборачивать в SwingUtilities.invokeLater
SwingUtilities.invokeLater(() -> myLabel.setText("Hello"));

// 3. JavaFX — Platform.runLater
Platform.runLater(() -> label.setText("Hello"));
```

## 🔗 Связи с другими темами

| Тема | Связь |
|------|-------|
| **Event Handling** | `addActionListener()` (Swing), `setOnAction()` (JavaFX) |
| **Многопоточность** | EDT в Swing (`invokeLater`), Platform.runLater в JavaFX |
| **MVC** | Swing (Model-Delegate), JavaFX (FXML + Controller) |
| **Layout Managers** | AWT/Swing (`FlowLayout`, `BorderLayout`, `GridBagLayout`), JavaFX (`VBox`, `HBox`, `GridPane`) |
| **Стили** | PLAF в Swing, CSS в JavaFX |

---

### Шпаргалка

* **AWT** → `Component` → устаревшая, тяжёлые компоненты
* **Swing** → `JComponent` → легкие компоненты, требует EDT (`SwingUtilities.invokeLater`)
* **JavaFX** → `Node` → современный граф сцены, CSS-стилизация, требует `Platform.runLater()`
* **Иерархия:** AWT и Swing — одно дерево, JavaFX — отдельное
* **Сегодня:** JavaFX — рекомендованный выбор для новых проектов
* **Swing ещё жив** в легаси-системах, банковском ПО
* **Всегда обновляй UI в правильном потоке** (EDT для Swing, JavaFX Thread для JavaFX)
