# 31. Атомарные операции. Пакет `java.util.concurrent.atomic`. Класс `AtomicInteger`

## Понятие атомарности

**Атомарная операция** — это операция, которая выполняется целиком, без промежуточных состояний, видимых другим потокам.

Это означает:
* операция либо выполнена полностью, либо не выполнена вообще;
* исключаются состояния "наполовину обновлено".

---

## Проблема обычных операций

Операция:

```java
count++;
```

на самом деле состоит из трёх шагов:

1. чтение значения;
2. увеличение;
3. запись обратно.

---

## Проблема в многопоточности

Если два потока выполняют:

```java
count++;
```

одновременно:

* оба читают одно и то же значение;
* оба увеличивают его;
* один результат теряется.

Это называется **race condition (гонка потоков)**.

---

# 2. Пакет `java.util.concurrent.atomic`

## Назначение

Пакет содержит классы для атомарных операций **без использования блокировок (lock-free)**.

---

## Основные атомарные классы

| Класс | Назначение |
|------|------------|
| `AtomicInteger` | атомарный `int` |
| `AtomicLong` | атомарный `long` |
| `AtomicBoolean` | атомарный `boolean` |
| `AtomicReference<T>` | атомарная ссылка |

---

## Принцип работы

Атомарные классы используют механизм:

> CAS (Compare-And-Swap)

---

# 3. Класс `AtomicInteger`

## Назначение

`AtomicInteger` обеспечивает потокобезопасные операции над `int` без `synchronized`.

---

## Создание

```java
AtomicInteger counter = new AtomicInteger();
```

```java
AtomicInteger counter = new AtomicInteger(10);
```

---

## Основные методы

| Метод | Описание |
|------|----------|
| `get()` | получить значение |
| `set(int)` | установить значение |
| `incrementAndGet()` | ++ и вернуть новое значение |
| `getAndIncrement()` | вернуть значение и ++ |
| `decrementAndGet()` | -- и вернуть новое |
| `addAndGet(int)` | прибавить и вернуть |
| `getAndAdd(int)` | вернуть и прибавить |

---

## Пример использования

```java
AtomicInteger counter = new AtomicInteger();

counter.incrementAndGet();
counter.addAndGet(5);
```

---

# 4. CAS (Compare-And-Swap)

## Основная идея

CAS обновляет значение только если оно совпадает с ожидаемым.

---

## Метод `compareAndSet`

```java
counter.compareAndSet(0, 10);
```

### Логика:

```text
если текущее значение == 0
    заменить на 10
иначе
    ничего не делать
```

---

## Пример

```java
AtomicInteger counter = new AtomicInteger(0);

boolean updated = counter.compareAndSet(0, 1);
```

---

## Почему CAS важен

CAS — основа **lock-free алгоритмов**:

* нет блокировок;
* высокая производительность;
* нет deadlock.

---

## Недостатки CAS

* возможен ABA-problem;
* активное повторение (spin);
* не подходит для сложной логики.

---

# 5. Сравнение с synchronized

| Критерий | synchronized | AtomicInteger |
|----------|--------------|---------------|
| блокировки | да | нет |
| производительность | ниже | выше |
| сложная логика | да | нет |
| атомарность | да | да |

---

# 6. Когда использовать AtomicInteger

✔ счетчики  
✔ статистика  
✔ флаги  
✔ простые операции изменения состояния  

---

# 7. Когда НЕ использовать

❌ сложные транзакции  
❌ несколько зависимых операций  
❌ работа с несколькими переменными одновременно  

---

# 8. Пример потокобезопасного счетчика

```java
AtomicInteger counter = new AtomicInteger();

Runnable task = () -> {
    for (int i = 0; i < 1000; i++) {
        counter.incrementAndGet();
    }
};
```

---

# Шпаргалка

* Атомарная операция — выполняется целиком
* `count++` — не атомарна
* `AtomicInteger` — потокобезопасный счетчик без lock
* CAS — Compare-And-Swap основа атомиков
* `compareAndSet` — ключевой метод
* подходит только для простых операций
