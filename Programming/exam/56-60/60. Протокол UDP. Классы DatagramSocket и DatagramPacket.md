# 60. Протокол UDP. Классы DatagramSocket и DatagramPacket

`DatagramSocket` и `DatagramPacket` (пакет `java.net`) — классический **блокирующий** API Java для UDP. `DatagramSocket` отправляет и принимает датаграммы, `DatagramPacket` — контейнер одной датаграммы (данные + адресация).

## `DatagramSocket`

* `new DatagramSocket()` — сокет на эфемерном порту (типично для **клиента-отправителя**);
* `new DatagramSocket(port)` — привязка к конкретному порту (типично для **получателя/сервера**);
* `send(packet)` — отправить датаграмму (обычно не блокируется);
* `receive(packet)` — **блокируется** до прихода датаграммы;
* `setSoTimeout(ms)` — таймаут `receive()` → `SocketTimeoutException` (иначе можно ждать вечно — пакет мог потеряться);
* `connect(addr, port)` — зафиксировать единственного пира: чужие пакеты отбрасываются (соединение при этом не устанавливается).

## `DatagramPacket`

Объединяет **буфер данных**, его длину и **адресацию**:

```java
// для отправки: данные + куда
new DatagramPacket(byte[] buf, int length, InetAddress addr, int port);

// для приёма: только буфер
new DatagramPacket(byte[] buf, int length);
```

После `receive()` пакет заполняется: `getData()` + `getLength()` — данные и **фактический** размер (читать ровно `getLength()` байт, а не весь массив!), `getAddress()` + `getPort()` — отправитель (можно ответить).

## Пример: отправитель и получатель

```java
// Получатель
try (DatagramSocket socket = new DatagramSocket(9999)) {
    byte[] buf = new byte[1500];
    DatagramPacket packet = new DatagramPacket(buf, buf.length);
    socket.receive(packet);                       // блокируется
    String msg = new String(packet.getData(), 0,
            packet.getLength(), StandardCharsets.UTF_8);
    // ответ отправителю
    byte[] reply = "ok".getBytes(StandardCharsets.UTF_8);
    socket.send(new DatagramPacket(reply, reply.length,
            packet.getAddress(), packet.getPort()));
}

// Отправитель
try (DatagramSocket socket = new DatagramSocket()) {
    byte[] data = "hello".getBytes(StandardCharsets.UTF_8);
    socket.send(new DatagramPacket(data, data.length,
            InetAddress.getByName("localhost"), 9999));
}
```

## Тонкости

* **Усечение:** если принятая датаграмма больше буфера пакета, лишние байты **молча отбрасываются** — буфер берут с запасом (≥ максимального сообщения протокола);
* **Переиспользование пакета:** после приёма `length` уменьшается до фактического размера — перед следующим `receive()` нужно `packet.setLength(buf.length)`, иначе следующая датаграмма усечётся;
* размер: датаграмму держат меньше MTU (~1472 байта) во избежание IP-фрагментации; теоретический максимум ~65 507 байт;
* **broadcast:** `socket.setBroadcast(true)` + адрес `255.255.255.255` — рассылка всем в локальной сети;
* **multicast:** подкласс `MulticastSocket` — `joinGroup()` на групповой адрес (224.0.0.0–239.255.255.255);
* надёжность (подтверждения, повторы, нумерация) — **забота приложения**, UDP её не даёт.

## Сравнение с `DatagramChannel`

Это блокирующий IO-API: нет неблокирующего режима и `Selector`. Для мультиплексирования множества клиентов одним потоком используется NIO-аналог — `DatagramChannel`.

### Шпаргалка

* `DatagramSocket(port)` — приём; `receive()` блокируется → ставить `setSoTimeout()`.
* `DatagramPacket` = буфер + длина + адрес/порт; после приёма читать **`getLength()`** байт, отвечать на `getAddress()/getPort()`.
* Малый буфер → датаграмма молча **усекается**; перед повторным `receive()` — `setLength(buf.length)`.
* Датаграммы < MTU (~1472 байта); broadcast — `setBroadcast(true)` + `255.255.255.255`; multicast — `MulticastSocket`.
* Гарантий доставки/порядка нет — надёжность реализует приложение; NIO-аналог — `DatagramChannel`.
