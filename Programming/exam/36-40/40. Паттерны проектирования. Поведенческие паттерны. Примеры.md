# 40. Паттерны проектирования. Поведенческие паттерны. Примеры

**Поведенческие паттерны** (behavioral patterns) описывают взаимодействие объектов, распределение обязанностей и алгоритмы взаимодействия между классами. Они фокусируются на **как**, а не на **что**.

---

## 📋 Классификация поведенческих паттернов (GoF)

| Паттерн | Суть | Ключевая идея |
|---------|------|---------------|
| **Strategy** | Семейство взаимозаменяемых алгоритмов | Инкапсуляция алгоритма |
| **Observer** | Уведомление зависимых объектов об изменениях | Событийная модель |
| **Command** | Инкапсуляция запроса как объекта | Отложенное выполнение |
| **Iterator** | Последовательный обход коллекции | Единый интерфейс обхода |
| **State** | Изменение поведения при смене состояния | Объект как конечный автомат |
| **Template Method** | Скелет алгоритма с переопределяемыми шагами | Инверсия управления |
| **Chain of Responsibility** | Передача запроса по цепочке обработчиков | Децентрализация обработки |

---

## 1. Strategy (Стратегия)

Определяет семейство алгоритмов, инкапсулирует каждый из них и делает их взаимозаменяемыми. Алгоритм можно менять независимо от клиентов.

### Пример: система оплаты

```java
// Strategy interface
interface PaymentStrategy {
    void pay(int amount);
}

// Concrete strategies
class CreditCardPayment implements PaymentStrategy {
    private String cardNumber;
    
    CreditCardPayment(String cardNumber) { this.cardNumber = cardNumber; }
    
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " via credit card " + cardNumber);
    }
}

class PayPalPayment implements PaymentStrategy {
    private String email;
    
    PayPalPayment(String email) { this.email = email; }
    
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " via PayPal (" + email + ")");
    }
}

class CryptoPayment implements PaymentStrategy {
    private String walletAddress;
    
    CryptoPayment(String walletAddress) { this.walletAddress = walletAddress; }
    
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " in BTC to " + walletAddress);
    }
}

// Context
class ShoppingCart {
    private PaymentStrategy paymentStrategy;
    
    void setPaymentStrategy(PaymentStrategy strategy) {
        this.paymentStrategy = strategy;
    }
    
    void checkout(int amount) {
        if (paymentStrategy == null) {
            throw new IllegalStateException("No payment strategy set");
        }
        paymentStrategy.pay(amount);
    }
}

// Usage
ShoppingCart cart = new ShoppingCart();
cart.setPaymentStrategy(new CreditCardPayment("1234-5678"));
cart.checkout(100);
cart.setPaymentStrategy(new PayPalPayment("user@example.com"));
cart.checkout(50);
```

### Стратегия с лямбда-выражениями (Java 8+)

```java
// Вместо иерархии классов можно использовать функциональные интерфейсы
enum PaymentMethod {
    CREDIT_CARD(card -> "Paid via card: " + card),
    PAYPAL(email -> "Paid via PayPal: " + email);
    
    private final Function<String, String> processor;
    
    PaymentMethod(Function<String, String> processor) {
        this.processor = processor;
    }
    
    String process(String identifier) {
        return processor.apply(identifier);
    }
}
```

### Complexity & подкапот

| Операция | Complexity |
|----------|------------|
| Выбор стратегии | O(1) |
| Выполнение алгоритма | O(1) / O(n) (зависит от конкретной стратегии) |
| Добавление новой стратегии | O(1) (не требует изменения контекста) |

## 2. Observer (Наблюдатель)

Определяет зависимость "один-ко-многим", при которой при изменении состояния одного объекта все зависимые уведомляются автоматически.

### Пример: система подписок на новости

```java
// Observer interface
interface Subscriber {
    void update(String news);
}

// Concrete observers
class EmailSubscriber implements Subscriber {
    private String email;
    
    EmailSubscriber(String email) { this.email = email; }
    
    @Override
    public void update(String news) {
        System.out.println("Email to " + email + ": " + news);
    }
}

class SmsSubscriber implements Subscriber {
    private String phone;
    
    SmsSubscriber(String phone) { this.phone = phone; }
    
    @Override
    public void update(String news) {
        System.out.println("SMS to " + phone + ": " + news);
    }
}

// Subject
class NewsPublisher {
    private List<Subscriber> subscribers = new ArrayList<>();
    
    void subscribe(Subscriber subscriber) {
        subscribers.add(subscriber);
    }
    
    void unsubscribe(Subscriber subscriber) {
        subscribers.remove(subscriber);
    }
    
    void publish(String news) {
        // Важно: копируем список, чтобы избежать ConcurrentModificationException
        new ArrayList<>(subscribers).forEach(s -> s.update(news));
    }
}

// Usage
NewsPublisher publisher = new NewsPublisher();
Subscriber alice = new EmailSubscriber("alice@example.com");
Subscriber bob = new SmsSubscriber("+123456789");

publisher.subscribe(alice);
publisher.subscribe(bob);
publisher.publish("Java 21 released!"); // Оба получат уведомление

publisher.unsubscribe(alice);
publisher.publish("New LTS version");    // Только bob
```

### Стандартная реализация в JDK (устаревшая)

```java
// Observable (устарел в Java 9) и Observer (устарел)
// Не рекомендуется к использованию, но знать стоит

```

### Современная альтернатива: PropertyChangeSupport

```java
import java.beans.PropertyChangeListener;
import java.beans.PropertyChangeSupport;

class User {
    private String name;
    private PropertyChangeSupport support = new PropertyChangeSupport(this);
    
    void addPropertyChangeListener(PropertyChangeListener listener) {
        support.addPropertyChangeListener(listener);

        String old = this.name;
        this.name = name;
        support.firePropertyChange("name", old, name);
    }
}
```

### Complexity

| Операция | Complexity |
|----------|------------|
| Добавление наблюдателя | O(1) |
| Удаление наблюдателя | O(n) (поиск) |
| Уведомление всех | O(k) где k — число наблюдателей |

## 3. Command (Команда)

Превращает запрос в объект, позволяя передавать его как аргумент, ставить в очередь, логировать и поддерживать отмену операций.

### Пример: текстовый редактор с undo/redo

```java
// Command interface
interface Command {
    void execute();
    void undo();
}

// Receiver
class TextBuffer {
    private StringBuilder content = new StringBuilder();
    
    void append(String text) { content.append(text); }
    void deleteLast(int length) { 
        content.delete(content.length() - length, content.length());
    }
    String getContent() { return content.toString(); }
}

// Concrete command
class AppendCommand implements Command {
    private TextBuffer buffer;
    private String text;
    
    AppendCommand(TextBuffer buffer, String text) {
        this.buffer = buffer;
        this.text = text;
    }
    
    @Override
    public void execute() { buffer.append(text); }
    
    @Override
    public void undo() { buffer.deleteLast(text.length()); }
}

// Invoker
class TextEditor {
    private Deque<Command> history = new ArrayDeque<>();
    private TextBuffer buffer = new TextBuffer();
    
    void executeCommand(Command command) {
        command.execute();
        history.push(command);
    }
    
    void undo() {
        if (!history.isEmpty()) {
            history.pop().undo();
        }
    }
    
    void showContent() {
        System.out.println(buffer.getContent());
    }
    
    void append(String text) {
        executeCommand(new AppendCommand(buffer, text));
    }
}

// Usage
TextEditor editor = new TextEditor();
editor.append("Hello ");
editor.append("World");
editor.showContent();      // "Hello World"
editor.undo();
editor.showContent();      // "Hello "
editor.undo();
editor.showContent();      // ""
```

### Command с лямбда-выражениями

```java
// Функциональный интерфейс Command
@FunctionalInterface
interface LambdaCommand {
    void execute();
}

// Можно использовать готовые Runnable или Supplier
Runnable printHello = () -> System.out.println("Hello");
printHello.run();
```

### Complexity

| Операция | Complexity |
|----------|------------|
| Выполнение команды | O(1) (обычно) |
| Undo/Redo | O(1) |
| Хранение истории | O(n) где n — число команд |

## 4. Iterator (Итератор)

Предоставляет способ последовательного доступа к элементам коллекции без раскрытия её внутреннего представления.

### Реализация своего итератора

```java
// Aggregate
interface CustomCollection<T> {
    CustomIterator<T> iterator();
}

// Iterator
interface CustomIterator<T> {
    boolean hasNext();
    T next();
}

// Concrete aggregate
class MyList<T> implements CustomCollection<T> {
    private Object[] elements;
    private int size = 0;
    
    MyList(int capacity) { elements = new Object[capacity]; }
    
    void add(T item) {
        if (size == elements.length) resize();
        elements[size++] = item;
    }
    
    private void resize() { elements = Arrays.copyOf(elements, elements.length * 2); }
    
    @Override
    public CustomIterator<T> iterator() {
        return new MyIterator();
    }
    
    // Concrete iterator
    private class MyIterator implements CustomIterator<T> {
        private int cursor = 0;
        
        @Override
        public boolean hasNext() {
            return cursor < size;
        }
        
        @Override
        @SuppressWarnings("unchecked")
        public T next() {
            if (!hasNext()) throw new NoSuchElementException();
            return (T) elements[cursor++];
        }
    }
}
```

### Итератор в Java Collections Framework

```java
// Стандартный Iterator
List<String> list = Arrays.asList("A", "B", "C");
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    System.out.println(it.next());
}

// ListIterator (двунаправленный)
ListIterator<String> listIt = list.listIterator();
listIt.next();
listIt.add("D");      // Можно модифицировать

// Iterable.forEach
list.forEach(System.out::println);

// Spliterator (для параллельной обработки)
Spliterator<String> spliterator = list.spliterator();
spliterator.tryAdvance(System.out::println);
```

### Complexity

| Операция | Complexity |
|----------|------------|
| `hasNext()` | O(1) |
| `next()` | O(1) (для большинства коллекций) |
| `remove()` | O(1) для некоторых (но не всех) |

## 5. State (Состояние)

Позволяет объекту изменять своё поведение при изменении внутреннего состояния, создавая впечатление, что объект изменил свой класс.

### Пример: автомат по продаже напитков

```java
// State interface
interface VendingMachineState {
    void insertCoin();
    void selectProduct();
    void dispense();
}

// Concrete states
class NoCoinState implements VendingMachineState {
    private VendingMachine machine;
    
    NoCoinState(VendingMachine machine) { this.machine = machine; }
    
    @Override
    public void insertCoin() {
        System.out.println("Coin inserted");
        machine.setState(machine.getHasCoinState());
    }
    
    @Override
    public void selectProduct() {
        System.out.println("Insert coin first");
    }
    
    @Override
    public void dispense() {
        System.out.println("Insert coin first");
    }
}

class HasCoinState implements VendingMachineState {
    private VendingMachine machine;
    
    HasCoinState(VendingMachine machine) { this.machine = machine; }
    
    @Override
    public void insertCoin() {
        System.out.println("Already has coin");
    }
    
    @Override
    public void selectProduct() {
        System.out.println("Product selected");
        machine.setState(machine.getDispensingState());
    }
    
    @Override
    public void dispense() {
        System.out.println("Select product first");
    }
}

class DispensingState implements VendingMachineState {
    private VendingMachine machine;
    
    DispensingState(VendingMachine machine) { this.machine = machine; }
    
    @Override
    public void insertCoin() {
        System.out.println("Wait, dispensing...");
    }
    
    @Override
    public void selectProduct() {
        System.out.println("Already dispensing");
    }
    
    @Override
    public void dispense() {
        System.out.println("Dispensing product...");
        machine.setState(machine.getNoCoinState());
    }
}

// Context
class VendingMachine {
    private VendingMachineState noCoinState;
    private VendingMachineState hasCoinState;
    private VendingMachineState dispensingState;
    private VendingMachineState currentState;
    
    VendingMachine() {
        noCoinState = new NoCoinState(this);
        hasCoinState = new HasCoinState(this);
        dispensingState = new DispensingState(this);
        currentState = noCoinState;
    }
    
    void setState(VendingMachineState state) { this.currentState = state; }
    VendingMachineState getNoCoinState() { return noCoinState; }
    VendingMachineState getHasCoinState() { return hasCoinState; }
    VendingMachineState getDispensingState() { return dispensingState; }
    
    void insertCoin() { currentState.insertCoin(); }
    void selectProduct() { currentState.selectProduct(); }
    void dispense() { currentState.dispense(); }
}

// Usage
VendingMachine machine = new VendingMachine();
machine.selectProduct();  // Insert coin first
machine.insertCoin();     // Coin inserted
machine.selectProduct();  // Product selected
machine.dispense();       // Dispensing product...
```

### Complexity

| Операция | Complexity |
|----------|------------|
| Смена состояния | O(1) |
| Выполнение действия | O(1) (делегация состоянию) |
| Количество состояний | O(k) памяти |

## 6. Template Method (Шаблонный метод)

Определяет скелет алгоритма в методе, откладывая реализацию некоторых шагов подклассам.

### Пример: приготовление напитков

```java
abstract class Beverage {
    // Template method (final чтобы нельзя было переопределить)
    public final void prepare() {
        boilWater();
        brew();
        pourInCup();
        addCondiments();
    }
    
    void boilWater() { System.out.println("Boiling water"); }
    void pourInCup() { System.out.println("Pouring into cup"); }
    
    abstract void brew();      // Шаг, зависящий от подкласса
    abstract void addCondiments(); // Шаг, зависящий от подкласса
    
    // Hook (переопределяется опционально)
    protected boolean wantsCondiments() { return true; }
}

class Tea extends Beverage {
    @Override
    void brew() { System.out.println("Steeping tea bag"); }
    
    @Override
    void addCondiments() { System.out.println("Adding lemon"); }
}

class Coffee extends Beverage {
    @Override
    void brew() { System.out.println("Brewing coffee grounds"); }
    
    @Override
    void addCondiments() { System.out.println("Adding sugar and milk"); }
}

// Usage
Beverage tea = new Tea();
tea.prepare();
// Boiling water -> Steeping tea bag -> Pouring into cup -> Adding lemon
```

### Template Method с callback (современный подход)

```java
class DataProcessor {
    // "Template method" через Consumer/BiConsumer
    public void processFile(String path, 
                            Function<String, String> lineProcessor,
                            Predicate<String> lineFilter) {
        try (Stream<String> lines = Files.lines(Path.of(path))) {
            lines.filter(lineFilter)
                 .map(lineProcessor)
                 .forEach(System.out::println);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

### Complexity

| Операция | Complexity |
|----------|------------|
| Выполнение шаблонного метода | Сумма сложностей шагов |

## 7. Chain of Responsibility (Цепочка обязанностей)

Позволяет передавать запрос последовательно по цепочке обработчиков, пока один из них не обработает запрос.

### Пример: система авторизации запросов

```java
// Request
record Request(String username, int userId, String ip, String action) {}

// Handler interface
interface AuthHandler {
    void setNext(AuthHandler next);
    void handle(Request request);
}

// Base handler (Chain of Responsibility classic)
abstract class BaseHandler implements AuthHandler {
    private AuthHandler next;
    
    @Override
    public void setNext(AuthHandler next) { this.next = next; }
    
    protected void handleNext(Request request) {
        if (next != null) {
            next.handle(request);
        }
    }
}

// Concrete handlers
class RateLimitHandler extends BaseHandler {
    private static final int MAX_REQUESTS = 100;
    private int requestCount = 0;
    
    @Override
    public void handle(Request request) {
        if (requestCount++ > MAX_REQUESTS) {
            System.out.println("Rate limit exceeded for " + request.ip());
            return;
        }
        System.out.println("Rate limit passed");
        handleNext(request);
    }
}

class AuthenticationHandler extends BaseHandler {
    @Override
    public void handle(Request request) {
        if (request.username() == null || request.userId() <= 0) {
            System.out.println("Authentication failed");
            return;
        }
        System.out.println("Authenticated: " + request.username());
        handleNext(request);
    }
}

class AuthorizationHandler extends BaseHandler {
    @Override
    public void handle(Request request) {
        if (request.action().equals("DELETE") && request.userId() != 1) {
            System.out.println("Not authorized to delete");
            return;
        }
        System.out.println("Authorized for: " + request.action());
        handleNext(request);
    }
}

class AuditLogHandler extends BaseHandler {
    @Override
    public void handle(Request request) {
        System.out.println("Logged: " + request.username() + " -> " + request.action());
        handleNext(request);
    }
}

// Builder для цепочки
class AuthChain {
    private List<AuthHandler> handlers = new ArrayList<>();
    
    AuthChain add(AuthHandler handler) {
        handlers.add(handler);
        return this;
    }
    
    AuthHandler build() {
        for (int i = 0; i < handlers.size() - 1; i++) {
            handlers.get(i).setNext(handlers.get(i + 1));
        }
        return handlers.get(0);
    }
}

// Usage
AuthHandler chain = new AuthChain()
    .add(new RateLimitHandler())
    .add(new AuthenticationHandler())
    .add(new AuthorizationHandler())
    .add(new AuditLogHandler())
    .build();

Request req = new Request("john", 42, "192.168.1.1", "READ");
chain.handle(req);
```

### Chain с лямбда-выражениями (modern approach)

```java
// Функциональная цепочка
Predicate<Request> rateLimit = r -> true;
Predicate<Request> auth = r -> r.username() != null;
Predicate<Request> authz = r -> !r.action().equals("DELETE") || r.userId() == 1;

Predicate<Request> chain = rateLimit.and(auth).and(authz);

if (chain.test(req)) {
    System.out.println("Request allowed: " + req.action());
}
```

### Complexity

| Операция | Complexity |
|----------|------------|
| Проход по цепочке | O(n) где n — число обработчиков |
| Добавление обработчика | O(1) |

## 📊 Сравнительная таблица поведенческих паттернов

| Паттерн | Основная проблема | Решение | Гибкость | Сложность |
|---------|-------------------|---------|----------|-----------|
| **Strategy** | Семейство алгоритмов | Композиция стратегии | Высокая | Низкая |
| **Observer** | Уведомление зависимых | Подписка/публикация | Высокая | Средняя |
| **Command** | Отложенное выполнение/undo | Инкапсуляция запроса | Высокая | Средняя |
| **Iterator** | Обход коллекции | Единый интерфейс | Средняя | Низкая |
| **State** | Поведение от состояния | Конечный автомат | Высокая | Средняя |
| **Template Method** | Инварианты алгоритма | Наследование | Низкая | Низкая |
| **Chain of Responsibility** | Обработка запроса | Децентрализация | Высокая | Средняя |

## 🧠 Реальный пример — комбинация Command + Observer + State

```java
// Умный дом: команды, уведомления и состояния
interface DeviceCommand extends Command {}
interface DeviceState { void handle(); }

class Light {
    private boolean on;
    private DeviceState state;
    private List<Runnable> listeners = new ArrayList<>();
    
    void addListener(Runnable listener) { listeners.add(listener); }
    
    void setState(DeviceState state) { 
        this.state = state;
        listeners.forEach(Runnable::run);
    }
    
    void turnOn() { 
        on = true;
        setState(() -> System.out.println("Light is ON"));
    }
    
    void turnOff() { 
        on = false;
        setState(() -> System.out.println("Light is OFF"));
    }
}

class TurnOnCommand implements DeviceCommand {
    private Light light;
    TurnOnCommand(Light light) { this.light = light; }
    public void execute() { light.turnOn(); }
    public void undo() { light.turnOff(); }
}

// Использование всех паттернов
Light light = new Light();
light.addListener(() -> System.out.println("State changed notification"));

Command turnOn = new TurnOnCommand(light);
turnOn.execute(); // Light is ON -> notification
((Command)turnOn).undo(); // Light is OFF -> notification
```

## ⚠️ Антипаттерны и частые ошибки

### Strategy
- ❌ Создавать отдельную стратегию для одноразового использования — используйте лямбды
- ❌ Стратегия, которая зависит от контекста (нарушение инкапсуляции)

### Observer
- ❌ Утечки памяти из-за неотписанных наблюдателей (используйте `WeakReference` или отписку)
- ❌ Уведомления во время итерации → `ConcurrentModificationException`

### Command
- ❌ Тяжёлые команды (делайте `execute()` быстрым или асинхронным)
- ❌ Неверный `undo()` для stateful команд

### Iterator
- ❌ Модификация коллекции во время итерации (исключение или undefined behavior)
- ✅ Используйте `CopyOnWriteArrayList` или `ConcurrentHashMap` для параллельных итераций

### Шпаргалка

* **Strategy** — меняем алгоритм через композицию (например, `Comparator`)
* **Observer** — подписка на события (`PropertyChangeSupport`)
* **Command** — действие как объект (для очередей, undo, логов)
* **Iterator** — единый способ обхода коллекций (встроен в JDK)
* **State** — объект ведёт себя по-разному в разных состояниях
* **Template Method** — скелет алгоритма в абстрактном классе
* **Chain of Responsibility** — цепочка обработчиков (например, фильтры в вебе)
* **Все поведенческие паттерны уменьшают связанность** (loose coupling) между объектами
* **Выбор зависит от того, что именно меняется:** алгоритм, состояние, способ обхода, обработчик запроса
