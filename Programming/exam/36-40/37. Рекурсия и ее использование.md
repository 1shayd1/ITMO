# 37. Рекурсия и ее использование

**Рекурсия** — это техника программирования, при которой метод вызывает сам себя для решения задачи путем сведения её к более простым подзадачам того же типа.

---

## 🧱 Два обязательных элемента

1. **Базовый случай** (termination condition) — условие, при котором рекурсия останавливается и возвращает конкретное значение без новых вызовов.
2. **Рекурсивный шаг** (recursive case) — вызов того же метода с изменёнными параметрами, приближающими к базовому случаю.

```java
int factorial(int n) {
    if (n <= 1)          // базовый случай
        return 1;
    return n * factorial(n - 1); // рекурсивный шаг
}
```

**Как это работает под капотом:**

Каждый рекурсивный вызов создаёт новый **кадр стека** (stack frame) в памяти JVM:
- Содержит локальные переменные, параметры, адрес возврата
- Кадры складываются в `call stack` (последним пришёл — первым ушёл)
- При возврате управления кадры последовательно выталкиваются (pop)

```
factorial(4) → возвращает 4 * factorial(3)
                    → возвращает 3 * factorial(2)
                        → возвращает 2 * factorial(1)
                            → возвращает 1 (базовый случай)
```

## 📊 Виды рекурсии

| Вид | Описание | Пример |
|-----|----------|--------|
| **Прямая** | Метод вызывает сам себя | `factorial(n - 1)` |
| **Косвенная** | A вызывает B, B вызывает A | Редко используется |
| **Хвостовая** | Рекурсивный вызов — последняя операция | `tailFactorial(n, acc)` |
| **Линейная** | Один рекурсивный вызов на шаг | `factorial` |
| **Древовидная** | Несколько рекурсивных вызовов на шаг | обход бинарного дерева |

### Хвостовая рекурсия (Tail Recursion)

```java
// Обычная рекурсия — не хвостовая (нужно умножить после возврата)
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1); // ❌ умножение ПОСЛЕ рекурсии
}

// Хвостовая рекурсия — аккумулятор
int tailFactorial(int n, int accumulator) {
    if (n <= 1) return accumulator;
    return tailFactorial(n - 1, n * accumulator); // ✅ рекурсия последняя
}
```

**Важно:** Java **не оптимизирует хвостовую рекурсию** (в отличие от Scala, Kotlin с `tailrec`). Каждый вызов всё равно создаёт кадр стека.

## 🗺️ Где рекурсия полезна (и где неизбежна)

### 1. Обход деревьев (наиболее естественное применение)

```java
class TreeNode {
    int value;
    TreeNode left;
    TreeNode right;
    
    // рекурсивный обход в глубину (DFS) — естественнее итерации
    void inorder(TreeNode node) {
        if (node == null) return;      // базовый случай
        inorder(node.left);            // левое поддерево
        System.out.print(node.value);  // корень
        inorder(node.right);           // правое поддерево
    }
}
```

### 2. Разбор вложенных структур (JSON, XML, HTML)

```java
// Рекурсивный поиск в глубоковложенном JSON
Object findValue(Map<String, Object> json, String targetKey) {
    for (Map.Entry<String, Object> entry : json.entrySet()) {
        if (entry.getKey().equals(targetKey))
            return entry.getValue();
        if (entry.getValue() instanceof Map)
            return findValue((Map) entry.getValue(), targetKey);
    }
    return null;
}
```

### 3. Задачи "разделяй и властвуй" (Divide & Conquer)

```java
// Быстрая сортировка (QuickSort)
void quickSort(int[] arr, int low, int high) {
    if (low < high) {                     // базовый случай: 0 или 1 элемент
        int pivot = partition(arr, low, high);
        quickSort(arr, low, pivot - 1);   // левая часть
        quickSort(arr, pivot + 1, high);  // правая часть
    }
}
```

### 4. Задачи с backtracking (поиск с возвратом)

```java
// Генерация всех перестановок строки
void permute(String str, String prefix, List<String> result) {
    if (str.isEmpty()) {
        result.add(prefix);               // базовый случай: нет символов
        return;
    }
    for (int i = 0; i < str.length(); i++) {
        String newStr = str.substring(0, i) + str.substring(i + 1);
        permute(newStr, prefix + str.charAt(i), result);
    }
}
```

### 5. Рекурсивные алгоритмы на графах (DFS, топологическая сортировка)

```java
void dfs(Graph graph, int vertex, boolean[] visited) {
    visited[vertex] = true;
    for (int neighbor : graph.getNeighbors(vertex)) {
        if (!visited[neighbor])
            dfs(graph, neighbor, visited);
    }
}
```

## 📈 Memory Complexity (под капотом)

| Вид рекурсии | Глубина (n) | Память на стек | Общая Complexity |
|--------------|-------------|----------------|------------------|
| Линейная (факториал) | n | O(n) кадров | O(n) |
| Древовидная (Фибоначчи) | n (но вызовов ~2ⁿ) | O(n) (глубина) | O(n) на стек + O(2ⁿ) времени |
| Хвостовая в Java | n | O(n) (нет оптимизации) | O(n) |
| Итеративная замена | 1 | O(1) | O(1) |

**Размер кадра стека:** ~1-2KB на вызов.  
**Лимит по умолчанию:** ~500-1000 вызовов (зависит от JVM и ОС).

```java
// Проверка лимита глубины
public static void boom(int n) {
    System.out.println(n);
    boom(n + 1);
}
// boom(1) → StackOverflowError на ~10000 (Xss может увеличить)
```

## ⚠️ Риски и как их избежать

### 1. StackOverflowError

**Причина:** глубина рекурсии превышает размер стека потока (задаётся `-Xss`).

**Решение:**
- Увеличить стек: `java -Xss4m MyClass` (не всегда выход)
- Переписать на итерацию
- Использовать трамплин (trampoline) или собственную имитацию стека

```java
// Итеративная замена рекурсии (свой стек в куче)
void iterativeDFS(TreeNode root) {
    Deque<TreeNode> stack = new ArrayDeque<>();
    stack.push(root);
    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        if (node.right != null) stack.push(node.right);
        if (node.left != null) stack.push(node.left);
    }
}
```

### 2. Экспоненциальное время (дублирование вычислений)

```java
// ❌ Плохая рекурсия — O(2ⁿ)
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2); // fib(5) вызывает fib(4) и fib(3) и т.д.
}
// fib(40) → ~300 млн вызовов

// ✅ Мемоизация (кэширование)
Map<Integer, Integer> cache = new HashMap<>();
int fibMemo(int n) {
    if (n <= 1) return n;
    if (cache.containsKey(n)) return cache.get(n);
    int result = fibMemo(n - 1) + fibMemo(n - 2);
    cache.put(n, result);
    return result;
}
// O(n) времени и памяти
```

### 3. Отсутствие базового случая

```java
// ❌ Бесконечная рекурсия
int badFact(int n) {
    return n * badFact(n - 1); // нет if (n <= 1)
}
```

## 📊 Сравнение: рекурсия vs итерация

| Характеристика | Рекурсия | Итерация |
|----------------|----------|----------|
| Читаемость (для деревьев/графов) | ✅ Высокая | ❌ Низкая (нужен явный стек) |
| Читаемость (для линейных задач) | ❌ Избыточна | ✅ Простая |
| Риск переполнения стека | ✅ (глубина > 1000) | ❌ (петля в куче) |
| Memory overhead | O(глубина) кадров | O(1) или свой стек |
| Производительность | Медленнее (создание кадров) | Быстрее |
| Отладка | Сложнее (много вложенных вызовов) | Легче |

**Когда использовать рекурсию:**
- Деревья, графы (DFS естественно рекурсивен)
- Разделяй-и-властвуй (QuickSort, MergeSort)
- Задачи с backtracking (лабиринты, перестановки)
- Код заметно короче и понятнее итеративного аналога

**Когда использовать итерацию:**
- Простые линейные задачи (сумма массива, факториал)
- Глубина рекурсии > 1000
- Высокие требования к производительности
- Ограниченный стек (встраиваемые системы, Android)

## 🧠 Полный пример: обход файловой системы

```java
import java.io.File;

public class DirectoryWalker {
    
    // Рекурсивный обход — идеальное применение
    public static void printTree(File dir, String indent) {
        if (!dir.isDirectory()) {
            System.out.println(indent + "📄 " + dir.getName());
            return;
        }
        
        System.out.println(indent + "📁 " + dir.getName());
        File[] files = dir.listFiles();
        if (files == null) return; // базовый случай: нет доступа
        
        for (File file : files) {
            printTree(file, indent + "  "); // рекурсивный шаг
        }
    }
    
    // Поиск файла с ограничением глубины (безопасная рекурсия)
    public static File findFile(File dir, String target, int maxDepth, int currentDepth) {
        if (currentDepth > maxDepth) return null; // принудительная защита
        
        for (File file : dir.listFiles()) {
            if (file.getName().equals(target))
                return file;
            if (file.isDirectory()) {
                File found = findFile(file, target, maxDepth, currentDepth + 1);
                if (found != null) return found;
            }
        }
        return null;
    }
    
    public static void main(String[] args) {
        printTree(new File("/home/user/projects"), "");
        File found = findFile(new File("/"), "pom.xml", 10, 0);
        System.out.println(found != null ? "Found!" : "Not found (depth limit)");
    }
}
```

## 🔗 Связи с другими темами Java

- `StackOverflowError` — наследуется от `VirtualMachineError`
- `Call Stack` — JVM управляет памятью для вызовов
- `Divide and Conquer` — парадигма, используемая в `ForkJoinPool` (Java 7+)
- `RecursiveTask<T>` — для рекурсивных параллельных задач
- Комбинаторика и `backtracking` — часто на собеседованиях

### Рекурсия в Stream API (косвенная)

```java
// Рекурсивное разворачивание вложенных списков через flatMap
Stream<Object> flatten(List<?> nested) {
    return nested.stream()
        .flatMap(e -> e instanceof List ? flatten((List<?>) e) : Stream.of(e));
}
```

### Шпаргалка

* **Рекурсия = метод вызывает сам себя** + базовый случай + рекурсивный шаг.
* **Каждый вызов = новый кадр в стеке** → `StackOverflowError` при глубине ~10000.
* **Java не оптимизирует хвостовую рекурсию** — каждый вызов всё равно жрёт стек.
* **Идеальное применение:** деревья, графы, "разделяй-и-властвуй", backtracking.
* **Плохое применение:** факториал, сумма массива, Фибоначчи (лучше итерация).
* **Мемоизация** спасает от экспоненциального времени при рекурсии.
* **Всегда проверяй базовый случай** — иначе бесконечная рекурсия и крах.
