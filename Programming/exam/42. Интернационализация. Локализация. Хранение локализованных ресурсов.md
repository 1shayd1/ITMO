# 42. Интернационализация. Локализация. Хранение локализованных ресурсов

**Интернационализация (i18n)** — процесс проектирования приложения для работы с разными языками и региональными форматами без изменений в коде.  
**Локализация (l10n)** — адаптация интернационализированного приложения под конкретный язык, регион или культуру.

> `i18n` = 18 букв между `i` и `n` в слове "internationalization"  
> `l10n` = 10 букв между `l` и `n` в слове "localization"

---

## 🌍 Что подлежит локализации

| Категория | Примеры |
|-----------|---------|
| **Текст** | Надписи, меню, сообщения об ошибках |
| **Дата и время** | `dd.MM.yyyy` vs `MM/dd/yyyy` vs `yyyy-MM-dd` |
| **Числа** | Разделитель тысяч: `1,234.56` vs `1.234,56` |
| **Валюта** | Символ и позиция: `$100` vs `100€` vs `100 ₽` |
| **Единицы измерения** | Мили / километры, фунты / килограммы |
| **Изображения и иконки** | Флаги, культурно-зависимые символы |
| **Сортировка** | Порядок букв в разных алфавитах |
| **Направление текста** | LTR (left-to-right) vs RTL (right-to-left, например, арабский) |

## 📦 Locale — основа локализации

`Locale` — идентификатор конкретной культурной среды. Состоит из:
- **Язык** (обязательный) — код по ISO 639: `ru`, `en`, `de`
- **Страна/регион** (опционально) — код по ISO 3166: `RU`, `US`, `DE`
- **Вариант** (редко) — `WIN`, `MAC`

```java
// Создание Locale
Locale loc1 = new Locale("en");              // только язык
Locale loc2 = new Locale("en", "US");        // язык + страна
Locale loc3 = new Locale("ru", "RU");
Locale loc4 = Locale.forLanguageTag("de-CH"); // немецкий (Швейцария)

// Константы
Locale locale = Locale.US;                   // en-US
Locale chinese = Locale.CHINA;               // zh-CN

// Системная локаль по умолчанию
Locale defaultLocale = Locale.getDefault();
```

### Порядок поиска ресурсов

`ResourceBundle` ищет файлы в порядке **специфичности**:

```
Файлы: messages.properties (базовый)
       messages_en.properties (английский)
       messages_en_US.properties (английский, США)

Запрос Locale.US:
1. messages_en_US.properties
2. messages_en.properties
3. messages.properties (fallback)
```

## 🗂️ ResourceBundle — хранение локализованных ресурсов

### Структура файлов свойств

**messages_en.properties**
```properties
# English
hello=Hello
goodbye=Goodbye
welcome.message=Welcome to our application!
error.notfound=File not found: {0}
```

**messages_ru.properties**
```properties
# Russian
hello=Привет
goodbye=До свидания
welcome.message=Добро пожаловать в наше приложение!
error.notfound=Файл не найден: {0}
```

**messages_de.properties**
```properties
# German
hello=Hallo
goodbye=Auf Wiedersehen
welcome.message=Willkommen in unserer Anwendung!
error.notfound=Datei nicht gefunden: {0}
```

### Загрузка ResourceBundle

```java
ResourceBundle bundle = ResourceBundle.getBundle("messages", Locale.GERMANY);
String hello = bundle.getString("hello");  // "Hallo"

// С параметрами (MessageFormat)
String pattern = bundle.getString("error.notfound");
String error = MessageFormat.format(pattern, "config.xml");
System.out.println(error); // Datei nicht gefunden: config.xml
```

### Способы загрузки

```java
// 1. С явной локалью
ResourceBundle bundle = ResourceBundle.getBundle("messages", Locale.FRENCH);

// 2. С системной локалью
ResourceBundle bundle = ResourceBundle.getBundle("messages");

// 3. С указанием ClassLoader
ResourceBundle bundle = ResourceBundle.getBundle(
    "messages", 
    Locale.JAPANESE, 
    Thread.currentThread().getContextClassLoader()
);

// 4. С контроллером для кастомного поиска
ResourceBundle.Control control = ResourceBundle.Control.getControl(ResourceBundle.Control.FORMAT_PROPERTIES);
ResourceBundle bundle = ResourceBundle.getBundle("messages", locale, control);
```

### Обработка отсутствующих ключей

```java
ResourceBundle bundle = ResourceBundle.getBundle("messages", Locale.ITALIAN);

try {
    String value = bundle.getString("nonexistent.key");
} catch (MissingResourceException e) {
    System.out.println("Key missing: " + e.getKey());
}

// Проверка наличия ключа
if (bundle.containsKey("hello")) {
    System.out.println(bundle.getString("hello"));
}
```

## 🔧 Форматирование данных под локаль

### NumberFormat (числа и валюта)

```java
Locale us = Locale.US;           // 1,234.56
Locale germany = Locale.GERMANY; // 1.234,56

double number = 1234.56;

// Общий числовой формат
NumberFormat nfUS = NumberFormat.getInstance(us);
NumberFormat nfDE = NumberFormat.getInstance(germany);
System.out.println(nfUS.format(number)); // 1,234.56
System.out.println(nfDE.format(number)); // 1.234,56

// Валюта
NumberFormat currencyUS = NumberFormat.getCurrencyInstance(us);
NumberFormat currencyDE = NumberFormat.getCurrencyInstance(germany);
System.out.println(currencyUS.format(number)); // $1,234.56
System.out.println(currencyDE.format(number)); // 1.234,56 €

// Парсинг числа из строки (с учётом локали)
Number parsed = nfDE.parse("1.234,56");
System.out.println(parsed.doubleValue()); // 1234.56
```

### DateFormat (даты и время)

```java
Locale us = Locale.US;
Locale germany = Locale.GERMANY;

Date now = new Date();

// Дата
DateFormat dfUS = DateFormat.getDateInstance(DateFormat.LONG, us);
DateFormat dfDE = DateFormat.getDateInstance(DateFormat.LONG, germany);
System.out.println(dfUS.format(now)); // January 1, 2025
System.out.println(dfDE.format(now)); // 1. Januar 2025

// Время
DateFormat tfUS = DateFormat.getTimeInstance(DateFormat.MEDIUM, us);
DateFormat tfDE = DateFormat.getTimeInstance(DateFormat.MEDIUM, germany);
System.out.println(tfUS.format(now)); // 3:45:12 PM
System.out.println(tfDE.format(now)); // 15:45:12

// Дата + время
DateFormat dtfUS = DateFormat.getDateTimeInstance(DateFormat.MEDIUM, DateFormat.SHORT, us);
```

### Стили DateFormat

| Стиль | Пример (en-US) | Пример (ru-RU) |
|-------|----------------|----------------|
| `SHORT` | 1/1/25 | 01.01.25 |
| `MEDIUM` | Jan 1, 2025 | 1 янв. 2025 г. |
| `LONG` | January 1, 2025 | 1 января 2025 г. |
| `FULL` | Thursday, January 1, 2025 | четверг, 1 января 2025 г. |

## 📅 DateTimeFormatter (Java 8+)

Современный API для работы с датами и временем.

```java
LocalDate date = LocalDate.of(2025, 12, 31);
LocalTime time = LocalTime.of(14, 30, 15);
LocalDateTime dateTime = LocalDateTime.of(date, time);

// Предопределённые форматы
DateTimeFormatter formatterUS = DateTimeFormatter.ofLocalizedDateTime(FormatStyle.LONG)
    .withLocale(Locale.US);
DateTimeFormatter formatterDE = DateTimeFormatter.ofLocalizedDateTime(FormatStyle.LONG)
    .withLocale(Locale.GERMANY);

System.out.println(formatterUS.format(dateTime)); // December 31, 2025 at 2:30:15 PM
System.out.println(formatterDE.format(dateTime)); // 31. Dezember 2025 um 14:30:15

// Кастомный формат (языконезависимые паттерны)
DateTimeFormatter custom = DateTimeFormatter.ofPattern("dd.MM.yyyy HH:mm");
System.out.println(custom.format(dateTime)); // 31.12.2025 14:30 (работает везде)
```

## 🌐 MessageFormat — параметризованные сообщения

```java
ResourceBundle bundle = ResourceBundle.getBundle("messages", Locale.GERMANY);

// Шаблон: "Welcome {0}! You have {1} new messages."
String pattern = bundle.getString("welcome.messages");
String result = MessageFormat.format(pattern, "John", 42);
System.out.println(result); // Welcome John! You have 42 new messages.

// С форматированием чисел внутри шаблона
String template = "Price: {0,number,currency}";
String formatted = MessageFormat.format(template, 1234.56);
System.out.println(formatted); // Price: 1.234,56 € (зависит от локоли)
```

### Форматные типы в MessageFormat

| Тип | Стиль | Пример | Результат |
|-----|-------|--------|-----------|
| `number` | `integer` | `{0,number,integer}` | 123 → 123 |
| `number` | `currency` | `{0,number,currency}` | 123.45 → $123.45 |
| `number` | `percent` | `{0,number,percent}` | 0.5 → 50% |
| `date` | `short/medium/long` | `{0,date,long}` | Jan 1, 2025 |
| `time` | `short/medium/long` | `{0,time,short}` | 3:45 PM |
| `choice` | — | `{0,choice,0#no|1#one|1<many}` | множественные формы |

### Множественные формы (choice)

```properties
# messages_en.properties
messages=You have {0,choice,0#no messages|1#one message|1<{0} messages}
```

```java
ResourceBundle bundle = ResourceBundle.getBundle("messages", Locale.ENGLISH);
String pattern = bundle.getString("messages");

System.out.println(MessageFormat.format(pattern, 0)); // You have no messages
System.out.println(MessageFormat.format(pattern, 1)); // You have one message
System.out.println(MessageFormat.format(pattern, 5)); // You have 5 messages
```

## 🧩 Альтернативные форматы ресурсов (Java 9+)

### ResourceBundle с PropertyResourceBundle (классический)

```java
ResourceBundle bundle = ResourceBundle.getBundle("MyResources", locale);
// ищет MyResources_en_US.properties, MyResources_en.properties, MyResources.properties
```

### ResourceBundle с ListResourceBundle (Java-классы)

```java
// MyResources_ru.java
import java.util.*;

public class MyResources_ru extends ListResourceBundle {
    private static final Object[][] CONTENTS = {
        {"hello", "Привет"},
        {"goodbye", "До свидания"},
        {"color", "красный"}
    };
    
    protected Object[][] getContents() { return CONTENTS; }
}

// Загрузка работает так же
ResourceBundle bundle = ResourceBundle.getBundle("MyResources", new Locale("ru"));
```

### PropertyResourceBundle с XML (устаревший)

```xml
<!-- messages.xml -->
<!DOCTYPE properties SYSTEM "http://java.sun.com/dtd/properties.dtd">
<properties>
    <entry key="hello">Hello</entry>
    <entry key="goodbye">Goodbye</entry>
</properties>
```

```java
try (InputStream is = getClass().getResourceAsStream("messages.xml")) {
    ResourceBundle bundle = new PropertyResourceBundle(is);
    System.out.println(bundle.getString("hello"));
}
```

## ⚙️ Подкапотные детали

### Кэширование ResourceBundle

`ResourceBundle.getBundle()` кэширует загруженные бандлы. Повторный запрос той же локали возвращает тот же объект.

```java
// Оба вызова вернут один объект из кэша
ResourceBundle bundle1 = ResourceBundle.getBundle("messages", Locale.US);
ResourceBundle bundle2 = ResourceBundle.getBundle("messages", Locale.US);
// bundle1 == bundle2
```

### Механизм поиска (ResourceBundle.Control)

```java
ResourceBundle.Control control = new ResourceBundle.Control() {
    @Override
    public List<String> getFormats(String baseName) {
        return List.of("properties"); // только .properties, без .class
    }
    
    @Override
    public Locale getFallbackLocale(String baseName, Locale locale) {
        return Locale.ENGLISH; // fallback на английский, а не на системный
    }
};
```

### Порядок поиска ресурсов в ClassLoader

1. Папка с классами
2. JAR-файлы в classpath
3. Модульный classpath (Java 9+)

## 🧠 Полный пример: интернационализированное приложение

```java
import java.text.*;
import java.time.*;
import java.time.format.*;
import java.util.*;

public class I18NDemo {
    
    public static void main(String[] args) {
        // Поддержка русской и английской локалей
        Locale[] locales = {Locale.US, new Locale("ru", "RU")};
        
        for (Locale locale : locales) {
            Locale.setDefault(locale);
            ResourceBundle bundle = ResourceBundle.getBundle("app", locale);
            
            System.out.println("\n=== " + locale.getDisplayName() + " ===");
            
            // 1. Простые строки
            System.out.println(bundle.getString("greeting"));
            
            // 2. Параметризованное сообщение
            String pattern = bundle.getString("user.login");
            String loginMsg = MessageFormat.format(pattern, "john_doe", ZonedDateTime.now());
            System.out.println(loginMsg);
            
            // 3. Форматирование чисел
            NumberFormat nf = NumberFormat.getCurrencyInstance(locale);
            System.out.println(bundle.getString("price") + ": " + nf.format(1999.99));
            
            // 4. Форматирование даты (Java 8)
            LocalDate today = LocalDate.now();
            DateTimeFormatter df = DateTimeFormatter.ofLocalizedDate(FormatStyle.LONG)
                .withLocale(locale);
            System.out.println(bundle.getString("today") + ": " + df.format(today));
            
            // 5. Множественные формы
            String pluralPattern = bundle.getString("messages.count");
            for (int count : new int[]{0, 1, 2, 5}) {
                String msg = MessageFormat.format(pluralPattern, count);
                System.out.println("  " + msg);
            }
        }
    }
}
```

**app_en.properties**
```properties
greeting=Welcome to our application!
user.login=User {0} logged in at {1,time,short}
price=Price
today=Today
messages.count=You have {0,choice,0#no messages|1#one message|1<{0} messages}
```

**app_ru.properties**
```properties
greeting=Добро пожаловать в приложение!
user.login=Пользователь {0} вошёл в {1,time,short}
price=Цена
today=Сегодня
messages.count={0,choice,0#нет сообщений|1#одно сообщение|1<{0} сообщений}
```

## ⚠️ Частые ошибки и best practices

### ❌ Неправильно

```java
// 1. Хардкод строк
button.setText("Submit");  // ❌

// 2. Конкатенация для плюрализации
String msg = "You have " + count + " messages";  // ❌

// 3. Загрузка ResourceBundle в цикле
for (int i = 0; i < 1000; i++) {
    ResourceBundle.getBundle("messages");  // ❌ плохо для производительности
}

// 4. Игнорирование локали при парсинге
double val = Double.parseDouble("1.234,56");  // ❌ NumberFormatException
```

### ✅ Правильно

```java
// 1. Использовать ResourceBundle
button.setText(bundle.getString("submit"));

// 2. Использовать MessageFormat с choice
String msg = MessageFormat.format(bundle.getString("message.count"), count);

// 3. Загрузить один раз и переиспользовать
private static final ResourceBundle BUNDLE = ResourceBundle.getBundle("messages");

// 4. Использовать NumberFormat для парсинга
NumberFormat nf = NumberFormat.getInstance(Locale.GERMANY);
double val = nf.parse("1.234,56").doubleValue();
```

### Best practices

1. **Используйте понятные ключи:** `user.login.title`, а не `msg1`
2. **Храните все строки в `.properties`** — ни одного хардкода
3. **Давайте значения по умолчанию** — базовый `messages.properties` (без суффикса)
4. **Используйте `MessageFormat`** для подстановки параметров
5. **Не кэшируйте `DateFormat`/`NumberFormat` статически** — они не thread-safe
6. **Для Java 8+ предпочитайте `DateTimeFormatter`** вместо `DateFormat`
7. **Тестируйте с разными локалями** — проблемы с Right-to-Left (арабский, иврит)

### Пример Collator (сравнение строк)

```java
// В разных языках разный порядок букв
Collator collatorRU = Collator.getInstance(new Locale("ru"));
Collator collatorUS = Collator.getInstance(Locale.US);

List<String> words = Arrays.asList("ёж", "еда", "ель");
words.sort(collatorRU); // Правильный русский порядок
```

### Шпаргалка

* **i18n** — подготовка кода, **l10n** — перевод и адаптация
* **Locale** = язык + страна (опционально): `new Locale("ru", "RU")`
* **ResourceBundle** загружает `messages_en_US.properties` → `messages_en.properties` → `messages.properties`
* **NumberFormat** — числа и валюта под локаль
* **DateTimeFormatter** (Java 8+) — даты и время
* **MessageFormat** — подстановка параметров в строки
* **ChoiceFormat** — правильные множественные формы ("1 message", "2 messages")
* **Никогда не хардкодьте строки** — всё в `.properties`
* **Кэшируйте ResourceBundle** — повторный вызов дёшев (из кэша)
* **Форматтеры не thread-safe** — создавайте на каждый вызов или используйте `ThreadLocal`
