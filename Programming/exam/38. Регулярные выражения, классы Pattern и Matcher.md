# 38. Регулярные выражения, классы Pattern и Matcher

**Регулярные выражения** (regex) — это формальный язык для описания шаблонов поиска, извлечения и замены подстрок в тексте. В Java реализованы через пакет `java.util.regex`.

---

## 🏗️ Архитектура: три ключевых класса

| Класс | Назначение | Immutable | Создание |
|-------|------------|-----------|----------|
| `Pattern` | Скомпилированное регулярное выражение | ✅ | `Pattern.compile(String regex)` |
| `Matcher` | Движок сопоставления для конкретной строки | ❌ (имеет состояние) | `pattern.matcher(CharSequence input)` |
| `PatternSyntaxException` | Непроверяемое исключение при синтаксической ошибке в regex | — | Выбрасывается при `compile()` |

```java
Pattern p = Pattern.compile("\\d+");           // 1. Компиляция (один раз)
Matcher m = p.matcher("abc123def456");         // 2. Привязка к строке
while (m.find()) {
    System.out.println(m.group());              // 3. Поиск и извлечение
}
```

## 📦 Pattern — подробно

### Создание и флаги

```java
// Базовое создание
Pattern p1 = Pattern.compile("\\w+");

// Со флагами (можно комбинировать через |)
Pattern p2 = Pattern.compile("foo", Pattern.CASE_INSENSITIVE | Pattern.UNICODE_CASE);
```

### Основные флаги (performance & behaviour)

| Флаг | Кратко | Значение | Встроенный синтаксис |
|------|--------|----------|---------------------|
| `Pattern.CASE_INSENSITIVE` | `CANON_EQ`Pattern.COMMENTS`Pattern.DOTALL`Pattern.LITERAL`Pattern.MULTILINE`Pattern.UNICODE_CASE`Pattern.UNIX_LINES` | `(?iu)` | |
| 未在指定位置找到匹配项。 |
| `Pattern.COMMENTS` | `(?x)` | Пробелы и комменты игнорируются (для читаемых regex) |
| `Pattern.DOTALL` | `(?s)` | `Pattern.COMMENTS`Pattern.DOTALL`Pattern.LITERAL`Pattern.MULTILINE`Pattern.UNICODE_CASE`Pattern.UNIX_LINES` | `Pattern.COMMENTS`Pattern.DOTALL`Pattern.LITERAL`Pattern.MULTILINE`Pattern.UNICODE_CASE`Pattern.UNIX_LINES` |
| `Pattern.LITERAL` | `\Q\E` | Всё воспринимается как буквальный текст (экранирование отключено) |
| `Pattern.MULTILINE` | `(?m)` | `^` и `$` работают с каждой строкой, а не со всем текстом |
| `Pattern.UNICODE_CASE` | `(?u)` | CASE_INSENSITIVE с учётом Юникода |
| `Pattern.UNIX_LINES` | `(?d)` | Только `\n` считается концом строки |

```java
// Пример DOTALL — точка включает перевод строки
Pattern dotall = Pattern.compile(".*", Pattern.DOTALL);
Matcher m = dotall.matcher("Hello\nWorld");
System.out.println(m.matches()); // true (без DOTALL — false)
```

### Кэширование Pattern (важно для производительности)

```java
// ❌ Плохо — компиляция на каждый вызов
public static boolean isNumber(String s) {
    return Pattern.compile("\\d+").matcher(s).matches();
}

// ✅ Хорошо — статический кэш (thread-safe)
private static final Pattern NUMBER_PATTERN = Pattern.compile("\\d+");
public static boolean isNumber(String s) {
    return NUMBER_PATTERN.matcher(s).matches();
}
```

**Complexity:** `Pattern.compile()` — O(2ⁿ) в худшем случае из-за backtracking (экспоненциальная сложность для некоторых regex). Кэширование критично.

## 🎯 Matcher — подробно

### Состояние Matcher (важно понимать!)

`Matcher` хранит позиции поиска: `start()`, `end()`, `group()` ссылаются на последнее найденное совпадение.

```java
Matcher m = Pattern.compile("\\d+").matcher("a12b34");
System.out.println(m.find()); // true
System.out.println(m.start()); // 1 (индекс начала '1')
System.out.println(m.end());   // 3 (индекс после '2')
System.out.println(m.group()); // "12"

System.out.println(m.find()); // true
System.out.println(m.group()); // "34"

System.out.println(m.find()); // false
System.out.println(m.group()); // ❌ IllegalStateException (нет совпадения)
```

### Основные методы Matcher

#### Поиск и проверка

| Метод | Что делает | Возвращает | Примечание |
|-------|------------|------------|-------------|
| `find()` | Ищет следующее совпадение (перемещая курсор) | `boolean` | Основной метод для итерации |
| `find(int start)` | Сбрасывает и ищет с позиции `start` | `boolean` | Позиция в символах |
| `matches()` | Совпадает ли **вся строка** полностью | `boolean` | Эквивалент `^...$` |
| `lookingAt()` | Совпадает ли **начало строки** | `boolean` | "Анкор на старте" |
| `hitEnd()` | Достигнут ли конец строки при последнем поиске | `boolean` | Полезен для валидации |

#### Извлечение групп

| Метод | Возвращает |
|-------|------------|
| `group()` | Всё совпадение (группа 0) |
| `group(int group)` | Конкретная захваченная группа |
| `groupCount()` | Количество групп (исключая группу 0) |
| `start(int group)` | Начало группы |
| `end(int group)` | Конец группы |

#### Модификация

| Метод | Что делает |
|-------|------------|
| `replaceAll(String replacement)` | Заменяет **все** совпадения |
| `replaceFirst(String replacement)` | Заменяет **первое** совпадение |
| `appendReplacement(StringBuffer, String)` | Построчная замена с логикой |
| `appendTail(StringBuffer)` | Добавляет остаток строки |

```java
// Мощный пример: замена с логикой
String input = "price: 10, discount: 3";
StringBuffer sb = new StringBuffer();
Matcher m = Pattern.compile("\\d+").matcher(input);
while (m.find()) {
    int val = Integer.parseInt(m.group());
    m.appendReplacement(sb, String.valueOf(val * 2));
}
m.appendTail(sb);
System.out.println(sb); // "price: 20, discount: 6"
```

#### Сброс состояния

```java
Matcher m = Pattern.compile("\\d+").matcher("123 456");
m.find(); // находит "123"
m.reset(); // сбрасывает позицию
m.find(); // снова "123"

m.reset("789 000"); // новая строка (переиспользование Matcher)
```

## 📖 Синтаксис регулярных выражений (Java-специфичный)

### Символьные классы

| Конструкция | Значение | Пример |
|-------------|----------|--------|
| `[abc]` | a, b или c | `[aeiou]` |
| `[^abc]` | Любой, кроме a,b,c | `[^0-9]` |
| `[a-z]` | Диапазон | `[A-Za-z]` |
| `\d` | Цифра `[0-9]` | `\\d{3}` |
| `\D` | Не цифра | |
| `\w` | Буква/цифра/`_` `[A-Za-z0-9_]` | |
| `\W` | Не `\w` | |
| `\s` | Пробельный символ | |
| `\S` | Не пробел | |

### Квантификаторы (жадные, ленивые, сверхжадные)

| Квантификатор | Значение | Пример (на строке `"aaa"`) |
|---------------|----------|---------------------------|
| `x?` | 0 или 1 | `a?` → `"a"`, `""` |
| `x*` | 0 или более | `a*` → `"aaa"`, `""` |
| `x+` | 1 или более | `a+` → `"aaa"` |
| `x{n}` | ровно n | `a{2}` → `"aa"` |
| `x{n,}` | n или более | `a{2,}` → `"aaa"` |
| `x{n,m}` | от n до m | `a{1,2}` → `"aa"` |

**Важнейший нюанс — жадность:**

```java
String s = "abbbc";

// Жадный (greedy) — захватывает максимум
Pattern.compile(".*b").matcher(s).find(); // "abbb"

// Ленивый (reluctant) — захватывает минимум (добавить ?)
Pattern.compile(".*?b").matcher(s).find(); // "ab"

// Сверхжадный (possessive) — не возвращает захваченное при неудаче (добавить +)
Pattern.compile(".*+b").matcher(s).find(); // false (никогда не откатывается)
```

### Группы и обратные ссылки

```java
// Захватывающие группы (номерные)
Pattern.compile("(\\d{3})-(\\d{2})-(\\d{2})"); // 3 группы

// Именованные группы (Java 7+)
Pattern.compile("(?<year>\\d{4})-(?<month>\\d{2})");
Matcher m = p.matcher("2024-12");
if (m.matches()) {
    System.out.println(m.group("year"));  // "2024"
    System.out.println(m.group(1));       // "2024" (номер тоже работает)
}

// Обратные ссылки
Pattern.compile("(\\w+)\\s+\\1"); // "foo foo" — повтор слова
```

### Якоря и границы

| Конструкция | Значение |
|-------------|----------|
| `^` | Начало строки (или строки при MULTILINE) |
| `$` | Конец строки |
| `\b` | Граница слова |
| `\B` | НЕ граница слова |
| `\A` | Начало всего ввода (игнорирует MULTILINE) |
| `\z` | Конец всего ввода |
| `\Z` | Конец ввода перед `\n` в конце |

## ⚙️ Подкапотная механика (NFA engine)

Java использует **NFA (Non-deterministic Finite Automaton)** с backtracking:

1. Движок проходит по regex слева направо
2. При выборе (например, `a*` + `b`) запоминает точку ветвления
3. Если дальнейший путь не подходит — возвращается (backtrack) и пробует альтернативу

**Проблема — Catastrophic Backtracking (экспоненциальная сложность):**

```java
// ❌ ОПАСНО — экспоненциальное время
Pattern.compile("(a+)+b");
// На строке "aaaaaaaaac" (без b в конце) → тысячи backtrack'ов
// На строке из 30 'a' → миллиарды операций

// ✅ Исправление — сверхжадный квантификатор (не возвращается)
Pattern.compile("(a++)+b");

// ✅ Или атомарная группа (Java 7+)
Pattern.compile("(?>(a+)+)b");
```

**Меры предосторожности:**
- Не используйте вложенные квантификаторы на непроверенных строках
- Устанавливайте таймаут (Java 20+): `Pattern.compile("...").matcher(input).results().limit(1)`
- Для экстремальных случаев — предварительная валидация длины

## 📊 Complexity таблица

| Операция | Time Complexity | Примечание |
|----------|----------------|-------------|
| `Pattern.compile()` | Обычно O(n), но может быть экспоненциальной | Зависит от сложности regex |
| `Matcher.find()` | O(m * n) в худшем случае (backtracking) | m — длина строки |
| `Matcher.matches()` | То же, что find | Полный обход |
| `replaceAll()` | O(m * n) | Проходит всю строку |
| `matches()` с простым regex `\d+` | O(n) | Линейное время |

## 🧠 Полный пример — парсинг логов

```java
import java.util.regex.*;
import java.util.*;

public class LogParser {
    private static final Pattern LOG_PATTERN = Pattern.compile(
        "^(?<timestamp>\\d{4}-\\d{2}-\\d{2}\\s+\\d{2}:\\d{2}:\\d{2})\\s+" +
        "(?<level>ERROR|WARN|INFO)\\s+" +
        "(?<message>.+?)(?:\\s+\\{(?<metadata>.*?)\\})?$"
    );
    
    public static Map<String, Object> parseLine(String line) {
        Matcher m = LOG_PATTERN.matcher(line);
        if (!m.matches()) return Collections.emptyMap();
        
        Map<String, Object> result = new LinkedHashMap<>();
        result.put("timestamp", m.group("timestamp"));
        result.put("level", m.group("level"));
        result.put("message", m.group("message"));
        
        String metadata = m.group("metadata");
        if (metadata != null) {
            // Дополнительный разбор метаданных
            Pattern metaPattern = Pattern.compile("(\\w+)=(\\w+)");
            Matcher metaMatcher = metaPattern.matcher(metadata);
            Map<String, String> meta = new HashMap<>();
            while (metaMatcher.find()) {
                meta.put(metaMatcher.group(1), metaMatcher.group(2));
            }
            result.put("metadata", meta);
        }
        return result;
    }
    
    public static void main(String[] args) {
        String log = "2024-12-10 14:30:00 ERROR Database connection failed {code=500}";
        System.out.println(parseLine(log));
    }
}
```

## ⚠️ Частые ошибки и контракты

### 1. Двойной слэш для экранирования

```java
// ❌ Ошибка — компилятор съест \d
Pattern.compile("\d+");  // Invalid escape sequence

// ✅ Правильно — \\d для Java, \d для regex
Pattern.compile("\\d+");
```

### 2. matches() vs find()

```java
String s = "123abc";
Pattern p = Pattern.compile("\\d+");

System.out.println(p.matcher(s).matches()); // false (есть ещё abc)
System.out.println(p.matcher(s).find());    // true (123 найдено)
System.out.println(p.matcher(s).lookingAt());// true (начало подходит)
```

### 3. Несброшенный Matcher

```java
Matcher m = Pattern.compile("\\d+").matcher("a1b2c3");
while (m.find()) {
    if (m.group().equals("2")) {
        m.find(); // ❌ Прыгает внутри цикла — ломает состояние
    }
}
```

### 4. Производительность replaceAll() в цикле

```java
// ❌ Плохо — компиляция на каждой итерации
for (String s : list) {
    s.replaceAll("\\d+", "X");
}

// ✅ Хорошо — один Pattern
Pattern digits = Pattern.compile("\\d+");
for (String s : list) {
    digits.matcher(s).replaceAll("X");
}
```

## 🔗 Связи с другими темами

- `String.matches()` — делегирует в `Pattern.matches()` (компилирует каждый вызов — медленно)
- `Scanner` — внутренне использует Pattern для токенизации
- `String.split()` — принимает regex, компилирует в Pattern
- `java.util.regex.MatchResult` — интерфейс, реализованный Matcher

```java
// Альтернативный подход через Matcher.results() (Java 9+)
Pattern.compile("\\d+").matcher("a1b2c3")
    .results()
    .map(MatchResult::group)
    .forEach(System.out::println);
```

### Шпаргалка

* **Pattern — скомпилированный regex** (immutable, thread-safe)  
* **Matcher — применяет Pattern к строке** (mutable, NOT thread-safe)  
* **`find()` ищет частичные совпадения**, `matches()` — только полное совпадение всей строки  
* **В Java regex нужен двойной слэш:** `\\d` вместо `\d`  
* **Жадные квантификаторы (`*`, `+`) могут вызвать катастрофический backtracking** → используй possessive (`++`, `*+`) или атомарные группы  
* **Кэшируй Pattern** — компиляция дорогая  
* **Именованные группы** `(?<name>...)` с `group("name")` — читаемее цифр  
* **Регулярки не подходят для сложного синтаксического разбора** (вложенные структуры) — там нужен парсер
