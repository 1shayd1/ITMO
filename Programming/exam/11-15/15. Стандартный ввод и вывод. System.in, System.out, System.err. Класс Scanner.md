# 15. Стандартный ввод и вывод. `System.in`, `System.out`, `System.err`. Класс `Scanner`

## 💡 Общая идея

В Java стандартный ввод-вывод реализован через класс:

```java
System
```

Он предоставляет доступ к консоли (и перенаправляемым потокам ввода/вывода).

---

# 🔵 Потоки `System`

## 1. `System.in`

```java
InputStream
```

👉 стандартный поток ввода (обычно клавиатура)

### Особенности:

- байтовый поток
- читает данные как `InputStream`
- неудобен напрямую для текста

---

### Проблема

```java
System.in.read()
```

читает **по одному байту**, поэтому обычно оборачивается:

- `Scanner`
- `BufferedReader`

---

## 2. `System.out`

```java
PrintStream
```

👉 стандартный поток вывода (обычно консоль)

---

### Методы:

```java
print()
println()
printf()
```

---

## 3. `System.err`

```java
PrintStream
```

👉 поток ошибок (stderr)

---

### Отличие от System.out:

| System.out | System.err |
|------------|------------|
| обычный вывод | ошибки |
| стандартный поток | диагностический поток |
| можно разделять в терминале | можно логировать отдельно |

---

### Пример

```java
System.out.println("OK");
System.err.println("ERROR");
```

---

# 🟡 Класс Scanner

## Что это

`Scanner` — удобный парсер входного потока.

```java
Scanner sc = new Scanner(System.in);
```

---

## Что умеет Scanner

✔ читать числа  
✔ читать строки  
✔ разбивать по разделителям  
✔ парсить токены  

---

## Пример

```java
Scanner sc = new Scanner(System.in);

int a = sc.nextInt();
String s = sc.next();
```

---

## Основные методы

```java
next()        // слово
nextLine()    // строка
nextInt()     // int
nextDouble()  // double
hasNext()     // проверка
```

---

## Как работает Scanner внутри

```text
System.in → byte stream → Scanner → tokens
```

---

## Разделители

По умолчанию:

```text
whitespace (пробел, \n, \t)
```

Можно менять:

```java
sc.useDelimiter(",");
```

---

# ⚠ Проблемы Scanner

## 1. nextLine после nextInt

```java
int a = sc.nextInt();
String s = sc.nextLine(); // ❌ проблема
```

👉 остается символ новой строки

---

## 2. Производительность

Scanner:

- медленный
- использует регулярные выражения

---

# ⚖ Scanner vs BufferedReader

| Характеристика | Scanner | BufferedReader |
|----------------|--------|----------------|
| удобство | ⭐⭐⭐⭐ | ⭐⭐ |
| скорость | ⭐⭐ | ⭐⭐⭐⭐⭐ |
| парсинг чисел | встроен | вручную |
| производительность | низкая | высокая |

---

## BufferedReader альтернатива

```java
BufferedReader br =
    new BufferedReader(new InputStreamReader(System.in));

int a = Integer.parseInt(br.readLine());
```

---

# 🔥 Типовой вопрос преподавателя

## ❓ Почему System.in неудобен?

✔ потому что он работает с байтами, а не строками

---

## ❓ Почему Scanner медленный?

✔ потому что:
- парсит токены
- использует регулярные выражения

---

## ❓ Зачем нужен System.err?

✔ для отделения ошибок от обычного вывода

---

# 🧠 Шпаргалка

* `System.in` = InputStream (ввод)
* `System.out` = PrintStream (вывод)
* `System.err` = PrintStream (ошибки)
* `Scanner` = удобный парсер ввода
* Scanner медленнее BufferedReader
* `nextLine()` + `nextInt()` → частая ошибка
* для скорости лучше BufferedReader
