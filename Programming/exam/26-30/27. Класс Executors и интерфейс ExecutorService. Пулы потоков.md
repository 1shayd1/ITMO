# 27. Класс `Executors` и интерфейс `ExecutorService`. Пулы потоков

## 💡 Общая идея

`Executors` — это утилитарный (factory) класс, который предоставляет готовые реализации `ExecutorService`.

Он позволяет быстро создавать **пулы потоков** без ручной настройки потоков.

---

# 🧠 Зачем нужны пулы потоков

Создание потока — дорогая операция:

- выделение памяти
- создание системного потока
- переключение контекста

---

## ❗ Проблема без пула

```text
каждая задача → новый Thread → большие затраты
```

---

## ✅ Решение: пул потоков

```text
задачи → очередь → ограниченное число потоков → переиспользование
```

---

# 🔵 Executors (фабрика пулов)

## Основные методы

### 1. FixedThreadPool

```java
Executors.newFixedThreadPool(n);
```

✔ фиксированное количество потоков  
✔ задачи ставятся в очередь  

---

### 2. CachedThreadPool

```java
Executors.newCachedThreadPool();
```

✔ динамическое количество потоков  
✔ создаёт новые при необходимости  
✔ переиспользует свободные  

---

### 3. SingleThreadExecutor

```java
Executors.newSingleThreadExecutor();
```

✔ один поток  
✔ задачи выполняются последовательно  

---

### 4. ScheduledThreadPool

```java
Executors.newScheduledThreadPool(n);
```

✔ выполнение задач по расписанию  
✔ задержки и периодичность  

---

# 🟢 ExecutorService

## Что это

`ExecutorService` — интерфейс, который управляет:

- выполнением задач
- пулом потоков
- жизненным циклом

---

## Основные методы

### submit()

```java
Future<?> f = pool.submit(() -> System.out.println("Task"));
```

---

### shutdown()

```java
pool.shutdown();
```

✔ завершает приём новых задач  

---

### shutdownNow()

```java
pool.shutdownNow();
```

✔ принудительное завершение  

---

### awaitTermination()

```java
pool.awaitTermination(10, TimeUnit.SECONDS);
```

✔ ожидание завершения задач  

---

# ⚙ Пример использования пула

```java
ExecutorService pool = Executors.newFixedThreadPool(4);

pool.submit(() -> System.out.println("Hello"));
pool.submit(() -> System.out.println("World"));

pool.shutdown();
```

---

# 🔗 Как работает пул потоков

```text
задача → очередь задач → пул потоков → выполнение
```

---

# ⚠ Важные особенности

## 1. Потоки переиспользуются

```text
один поток выполняет много задач
```

---

## 2. Очередь задач

Если потоков нет свободных:

```text
задачи ждут в очереди
```

---

## 3. shutdown обязателен

Иначе программа может не завершиться.

---

# 💥 Частые ошибки

### ❌ не вызывают shutdown()

→ приложение "зависает"

---

### ❌ создают new Thread вместо пула

→ падение производительности

---

### ❌ используют cached pool без контроля

→ риск создания слишком многих потоков

---

# 🧠 Шпаргалка

- `Executors` — фабрика пулов потоков
- пул потоков переиспользует Thread
- `newFixedThreadPool` — фиксированное число потоков
- `newCachedThreadPool` — динамический пул
- `newSingleThreadExecutor` — один поток
- `newScheduledThreadPool` — задачи по времени
- `shutdown()` обязателен
