# 26. Многопоточность. Интерфейсы Executor, ExecutorService, Callable, Future

## 💡 Общая идея

Современный подход к многопоточности в Java основан на принципе:

> отделение задачи от способа выполнения

---

# 🔵 Executor

## Что это

`Executor` — самый простой интерфейс для выполнения задач.

### Метод:

```java
void execute(Runnable command);
```

---

## Пример

```java
Executor ex = command -> new Thread(command).start();

ex.execute(() -> System.out.println("Task"));
```

---

## Особенности

- принимает Runnable
- не управляет потоками
- базовый уровень абстракции

---

# 🟢 ExecutorService

## Что это

`ExecutorService` — расширение Executor с управлением задачами и потоками.

---

## Возможности

- submit задач
- получение результата
- управление жизненным циклом

---

## Пример создания

```java
ExecutorService service = Executors.newFixedThreadPool(2);
```

---

## Завершение работы

```java
service.shutdown();
```

или

```java
service.shutdownNow();
```

---

# 🟡 Callable\<V\>

## Что это

`Callable` — задача, которая:

- возвращает результат
- может бросать исключения

---

## Пример

```java
Callable<Integer> task = () -> 42;
```

---

## Отличие от Runnable

| Runnable | Callable |
|----------|----------|
| void run() | V call() |
| нет результата | есть результат |
| нет checked exceptions | есть checked exceptions |

---

# 🔵 Future\<V\>

## Что это

`Future` — объект, который хранит результат асинхронной задачи.

---

## Основные методы

```java
V get()
boolean isDone()
boolean cancel(boolean mayInterruptIfRunning)
```

---

## Пример

```java
ExecutorService service = Executors.newSingleThreadExecutor();

Future<Integer> future = service.submit(() -> 42);

int result = future.get();

service.shutdown();
```

---

## Как работает Future

task → ExecutorService → Future → result

---

# ⚙ Полная схема

```text
Callable → ExecutorService → Future → result
```

---

# ⚠ Важные моменты

## get() блокирует поток

Пока результат не готов — поток ждёт

---

## обязательно shutdown()

Иначе потоки не завершатся

---

## Callable лучше Runnable

если нужен результат

---

# 💥 Частые ошибки

## ❌ забыли shutdown()

→ утечка потоков

---

## ❌ неправильное использование get()

→ блокировка программы

---

## ❌ путаница Runnable / Callable

→ отсутствие результата или ошибка компиляции

---

# 🧠 Шпаргалка

- Executor — выполняет задачи
- ExecutorService — управляет потоками
- Callable — задача с результатом
- Future — результат асинхронной операции
- get() — блокирует до результата
- shutdown() — обязателен
