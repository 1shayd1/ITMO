# 30. Интерфейсы Lock, ReadWriteLock, Condition и реализующие их классы

## Общая идея

В Java, помимо ключевого слова `synchronized`, существует более гибкий API для синхронизации потоков — пакет  
`java.util.concurrent.locks`.

Он позволяет:
* более точно управлять блокировками;
* использовать тайм-ауты и попытки захвата;
* разделять чтение и запись;
* создавать несколько условий ожидания.

---

# 1. Интерфейс `Lock`

## Назначение

`Lock` — это более гибкая альтернатива `synchronized`, которая позволяет вручную управлять блокировкой.

---

## Основные реализации

* `ReentrantLock`

---

## Основные методы

| Метод | Назначение |
|------|------------|
| `lock()` | захват блокировки |
| `unlock()` | освобождение |
| `tryLock()` | попытка захвата без ожидания |
| `tryLock(time)` | попытка с тайм-аутом |

---

## Пример использования

```java
Lock lock = new ReentrantLock();

lock.lock();
try {
    // критическая секция
} finally {
    lock.unlock();
}
```

---

## Особенности `Lock`

* нет автоматического освобождения (в отличие от `synchronized`)
* можно прерывать ожидание
* можно использовать тайм-ауты
* можно проверять доступность блокировки

---

## Дополнительные возможности `ReentrantLock`

```java
ReentrantLock lock = new ReentrantLock(true);
```

✔ fair lock (справедливый доступ)

---

# 2. Интерфейс `ReadWriteLock`

## Назначение

`ReadWriteLock` разделяет доступ на:

* множество потоков чтения одновременно;
* только один поток записи.

---

## Основная реализация

* `ReentrantReadWriteLock`

---

## Получение блокировок

```java
ReadWriteLock rw = new ReentrantReadWriteLock();

Lock readLock = rw.readLock();
Lock writeLock = rw.writeLock();
```

---

## Пример

```java
ReadWriteLock rw = new ReentrantReadWriteLock();

Lock readLock = rw.readLock();
Lock writeLock = rw.writeLock();

// чтение
readLock.lock();
try {
    // читаем данные
} finally {
    readLock.unlock();
}

// запись
writeLock.lock();
try {
    // изменяем данные
} finally {
    writeLock.unlock();
}
```

---

## Когда использовать

* много чтений → мало записей
* кэширование
* shared state (например, конфигурации)

---

## Особенности

* несколько читателей одновременно
* запись — эксклюзивная
* возможна блокировка чтения при активной записи

---

# 3. Интерфейс `Condition`

## Назначение

`Condition` — механизм ожидания и уведомления, аналог `wait()/notify()`, но работает с `Lock`.

---

## Создание

```java
Lock lock = new ReentrantLock();
Condition condition = lock.newCondition();
```

---

## Основные методы

| Метод | Назначение |
|------|------------|
| `await()` | ожидание |
| `signal()` | пробуждение одного потока |
| `signalAll()` | пробуждение всех |

---

## Пример

```java
Lock lock = new ReentrantLock();
Condition condition = lock.newCondition();

lock.lock();
try {
    while (!conditionMet) {
        condition.await();
    }

    // работа
} finally {
    lock.unlock();
}
```

---

## Уведомление

```java
lock.lock();
try {
    condition.signal();
} finally {
    lock.unlock();
}
```

---

## Отличие от `wait/notify`

| `synchronized` | `Lock` |
|----------------|--------|
| wait() | await() |
| notify() | signal() |
| notifyAll() | signalAll() |

---

## Преимущества `Condition`

* можно создавать несколько условий для одного lock
* более гибкая синхронизация
* управление очередями ожидания

---

# 4. Сравнение `synchronized` и `Lock`

| Характеристика | synchronized | Lock |
|----------------|-------------|------|
| автоматическое освобождение | да | нет |
| tryLock | нет | да |
| тайм-аут | нет | да |
| fairness | нет | да |
| Condition | нет | да |

---

# 5. Типовые реализации

## ReentrantLock

* основная реализация `Lock`
* поддерживает повторный вход (reentrant)

```java
Lock lock = new ReentrantLock();
```

---

## ReentrantReadWriteLock

```java
ReadWriteLock rw = new ReentrantReadWriteLock();
```

---

# 6. Типичные ошибки

## 1. Забыл unlock()

```java
lock.lock();
// ошибка: нет unlock()
```

✔ всегда использовать `try/finally`

---

## 2. Использование Condition без lock

```java
condition.await(); // ошибка
```

✔ всегда внутри `lock.lock()`

---

## 3. Deadlock

При неправильном порядке блокировок:

```text
Thread A -> Lock1 -> Lock2
Thread B -> Lock2 -> Lock1
```

---

# Шпаргалка

* `Lock` — гибкая альтернатива `synchronized`
* `ReentrantLock` — основная реализация
* `ReadWriteLock` — разделение чтения и записи
* `ReentrantReadWriteLock` — реализация RW-блокировки
* `Condition` — аналог wait/notify
* всегда использовать `try/finally` с `unlock()`
* `Condition` работает только внутри `Lock`
