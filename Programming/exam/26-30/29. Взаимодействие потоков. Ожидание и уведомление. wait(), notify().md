# 29. Взаимодействие потоков. Ожидание и уведомление. `wait()`, `notify()`

## 💡 Общая идея

В Java потоки могут **синхронизироваться через общий монитор объекта**.

Для этого используются методы:

- `wait()`
- `notify()`
- `notifyAll()`

---

# 🔴 wait()

## Что делает

`wait()` переводит поток в состояние ожидания и **освобождает монитор объекта**.

---

## Что происходит при wait()

```text
поток → освобождает lock → WAITING
```

---

## Пример

```java
synchronized (lock) {
    while (!ready) {
        lock.wait();
    }
}
```

---

# 🟢 notify()

## Что делает

`notify()` пробуждает **один случайный поток**, который ждёт на этом же мониторе.

---

## Пример

```java
synchronized (lock) {
    ready = true;
    lock.notify();
}
```

---

# 🔵 notifyAll()

## Что делает

`notifyAll()` пробуждает **все потоки**, ожидающие на мониторе.

---

## Пример

```java
synchronized (lock) {
    ready = true;
    lock.notifyAll();
}
```

---

# ⚙ Важные правила

## 1. Только внутри synchronized

```text
❌ нельзя вызывать без монитора
```

```java
synchronized (lock) {
    lock.wait(); // OK
}
```

---

## 2. Один и тот же объект-монитор

```text
wait/notify работают только на одном объекте
```

---

## 3. Поток должен владеть монитором

Иначе:

```text
IllegalMonitorStateException
```

---

# 🔗 Как это работает

## wait()

```text
поток захватывает lock
→ вызывает wait()
→ освобождает lock
→ переходит в WAITING
```

---

## notify()

```text
поток захватывает lock
→ вызывает notify()
→ будит другой поток
→ освобождает lock
```

---

# 🧠 Почему используется while

## ❗ Причина

Поток может проснуться:

- без notify (spurious wakeup)
- раньше, чем условие станет true

---

## Правильный вариант

```java
while (!condition) {
    lock.wait();
}
```

---

## ❌ Неправильно

```java
if (!condition) {
    lock.wait();
}
```

---

# ⚠ Частые ошибки

### ❌ вызов wait() без synchronized

→ IllegalMonitorStateException

---

### ❌ notify() без изменения состояния

→ поток может продолжить ждать

---

### ❌ использование if вместо while

→ логические ошибки

---

# 💥 Пример Producer–Consumer

## Producer

```java
synchronized (lock) {
    queue.add(item);
    lock.notifyAll();
}
```

---

## Consumer

```java
synchronized (lock) {
    while (queue.isEmpty()) {
        lock.wait();
    }
    queue.remove();
}
```

---

# 🧠 Шпаргалка

- `wait()` → поток засыпает и освобождает монитор
- `notify()` → будит один поток
- `notifyAll()` → будит все потоки
- работать можно только внутри synchronized
- всегда использовать `while`, а не `if`
- все методы работают через один и тот же объект-монитор
