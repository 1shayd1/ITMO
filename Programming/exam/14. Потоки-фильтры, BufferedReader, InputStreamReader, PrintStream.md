# 14. Потоки-фильтры, `BufferedReader`, `InputStreamReader`, `PrintStream`

## 💡 Общая идея потоков-фильтров

**Потоки-фильтры (filter streams)** — это обертки над базовыми потоками, которые:

- не изменяют источник данных
- добавляют новую функциональность
- строятся по принципу декоратора (Decorator Pattern)

---

## 🧠 Основная концепция

```text
базовый поток → фильтр → фильтр → удобный поток
```

Пример:

```text
FileInputStream
   ↓
InputStreamReader
   ↓
BufferedReader
```

---

# 🟢 InputStreamReader

## Что это

`InputStreamReader` — мост между:

```text
байты → символы
```

---

## Зачем нужен

Потому что:

- `InputStream` работает с байтами
- `Reader` работает с символами

👉 нужен переход между ними

---

## Важная роль

Он учитывает:

```text
кодировку (UTF-8, UTF-16 и т.д.)
```

---

## Пример

```java
try (Reader r = new InputStreamReader(System.in)) {
    int ch = r.read();
}
```

---

## Часто используется с файлами

```java
Reader r = new InputStreamReader(
    new FileInputStream("file.txt")
);
```

---

# 🔵 BufferedReader

## Что это

`BufferedReader` — буферизированный символьный поток.

---

## Что добавляет

✔ буфер (ускорение работы)  
✔ удобный метод `readLine()`

---

## Почему он быстрее

Без буфера:

```text
каждый read() → системный вызов
```

С буфером:

```text
читается блок данных сразу
```

---

## Пример

```java
try (BufferedReader br =
         new BufferedReader(new FileReader("a.txt"))) {

    String line = br.readLine();
}
```

---

## Основной метод

```java
String readLine()
```

👉 читает строку целиком

---

## Типичный цикл

```java
String line;
while ((line = br.readLine()) != null) {
    System.out.println(line);
}
```

---

# 🟡 PrintStream

## Что это

`PrintStream` — поток для **удобного вывода текста и примитивов**.

---

## Где используется

```java
System.out
System.err
```

---

## Основные методы

```java
print()
println()
printf()
```

---

## Особенности

✔ автоматически преобразует типы в строки  
✔ не требует ручного преобразования  
✔ не бросает checked IOException  

---

## Пример

```java
PrintStream out = System.out;

out.println("Hello");
out.printf("Age: %d%n", 20);
```

---

## Важный факт

```text
System.out = PrintStream
```

---

# ⚙ Как всё связано (цепочка)

## Типичная конструкция ввода текста:

```java
BufferedReader br =
    new BufferedReader(
        new InputStreamReader(
            System.in
        )
    );
```

---

## Что происходит по слоям:

| Уровень | Роль |
|---------|------|
| System.in | байты |
| InputStreamReader | байты → символы |
| BufferedReader | буфер + удобство |

---

# ⚖ Сравнение компонентов

| Класс | Тип | Роль |
|------|-----|------|
| InputStreamReader | мост | byte → char |
| BufferedReader | фильтр | ускорение + readLine |
| PrintStream | output | удобный вывод |

---

# 💥 Частые ошибки

### ❌ Использовать InputStream напрямую для текста

→ проблемы с кодировкой

---

### ❌ Не использовать BufferedReader

→ медленное чтение

---

### ❌ Читать System.in через Scanner без понимания буферов

→ скрытая сложность и overhead

---

# 🔥 Типовой вопрос преподавателя

## ❓ Зачем нужен InputStreamReader?

✔ чтобы преобразовать байты в символы с учетом кодировки

---

## ❓ Почему BufferedReader быстрее?

✔ потому что читает данные блоками, а не по одному символу

---

## ❓ Что такое PrintStream?

✔ поток для форматированного текстового вывода

---

# 🧠 Шпаргалка

* Потоки-фильтры = обертки над потоками
* `InputStreamReader` = byte → char
* `BufferedReader` = буфер + readLine()
* `PrintStream` = удобный вывод
* `System.out` = PrintStream
* лучшая цепочка ввода текста:
  ```text
  System.in → InputStreamReader → BufferedReader
  ```
