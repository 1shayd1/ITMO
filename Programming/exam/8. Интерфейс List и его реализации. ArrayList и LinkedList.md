# 8. Интерфейс `List` и его реализации. `ArrayList` и `LinkedList`

## Что такое `List`

`List` — это упорядоченная коллекция, которая:

- хранит элементы в порядке добавления
- допускает дубликаты
- поддерживает доступ по индексу

---

## Основные операции `List`

```java
list.add(E e)
list.get(int index)
list.set(int index, E e)
list.remove(int index)
list.add(int index, E e)
```

---

## Внутренняя иерархия

```text
Collection
   ↓
List
 ├── ArrayList
 ├── LinkedList
 └── Vector (устаревший)
```

---

# 🔵 ArrayList

## Что это

`ArrayList` — это динамический массив (`Object[]`), который автоматически увеличивается при необходимости.

---

## Внутреннее устройство (ВАЖНО)

```text
Object[] elementData
size
capacity
```

При переполнении:

```text
новый массив = old * 1.5
копирование элементов → O(n)
```

---

## Сложность операций

| операция | сложность |
|----------|----------|
| get(i) | O(1) |
| add (в конец) | O(1) амортизированно |
| insert (в середину) | O(n) |
| remove | O(n) |

---

## Почему get O(1)

Потому что:

```text
адрес = base + index * size
```

---

## Когда использовать ArrayList

✔ частое чтение  
✔ доступ по индексу  
✔ минимум вставок в середину  

---

## Пример

```java
List<String> list = new ArrayList<>();

list.add("A");
list.add("B");

System.out.println(list.get(0));
```

---

# 🟡 LinkedList

## Что это

`LinkedList` — это двусвязный список:

```text
Node {
    E item;
    Node next;
    Node prev;
}
```

---

## Внутреннее устройство

```text
head ↔ node ↔ node ↔ node ↔ tail
```

---

## Сложность операций

| операция | сложность |
|----------|----------|
| addFirst | O(1) |
| addLast | O(1) |
| get(i) | O(n) |
| insert/delete (если узел найден) | O(1) |

---

## Почему get медленный

Потому что:

```text
нужно пройти по цепочке узлов
```

---

## Особенность LinkedList

Он реализует сразу 3 интерфейса:

```text
List
Deque
Queue
```

---

## Когда использовать LinkedList

✔ частые вставки/удаления  
✔ работа как очередь  
✔ работа с концами структуры  

---

## Пример

```java
LinkedList<String> list = new LinkedList<>();

list.add("A");
list.addFirst("B");
list.addLast("C");
```

---

# ⚖ ArrayList vs LinkedList (главное сравнение)

| характеристика | ArrayList | LinkedList |
|----------------|------------|-------------|
| структура | массив | двусвязный список |
| доступ по индексу | O(1) | O(n) |
| вставка в середину | O(n) | O(1)* |
| память | компактнее | больше (узлы) |
| скорость обхода | быстрее | медленнее |

\* если уже есть ссылка на узел

---

## Важный экзаменационный момент

👉 В реальных задачах:

> ArrayList почти всегда быстрее LinkedList

Потому что:
- CPU cache friendly
- меньше объектов
- меньше переходов по памяти

---

## Частая ошибка студентов

❌ “LinkedList быстрее для вставок всегда”

✔ правильно:

> LinkedList быстрее только если уже есть доступ к позиции

---

## Внутренний нюанс (важно для 5+)

### ArrayList:
- contiguous memory (локальность данных)
- быстрый cache access

### LinkedList:
- scattered memory
- pointer chasing (дорого для CPU)

---

## Когда LinkedList действительно полезен

- реализация deque/queue
- частые операции addFirst/removeFirst
- когда не нужен random access

---

# 💥 Типовой вопрос преподавателя

❓ Почему ArrayList чаще используют, чем LinkedList?

✔ потому что:
- быстрее доступ по индексу (O(1))
- лучше работает с кэшем процессора
- меньше накладных расходов памяти

---

## Шпаргалка

* `List` = порядок + индексы + дубликаты
* `ArrayList` = динамический массив, быстрый доступ O(1)
* `LinkedList` = двусвязный список, быстрые вставки на концах
* `ArrayList` почти всегда быстрее в реальных задачах
* `LinkedList` полезен для deque/queue
