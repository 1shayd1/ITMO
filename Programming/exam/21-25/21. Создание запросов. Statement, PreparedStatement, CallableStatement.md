# 21. Создание запросов. `Statement`, `PreparedStatement`, `CallableStatement`

## 💡 Общая идея

В JDBC для выполнения SQL-запросов используются три основных интерфейса:

- `Statement` — простые запросы
- `PreparedStatement` — параметризованные запросы
- `CallableStatement` — вызов хранимых процедур

---

# 🔵 Statement

## Что это

`Statement` используется для выполнения **простых SQL-запросов без параметров**.

---

## Пример

```java
Statement st = conn.createStatement();
ResultSet rs = st.executeQuery("SELECT * FROM users");
```

---

## Особенности

✔ подходит для статических SQL  
❌ нельзя использовать параметры  
❌ уязвим к SQL-инъекциям при конкатенации строк  
❌ менее эффективен при повторных запросах  

---

## Пример плохой практики

```java
String sql = "SELECT * FROM users WHERE id = " + id;
```

---

# 🟢 PreparedStatement

## Что это

`PreparedStatement` — это **параметризованный SQL-запрос**.

---

## Пример

```java
PreparedStatement ps = conn.prepareStatement(
    "SELECT * FROM users WHERE id = ?"
);

ps.setInt(1, 10);

ResultSet rs = ps.executeQuery();
```

---

## Особенности

✔ защита от SQL-инъекций  
✔ поддержка параметров (`?`)  
✔ лучше производительность при повторном выполнении  
✔ более читаемый код  

---

## Как работают параметры

```text
? → заменяется через setInt, setString и т.д.
```

---

## Примеры методов

```java
ps.setInt(1, 10);
ps.setString(2, "Alex");
ps.setDouble(3, 5.5);
```

---

# 🟡 CallableStatement

## Что это

`CallableStatement` используется для вызова **хранимых процедур** в базе данных.

---

## Пример

```java
CallableStatement cs = conn.prepareCall("{call get_user(?)}");

cs.setInt(1, 10);

ResultSet rs = cs.executeQuery();
```

---

## Особенности

✔ работа с процедурами БД  
✔ поддержка входных и выходных параметров  
✔ используется реже, чем PreparedStatement  

---

## Пример с OUT параметром

```java
CallableStatement cs =
    conn.prepareCall("{call get_count(?)}");

cs.registerOutParameter(1, java.sql.Types.INTEGER);

cs.execute();

int count = cs.getInt(1);
```

---

# ⚖ Сравнение

| Интерфейс | Назначение | Параметры | Безопасность |
|----------|------------|----------|--------------|
| Statement | простой SQL | нет | низкая |
| PreparedStatement | параметризованный SQL | да | высокая |
| CallableStatement | хранимые процедуры | да | высокая |

---

# 🔗 Когда что использовать

## Statement
- быстрые простые запросы (редко используется)

## PreparedStatement
- основной вариант в 90% случаев

## CallableStatement
- когда есть процедуры в БД

---

# 💥 Частые ошибки

### ❌ SQL-инъекции через Statement

```java
"SELECT * FROM users WHERE id = " + id;
```

---

### ❌ неправильное использование ?

- забыли setInt / setString
- перепутали индексы (начинаются с 1)

---

# 🧠 Шпаргалка

* `Statement` — простой SQL без параметров
* `PreparedStatement` — параметризованный SQL (лучший вариант)
* `CallableStatement` — хранимые процедуры
* `?` заменяется через setX()
* всегда предпочитать `PreparedStatement`
