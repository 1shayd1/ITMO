# 46. Размещение компонентов в контейнерах. Менеджеры компоновки

**Менеджер компоновки (layout manager)** — объект, который вычисляет **положение и размер** детей контейнера. Это стратегия (паттерн `Strategy`), подключаемая через `container.setLayout(...)`.

## Зачем

Жёсткие координаты (`null`-layout + `setBounds`) — антипаттерн: интерфейс ломается при изменении размера окна, шрифтов, разрешения и платформы. Менеджер пересчитывает раскладку автоматически, опираясь на **preferred/min/max size** компонентов.

## Контракт

* Интерфейс `LayoutManager`: `addLayoutComponent`, `layoutContainer(Container)`, `preferredLayoutSize`, `minimumLayoutSize`;
* `LayoutManager2` добавляет **constraints** (ограничения на компонент) — его реализуют `BorderLayout`, `GridBagLayout`, `BoxLayout`.
* Значения по умолчанию: у `JPanel` — `FlowLayout`, у content pane `JFrame` — `BorderLayout`.
* После динамического добавления/удаления компонентов: `revalidate()` + `repaint()`.

## Основные менеджеры (Swing/AWT)

| Менеджер | Идея |
|---|---|
| `FlowLayout` | укладывает в строку слева направо, переносит при нехватке места; компоненты — в preferred size |
| `BorderLayout` | 5 зон: `NORTH`, `SOUTH`, `EAST`, `WEST`, `CENTER`; центр растягивается, занимая остаток |
| `GridLayout` | сетка rows×cols с **одинаковыми** ячейками, все растянуты |
| `CardLayout` | стопка «карточек», видна одна (`show`, `next`) — мастера/вкладки вручную |
| `BoxLayout` | вдоль одной оси (`X_AXIS`/`Y_AXIS`); помощники `Box.createHorizontalGlue()`, struts |
| `GridBagLayout` | самый гибкий: сетка с весами; настраивается через `GridBagConstraints` (`gridx/gridy`, `weightx`, `fill`, `insets`) |

```java
JPanel panel = new JPanel(new BorderLayout(5, 5));
panel.add(new JScrollPane(textArea), BorderLayout.CENTER);  // constraint
panel.add(new JButton("Send"), BorderLayout.SOUTH);
```

Сложные формы обычно собирают **вложением** панелей с простыми layout'ами, а не одним `GridBagLayout`.

## Аналоги в JavaFX

В JavaFX менеджер не подключается к контейнеру — **сам контейнер и есть layout** (подклассы `Pane`):

| Swing | JavaFX |
|---|---|
| `FlowLayout` | `FlowPane`, `HBox` |
| `BorderLayout` | `BorderPane` |
| `GridLayout` / `GridBagLayout` | `GridPane` |
| `BoxLayout` | `HBox` / `VBox` |
| — | `StackPane` (наложение), `AnchorPane`, `TilePane` |

### Шпаргалка

* Layout manager = стратегия размещения; считает позиции из preferred/min/max size детей.
* Дефолты: `JPanel` → `FlowLayout`; content pane `JFrame` → `BorderLayout`.
* `BorderLayout` — 5 зон, центр растягивается; `GridLayout` — равные ячейки; `GridBagLayout` — сетка с весами и constraints.
* `null`-layout с абсолютными координатами — антипаттерн; после изменений — `revalidate()` + `repaint()`.
* В JavaFX layout — это сами контейнеры: `HBox`, `VBox`, `BorderPane`, `GridPane`, `StackPane`.
