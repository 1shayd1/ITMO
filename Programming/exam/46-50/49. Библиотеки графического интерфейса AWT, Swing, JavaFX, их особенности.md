# 49. Библиотеки графического интерфейса AWT, Swing, JavaFX, их особенности

Три поколения GUI-стеков Java; знание различий = понимание архитектуры рендеринга и потоков.

## `AWT` (Abstract Window Toolkit, 1995)

* `java.awt`; каждый компонент имеет **нативный peer** — реальный виджет ОС (**heavyweight**);
* «наименьший общий знаменатель» возможностей платформ → бедный набор компонентов, внешний вид зависит от ОС;
* остался **фундаментом**: событийная модель, `Graphics`/Java 2D, очередь событий (EDT) — всё это AWT, поверх которого живёт Swing.

## `Swing` (1998, `javax.swing`)

* Компоненты **легковесные**: рисуются Java 2D внутри одного нативного окна → одинаковое поведение на всех ОС;
* **Pluggable Look & Feel**: отрисовка делегирована UI-делегатам (`ComponentUI`) → темы Metal, Nimbus, системная;
* архитектура «**разделяемая модель**» (вариация MVC): данные отделены от вида — `JTable`+`TableModel`, `JList`+`ListModel`, `JTree`+`TreeModel`;
* богатые компоненты: `JTable`, `JTree`, `JTabbedPane`, текстовые компоненты;
* однопоточная модель: всё в **EDT** (`SwingUtilities.invokeLater`, `SwingWorker`).

## `JavaFX` (2008+, с JDK 11 — отдельный проект OpenJFX)

* **Граф сцены (scene graph)** — декларативное дерево узлов `Node`, рендеринг retained-mode с **аппаратным ускорением** (конвейер Prism);
* стилизация через **CSS**; декларативная разметка **FXML** (+ SceneBuilder) с контроллерами;
* **properties & bindings**: observable-свойства (`StringProperty` и т.д.) и связывание `label.textProperty().bind(field.textProperty())` — реактивное обновление UI;
* анимации (`Timeline`, transitions), эффекты, медиа, `WebView`;
* свой UI-поток — **JavaFX Application Thread** (`Platform.runLater`);
* начиная с JDK 11 не входит в JDK — подключается как зависимость/модули (`javafx.controls`, `javafx.fxml`), что и означает «требует отдельной настройки».

## Сравнение

| | `AWT` | `Swing` | `JavaFX` |
|---|---|---|---|
| Компоненты | нативные (heavyweight) | легковесные (Java 2D) | scene graph узлы |
| Внешний вид | как у ОС | pluggable L&F | CSS-стили |
| Разметка | код | код | код + FXML |
| Рендеринг | ОС | софтверный Java 2D | GPU (Prism) |
| UI-поток | EDT | EDT | JavaFX Application Thread |
| Статус | фундамент/легаси | классика, поддерживается | современный, отдельно от JDK |

Интероперабельность: `SwingNode` (JavaFX внутри Swing-контента наоборот: Swing-компонент в JavaFX), `JFXPanel` (JavaFX-сцена внутри Swing).

### Шпаргалка

* AWT — нативные peer-компоненты, общий знаменатель платформ; остаётся фундаментом (события, Java 2D, EDT).
* Swing — легковесные компоненты + pluggable Look & Feel + разделяемая модель (`JTable`/`TableModel`).
* JavaFX — граф сцены, CSS, FXML, properties/bindings, GPU-конвейер Prism.
* Потоки: Swing — EDT, JavaFX — Application Thread; правило «UI только из UI-потока» одинаково.
* С JDK 11 JavaFX (OpenJFX) — отдельная зависимость, не часть JDK.
