# 48. Обработка событий графического интерфейса. События и слушатели

GUI реагирует на действия пользователя через **делегирующую модель событий**: *источник → объект-событие → слушатели*. Это реализация паттерна **Observer**.

## Три участника

* **Источник** (кнопка, поле, окно) — генерирует события и хранит список слушателей;
* **Событие** — объект с информацией о произошедшем (`EventObject` → `AWTEvent` → `ActionEvent`, `MouseEvent`, `KeyEvent`, `WindowEvent`, `FocusEvent`...); метод `getSource()` возвращает источник;
* **Слушатель (listener)** — объект, реализующий интерфейс-обработчик; регистрируется методом `addXxxListener` (слушателей может быть несколько, порядок вызова не специфицирован).

## Swing

```java
JButton button = new JButton("OK");
button.addActionListener(e -> System.out.println("Clicked: " + e.getActionCommand()));
```

* `ActionListener` — функциональный интерфейс (`actionPerformed(ActionEvent)`) → можно лямбдой;
* многометодные интерфейсы (`MouseListener` — 5 методов, `WindowListener` — 7) имеют классы-**адаптеры** (`MouseAdapter`, `WindowAdapter`) с пустыми реализациями — переопределяется только нужное:

```java
field.addMouseListener(new MouseAdapter() {
    @Override public void mouseClicked(MouseEvent e) { /* ... */ }
});
```

## Потоки: ключевой контракт

* Все обработчики Swing вызываются в **EDT** → обработчик обязан быть **быстрым**, иначе интерфейс зависает;
* долгая работа — в фоне: `SwingWorker` (`doInBackground()` в фоне, `done()`/`process()` в EDT) либо `ExecutorService` + `SwingUtilities.invokeLater` для обновления UI.

## JavaFX

```java
Button btn = new Button("OK");
btn.setOnAction(e -> System.out.println("Clicked"));          // EventHandler<ActionEvent>
btn.addEventHandler(KeyEvent.KEY_PRESSED, e -> { /* ... */ });
```

* Единый функциональный интерфейс `EventHandler<T extends Event>`; иерархия `Event` → `ActionEvent`, `InputEvent` (`MouseEvent`, `KeyEvent`)...
* **Диспетчеризация в две фазы** по графу сцены:
  * **capturing** (сверху вниз, от `Stage` к узлу) — перехват через `addEventFilter`;
  * **bubbling** (снизу вверх) — обработка через `addEventHandler`/`setOnX`;
  * `event.consume()` останавливает дальнейшее распространение.
* Обработчики выполняются в **JavaFX Application Thread**; обновление UI из других потоков — `Platform.runLater`.

## Схема работы

1. Пользователь действует (клик).
2. Тулкит создаёт объект события и ставит в очередь UI-потока.
3. Событие доставляется зарегистрированным слушателям.
4. Слушатель выполняет реакцию (быстро!).

### Шпаргалка

* Делегирующая модель: источник + объект-событие + слушатели (`addXxxListener`) — это паттерн Observer.
* `ActionListener` функционален → лямбда; для многометодных интерфейсов есть адаптеры (`MouseAdapter`).
* Все обработчики идут в UI-потоке (EDT / JavaFX AT) → длинные задачи в фон (`SwingWorker`, `Platform.runLater` для ответа).
* JavaFX: фазы capturing (фильтры) → bubbling (хендлеры); `consume()` останавливает событие.
* Несколько слушателей на источнике допустимы; порядок вызова не гарантирован.
