# 47. Контейнеры верхнего уровня. Классы JFrame, SwingUtilities, Stage, Application

**Контейнеры верхнего уровня** — корни GUI-иерархии: окна, в которых живут все остальные компоненты. В Swing это `JFrame`/`JDialog`, в JavaFX — `Stage`.

## `JFrame`

Главное окно Swing (тяжеловесное). Внутреннее устройство:

```
JFrame
 └── JRootPane
      ├── glassPane (прозрачный слой сверху — перехват событий)
      └── layeredPane
           ├── menuBar (JMenuBar)
           └── contentPane  ← сюда добавляются компоненты
```

* `frame.add(...)` с Java 5 делегирует в `contentPane`;
* типовой стартовый код:

```java
JFrame frame = new JFrame("App");
frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
frame.add(mainPanel);
frame.pack();                       // размер по preferred size содержимого
frame.setLocationRelativeTo(null);  // по центру экрана
frame.setVisible(true);
```

## `SwingUtilities` и EDT

Swing **однопоточен**: все создания/изменения UI должны происходить в **Event Dispatch Thread (EDT)** — потоке, разбирающем очередь событий. Нарушение → гонки и редкие баги.

* `SwingUtilities.invokeLater(Runnable)` — поставить задачу в очередь EDT (асинхронно). Так запускают UI из `main`:

```java
SwingUtilities.invokeLater(() -> new JFrame("App").setVisible(true));
```

* `invokeAndWait(Runnable)` — то же, но блокирует вызывающий поток до выполнения (нельзя звать из самого EDT — deadlock);
* `isEventDispatchThread()` — проверка текущего потока;
* обратная сторона: **долгие операции нельзя выполнять в EDT** (UI замёрзнет) — выносить в фоновые потоки (`SwingWorker`).

## `Application` и `Stage` (JavaFX)

`Application` — базовый класс JavaFX-приложения с жизненным циклом:

```java
public class App extends Application {
    @Override
    public void start(Stage stage) {        // JavaFX Application Thread
        stage.setScene(new Scene(new StackPane(new Label("Hi")), 300, 200));
        stage.show();
    }
    // запуск: Application.launch(App.class, args);
}
```

Жизненный цикл: `launch()` → конструктор → `init()` (не UI-поток) → `start(Stage primaryStage)` (UI-поток) → ... → `stop()`.

* `Stage` — окно: `setScene`, `show()/hide()`, заголовок, модальность (`initModality`), стиль (`initStyle`); первичный `Stage` создаёт сама платформа и передаёт в `start`;
* аналог однопоточного правила: весь UI — в **JavaFX Application Thread**; аналог `invokeLater` — `Platform.runLater(Runnable)`;
* `launch()` можно вызвать только один раз на JVM.

## Параллель Swing ↔ JavaFX

| Swing | JavaFX |
|---|---|
| `JFrame` | `Stage` |
| content pane + компоненты | `Scene` + граф узлов |
| EDT | JavaFX Application Thread |
| `SwingUtilities.invokeLater` | `Platform.runLater` |

### Шпаргалка

* `JFrame` внутри: `JRootPane` → glassPane + layeredPane (menuBar, **contentPane**); `add` идёт в contentPane.
* Правило одного потока: весь Swing-UI — в EDT; старт через `SwingUtilities.invokeLater`.
* `invokeAndWait` блокирует и запрещён из EDT; долгие задачи — вне EDT (`SwingWorker`).
* JavaFX: `launch` → `init()` → `start(Stage)` в JavaFX Application Thread → `stop()`.
* `Stage` — окно, в него ставится `Scene`; аналог `invokeLater` — `Platform.runLater`.
